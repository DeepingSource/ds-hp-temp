처음 오셨나요

이곳은 DEEPINGSOURCE 홈페이지의 글과 카피를 코드 없이 직접 고치는 곳입니다.

  2. 왼쪽 메뉴에서 고칠 문서를 고릅니다. 저장하면 자동으로 커밋되고, 약 3분 뒤 사이트에 반영됩니다.

블로그 글 쓰기

  1. 블로그 글 → Create entry 로 새 글을 만듭니다.
  2. 제목·슬러그·요약·카테고리·게시일은 필수입니다. 읽기 시간은 비우면 자동 계산됩니다.
  3. 초안으로 두면(체크) 사이트에 노출되지 않습니다. 완성해서 발행할 때 체크를 해제하세요.
  4. 커버 이미지: 2:1 비율, .webp, 500KB 이하를 권장합니다. 대체 텍스트도 채워주세요(접근성).
  5. 본문 이미지: 본문 에디터에 바로 붙여넣거나 끌어다 놓으면 업로드됩니다.
  6. 예약 발행: 게시일을 미래 날짜로 두면 그날 00시(KST)에 자동 발행됩니다.
  7. 본문 블록: + 메뉴에서 Stat·Tip·Checklist·Callout·Quote·Source·PrivacyNote 를 넣을 수 있습니다.
    • 강조하고 싶은 한 줄 팁 → Tip, 주의·경고·성공 안내 박스 → Callout, 숫자 지표 → Stat.

케이스스터디 작성하기

「케이스스터디」 그룹에서 새 사례를 만들면, 재빌드 후 /resources/case-studies 목록과 개별 상세 페이지(/resources/case-studies/슬러그)에 자동으로 쌓입니다. 블로그와 같은 편집 루프(초안·예약 발행·언어별 파일)를 씁니다.

  1. 언어별로 파일 1개입니다. 한국어는 슬러그에 -ko, 일본어는 -jp를 붙입니다(예: smb-53-ko). 영어는 접미사 없음.
  2. Golden Case 단계를 하나 고릅니다(발견·검증·번역·전파·재측정). 한 단계에 여러 사례가 쌓여도 됩니다.
  3. 핵심 숫자(metrics)·**인용(quotes)**은 「항목 추가」 버튼으로 늘립니다. 지표는 카드에 4칸 그리드로 자동 정렬됩니다.
  4. ⚠️ 「실측 여부」 체크박스는 신뢰와 직결됩니다. 실제로 현장에서 측정한 값만 체크하세요. 체크하면 파란 실측 배지, 해제하면 회색 예시 배지가 붙습니다. 가상·추정 수치를 실측으로 표기하지 마세요 — 상단 면책 문구가 방문자에게 실측 표본 범위를 약속합니다.
  5. 커버 이미지는 선택입니다. 없으면 산업 아이콘 배지로 대체됩니다.
  6. **심화 서술(본문)**은 선택입니다. 채우면 상세 페이지 하단에 추가로 렌더링됩니다.
  7. 발행 판단: 도입 후 충분한 기간이 지나고, 고객 합의와 KPI 측정이 끝난 사례만 올리세요.

페이지 카피 고치기

  • 문서를 열어 KO/EN/JP 칸을 고치고 저장하면 됩니다.
  • {n}, {COMPANY} 같은 중괄호 토큰은 절대 지우지 마세요. 숫자·회사명이 자동으로 채워지는 자리입니다.
  • 저장 = 즉시 배포(약 3분). 되돌리려면 관리자(개발자)에게 요청하세요.

자주 묻는 질문 (편집자용)

  • 저장했는데 사이트에 안 보여요 — 배포에 약 3분 걸립니다. 브라우저 새로고침(캐시)도 해보세요.
  • 미래 날짜로 쓴 글이 안 보여요 — 정상입니다. 게시일이 되면 자동으로 뜹니다(예약 발행).
  • 이미지가 깨져요 — 본문 이미지는 에디터에 직접 업로드해야 합니다. 외부 주소를 붙여넣지 마세요.
  • 실수로 지웠어요 — 저장 전이면 새로고침으로 취소됩니다. 이미 저장했다면 관리자에게 복구를 요청하세요(git 이력에 남습니다).
  • 둘이 같은 문서를 동시에 열었어요 — 마지막에 저장한 사람 것이 남습니다. 미리 조율하세요.
