見るAIを超え、運営するAIへ。

次を動かす

ダッシュボードは数字を見せます。store agent は、その数字で何をすべきかまで伝え、店舗の次の一手をともに決めます。

数字から、次の一手まで

店舗を完璧にするには、見るべきものが多すぎます。store agent がその負担を分かち合い、今すべきことを示します。

01

今、棚から補充しますか？

ランチタイムの学生来店が普段より18%多めです。人気ドリンク3種が1時間以内に品切れしそうです。バックルームの在庫で今すぐ補充するか、お知らせします。

02

言葉ではなく、発注書で

繰り返すパターンは発注へつなげます。明日のランチ需要まで見越して、おにぎり40個を開店前入荷に — 発注書をその場で作成します。

03

推奨はAI、決定は人

推奨し、承認を受け、実行し、結果から学びます。推奨はAIが、決定はいつも人が。

店舗の自律運営のはしご

推奨から始め、店舗が自ら運営する段階まで、ともに上っていきます。推奨はAIが、決定は人が。

サンプル画面
saai|store agent

自律走行店舗 (Autonomous Store)

L0からL5まで — 一段ずつ上がります

最初から自動ではありません。カテゴリーごとに推奨 → 承認 → 自動 → 検証 → 学習のはしごを一段ずつ上がります。しかし「決定と責任」は最後まで人の場所です。

  1. 観察のみ

    人がすべての決定 (レガシーダッシュボード)

  2. 情報案内

    文脈アラート (天気・シーズン・イベント)

  3. 優先順位の推奨

    1〜3位 + 理由 (巡回動線・陳列整理)

  4. [承認] アクション

    具体アクション + シミュレーション (発注調整・勤務表)

  5. 低リスク自動 + 事後

    反復・低リスクの自動化 (アラート閾値・反復発注)

  6. 自律走行店舗

    人は戦略・責任・おもてなしのみ

次の段階へ上がる条件 — 店主の累積承認率80%以上、結果検証データ30回以上、本部・店主の明示的な合意。衝突時は直ちに一段ロールバックします。
