## 1. 최종 구조

- **기능 라이브러리(도구, 3모드 가로지름) = count · queue · pop · fit** (4개)
- **재배치:** gaze → saai ads insight(신규 제품) / keep·shelf → store care 감지 항목 / trail → 기술(Spatial AI·MTMC) / census → 기술(Vision Models) / talk → 숨김
- URL: `/products/store-{name}` · 라벨 DETECT·ANALYZE·ACT

## 2. functions hub 매트릭스 (4개 도구 × 3모드)

> (H1) 기능은 세 모드를 가로지릅니다
> (sub) 기능은 한 제품의 것이 아닙니다. 같은 기능을 care는 지금으로, insight는 어제로, agent는 다음으로 통과시킵니다.

| 기능 | care · DETECT·지금 | insight · ANALYZE·어제 | agent · ACT·다음 |
|---|---|---|---|
| **count** — 유입·재실 | 지금 재실 인원·혼잡도 감지, 임계 초과 알림 | 일·주·월 방문 시계열·추세 | 방문 예측 기반 인력·영업시간·프로모션 제안 |
| **queue** — 대기·혼잡 | 대기열·혼잡 임계 초과 실시간 알림 | 대기시간 피크·혼잡 주기 분석 | 인력 재배치·카운터 증설·셀프 유도 제안 |
| **pop** — 판촉물 | 게시된 POP 노출·훼손 감지 | POP별 주목·전환 효과 분석 | POP 문구·위치·교체 주기 제안·생성 |
| **fit** — 트렌드 적합 | 신상품 초기 반응 이상 감지 | 상품·매장 트렌드 적합도 분석 | 발주·상품 구성·바이어 의사결정 제안 |

> (하단) count는 기능입니다 — 네 번째 제품이 아닙니다. → store count

## 3. 도구 페이지 카피 (4개)

### 3.1 store count — 유입·상권 (라이브 · 반영사항만)
- ✅ 구조 확정, 정확도·설치 현행 유지.
- 반영: 표기 **store count** 통일 · "다음 한 수" → "다음" · **OBSERVE·입문 도구** 라벨 + "saai insight로 가는 첫 걸음" 명시 · URL `/products/store-count`.

### 3.2 store queue — 대기·혼잡  ( /products/store-queue )
> (eyebrow) 기능 · store queue · 대기·혼잡
> (H1) **줄이 길어지는 순간, 손님은 말없이 나갑니다**
> (sub) 계산대 대기(시간·인원)와 매장 혼잡을 실시간으로 재 시각화 — 붐비기 전에 인력을 옮길 수 있게.
> (안심) 개인 식별 없이, 대기·혼잡만
>
> **지금까지의 한계** — 줄은 길어진 뒤에야 눈에 띕니다. 언제 붐빌지 미리 알면 인력을 옮길 수 있는데, 그 예측이 없었습니다.
> **해결** — 대기열(대기 시간·대기 인원)과 매장 혼잡을 실시간으로 재, 임계를 넘기 전에 알립니다.
> **무엇을 알 수 있나** — 대기시간(지금 몇 분) · 대기 인원(몇 명) · 혼잡 주기(밀물·썰물) · 피크(요일·시간대)
> **3모드** — care: 대기열·혼잡 임계 초과 실시간 알림 / insight: 대기시간 피크·혼잡 주기 분석 / agent: 인력 재배치·카운터 증설·셀프 유도·혼잡 완화 제안
> **목업** — 대기 인원·시간 실시간 게이지 + 혼잡 피크 차트(예시). 연결: store count · saai care · saai agent.

### 3.3 store pop — 판촉물(POP)  ( /products/store-pop )
> (eyebrow) 기능 · store pop · 판촉물(POP)
> (H1) **붙여둔 POP이, 정말 보이고 있나요?**
> (sub) 게시된 판촉물의 노출·주목·전환을 재고 — 문구·위치·교체 시점까지 제안합니다.
> (안심) 개인 식별 없이, 반응만
>
> **지금까지의 한계** — POP은 붙이고 나면 효과를 모릅니다. 제대로 붙었는지, 눈에 들었는지, 매출로 이어졌는지 확인할 방법이 없었습니다.
> **해결** — 게시 상태(노출·훼손)부터 시선·전환까지 재, POP 한 장의 실제 효과를 숫자로.
> **무엇을 알 수 있나** — 게시 상태(붙었나·가려졌나·훼손) · 주목(본 비율) · 전환(구매로 이어진 비율)
> **3모드** — care: 게시된 POP 노출·훼손 상태 감지 / insight: POP별 주목·전환 효과 분석 / agent: POP 문구·위치·교체 주기 제안·생성
> **목업** — POP 효과 카드(주목·전환) + 게시 상태 체크리스트(전 매장). 연결: saai ads insight · saai agent(생성).

### 3.4 store fit — 트렌드 적합  ( /products/store-fit )
> (eyebrow) 기능 · store fit · 트렌드 적합
> (H1) **잘 나갈 상품인지, 진열 첫 주에 압니다**
> (sub) 신상품이 이 매장에 맞는지 — 시선·집어듦·재방문으로 트렌드 적합도를. 감이 아니라 첫 주 데이터로.
> (안심) 개인 식별 없이, 상품 반응만
>
> **지금까지의 한계** — 신상품의 운명은 보통 한 달 뒤에 압니다. 왜 안 팔렸는지(안 보여서·안 끌려서·집었다 놓아서)는 POS에 없습니다.
> **해결** — 진열 첫날부터 상품 앞 시선·집어듦·재방문을 익명으로 재, 이 매장·손님층에 맞는지 점수로.
> **무엇을 알 수 있나** — 관심도(본 비율) · gaze→pick(집은 비율) · 매장 적합(또래 대비)
> **3모드** — care: 신상품 초기 반응 이상 감지 / insight: 상품·매장 트렌드 적합도 분석 / agent: 발주·상품 구성·바이어 의사결정 제안
> **목업** — 적합도 스코어 카드 + gaze→pick 퍼널. 연결: saai ads insight · store shelf · saai agent.

## 4. 재배치 콘텐츠 (도구 아님 — 소스 보존)

### 4.1 → saai ads insight (신규 제품) · gaze 소스
- 대상: 사이니지·전시물 + 매대 상품. 지표: 통행·접근·주목(시선)·행동(interaction, opt).
- 3모드 소스: care "신규 진열·POP·사이니지 앞 시선 반응 실시간 확인" / insight "대상물별 주목도(gaze→pick·접근 전환) 분석" / agent "골든존·게시 위치·노출 우선순위 제안".
- → 별도 제품 페이지에서 정식 설계.

### 4.2 → store care 감지 항목 · keep·shelf 소스
- **keep(손실 방지):** 이상 행동·이탈·분실 위험 실시간 감지 (신원 아닌 위험 행동만).
- **shelf(매대·재고):** 결품(out-of-stock)·흐트러짐·가격표 오류·오염 실시간 감지.
- → store care 페이지 '감지 항목' 섹션으로 편입(감지 카탈로그).

### 4.3 → 기술(Spatial AI / MTMC) · trail 소스
- 구역별 통과·체류 히트맵, 대표 경로, 이상 동선. MTMC 기본 기능 → 기술 페이지 콘텐츠.

### 4.4 → 기술(Vision Models) · census 소스
- person attribute(연령대·성비 추정) — 모델 기본 배경 기능. 신원 식별 아님. → Vision Models 카탈로그.

### 4.5 → 숨김(보관) · talk
- 다국어 응대(외국어·도움 요청 감지, 문의 유형·언어 분석, 안내 문구 생성). 페이지 비노출, 카피는 보관.
