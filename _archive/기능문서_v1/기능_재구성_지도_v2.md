## 2. 정리된 제품·기능 구조

**기능 라이브러리 (3모드 가로지르는 도구) = 4개**
- count · queue · pop · fit
- → functions_국문화 매트릭스를 이 4개로 축소(각 care·insight·agent 3모드 유지).

**saai suite (3모드)** — 변화 없음: insight · care · agent
- **store care 하위 '감지 항목'** 섹션 신설: keep(이상·분실) · shelf(결품·오염) 등. care 페이지에 감지 카탈로그 추가.

**🆕 saai ads insight (별도 제품)**
- 대상: 사이니지 · 전시물 · 매대 상품
- 지표: 통행(지나감) · 접근 · 주목(시선) · 행동(interaction · optional)
- 매장 운영(suite)과 다른 축 — 광고·전시 효과 측정. gaze의 매대 gaze→pick·골든존도 여기로.
- → 별도 페이지 설계 필요.

**기술(technology)로 이동**
- Spatial AI / MTMC 페이지 ← trail(동선·경로) 내용
- Vision Models 페이지 ← census(person attribute) 내용

**숨김**
- talk (다국어 응대) — 보류, 페이지 비노출.

## 3. 초안 재배치 (기존 작업 활용)

- Wave 2/3에서 쓴 trail·census·keep·shelf·talk 초안은 폐기 아님 → 각 목적지로 이동:
  - trail → 기술(Spatial AI/MTMC) 콘텐츠 소스
  - census → 기술(Vision Models) 콘텐츠 소스
  - keep·shelf → store care '감지 항목' 소스
  - talk → 보관(숨김)
- gaze 초안 → saai ads insight 설계 시 매대 파트 소스로.

## 4. 남은 설계 작업

1. **queue 페이지** — 대기열(시간·인원 시각화) + 혼잡. (wave3 wait 초안 리네임·보강)
2. **saai ads insight** — 별도 제품 페이지 신규 설계(위상·구조·카피).
3. **store care 감지 항목 섹션** — keep·shelf 등 감지 카탈로그.
4. **기술 페이지 보강** — Spatial AI/MTMC(trail), Vision Models(census).
5. **functions 매트릭스 축소** — count·queue·pop·fit 4개로.
6. **count 반영** — 네이밍 store count 통일 · "다음 한 수" 제거 · OBSERVE 명시 · URL /products/store-count.
