## 세로축 — 세 모드

> (H) 제품은 세 개의 모드입니다
> (sub) 무엇이 셋을 가르냐면 — 같은 기능을 **어떤 시간축·목적으로 통과시키는가**입니다.
>
> · **DETECT · 지금** — saai care — "지금 무슨 일이 일어나고 있는가?"
> · **ANALYZE · 어제** — saai insight — "어제까지 무슨 일이 있었는가?"
> · **ACT · 다음** — saai agent — "다음에 무엇을 할까?"

## 12개 기능 × 3모드 매트릭스

| 기능 | saai care · DETECT·지금 | saai insight · ANALYZE·어제 | saai agent · ACT·다음 |
|---|---|---|---|
| **count** — 방문·재실 인원 | 지금 재실 인원·혼잡도 감지, 임계 초과 알림 | 일·주·월 방문 시계열, 요일·시간대·전년 대비 추세 | 방문 예측 기반 인력 배치·영업시간·프로모션 타이밍 제안 |
| **census** — 익명 인구통계 | 지금 방문객 구성(연령대·성비 추정) 파악 | 시간대·요일별 방문객 결(segment) 변화 분석 | 타깃 세그먼트 맞춤 상품 구성·시간대 운영 제안 |
| **trail** — 이동 동선 | 특정 구역 이상 정체·비정상 동선 감지 | 구역별 통과·체류 히트맵, 주요 경로 분석 | 진열·레이아웃·안내 동선 개선 제안 |
| **gaze** — 시선·주목(gaze→pick) | 신규 진열·POP 앞 시선 반응 실시간 확인 | 매대·상품별 주목도(gaze→pick 전환) 분석 | 골든존 배치·진열 우선순위 제안 |
| **wait** — 대기·줄 | 계산대·대기열 임계 초과 실시간 알림 | 대기시간 피크·요일/시간대 분석 | 인력 재배치·카운터 증설·셀프 유도 제안 |
| **tide** — 혼잡(밀물·썰물) | 매장 전체 혼잡 밀물/썰물 실시간 감지 | 혼잡 주기·피크 패턴 분석 | 혼잡 완화 운영(인력·프로모션 분산) 제안 |
| **keep** — 손실 방지 | 이상 행동·이탈·분실 위험 실시간 감지 | 손실 발생 구역·시간·패턴 분석 | 집기 배치·동선 차단·집중 관리 구역 제안 |
| **shelf** — 매대·재고 상태 | 결품·흐트러짐·가격표 오류 실시간 감지 | 결품 빈도·회전율·매대별 성과 분석 | 보충 주기·페이싱·진열 우선순위 제안 |
| **motion** — 행동 | 이상 행동(넘어짐·다툼 등) 실시간 감지 | 매장 내 행동 유형·빈도 분석 | 안전·서비스 개입 지점 제안 |
| **fit** — 트렌드 적합 | 신상품 초기 반응 이상 감지 | 상품·매장의 트렌드 적합도 분석 | 발주·상품 구성·바이어 의사결정 제안 |
| **pop** — 판촉물(POP) | 게시된 POP 노출·훼손 상태 감지 | POP별 주목·전환 효과 분석 | POP 문구·위치·교체 주기 제안·생성 |
| **talk** — 다국어 응대 | 외국어·도움 요청 상황 실시간 감지 | 문의 유형·언어·빈도 분석 | 다국어 응대·안내 문구 제안·생성 |

> **count는 기능입니다 — 네 번째 제품이 아닙니다.** → store count 자세히 보기

## 읽는 법

> (H) 읽는 법
> 하나의 **count**가 — care에선 **실시간 감지**로, insight에선 **추세 분석**으로, agent에선 **운영 제안**으로 나타납니다. 같은 기능, 세 개의 시간축.
>
> **한 줄이 늘면, 세 제품이 함께.**
> 새 기능은 표에 한 줄만 더해집니다 — 그리고 세 제품 경험이 함께 생깁니다. 새 제품도, 새 화면도 없이.
