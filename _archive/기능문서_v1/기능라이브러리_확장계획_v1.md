# 기능 라이브러리 확장 계획 v1 — 개별 기능 페이지 IA

근거: brand-system(Naming Grammar v1 §16 SOURCE 재정돈 · 5계열 · Now/Next/Later) + store count 페이지(표준 템플릿)

## 1. 정보구조 (IA)

```
/products/functions            ← Hub (12×3 매트릭스 · 5계열 그룹핑)
  ├ /products/store-count      (완료 · 현재 URL은 /products/saai-count → 정리 권장)
  ├ /products/store-fit
  ├ /products/store-pop
  ├ /products/store-trail …    (웨이브별 순차 개설)
```
- 각 개별 페이지 상단: **hub 회귀 링크** + 3모드 제품(care·insight·agent) 연결.

## 2. Hub 그룹핑 — 5계열 (brand-system)

| 계열 | 기능 |
|---|---|
| 들고남 | count · census |
| 흐름 | trail · gaze · wait |
| 지킴·상태 | keep · shelf |
| 만들기 | pop · fit |
| 응대 | talk |

- **motion·tide:** brand-system 통폐합 검토 대상(motion→trail·gaze, tide→wait 흡수, 기술 검증 조건부). 현재 사이트는 12개 노출. → **12 유지 vs 10 통폐합 결정 필요**(페이지화 전에).

## 3. 페이지화 우선순위 (웨이브)

| 웨이브 | 기능 | 근거 |
|---|---|---|
| **Wave 1** | count(완료) · **fit** · **pop** | 브랜드 Now(count·fit) + 사용자 지목(fit·pop) |
| **Wave 2** | trail · gaze · shelf · keep | 브랜드 Next |
| **Wave 3** | wait · census · talk (+ motion·tide 결정 후) | 브랜드 Later |

## 4. 개별 기능 페이지 템플릿 (store count 표준화)

1. 히어로 — 이 기능이 **무엇을 재는가** + 한 줄 가치 (+ "개인 식별 X" 안심)
2. 지금까지의 한계 — 없거나 · 비싸거나 · 감이거나
3. 해결 — 이 기능이 하는 일 (카메라 한 대로, 상시)
4. 무엇을 알 수 있나 — 핵심 지표 2~3
5. **3모드 통과** — care(지금 감지) / insight(어제 분석) / agent(다음 제안) : 매트릭스 행 확장
6. 목업·데모 — 해당 기능 화면
7. (해당 시) 정확도 · 설치 6단계
8. 연결 — 관련 기능·제품 + CTA(도입 상담)
- 상단 상시: functions hub 회귀.

## 5. URL · 표기 규칙

- 개별 페이지: **`/products/store-{name}`** (store fit · store pop …).
- 기존 count: `/products/saai-count` → **`/products/store-count`**로 정리 권장(제품명 store count와 일치).
- Hub 매트릭스 노출은 소문자 단독(fit·pop), 정식 표기·페이지는 `store {name}`.

## 6. 샘플 페이지 스케치

### store fit (트렌드 적합) — Wave 1
> (eyebrow) 기능 · store fit
> (H) **잘 나갈 상품인지, 진열 첫 주에 압니다**
> (sub) 신상품이 이 매장에 맞는지 — 시선·집어듦·재방문으로 트렌드 적합도를 읽습니다. 감이 아니라, 첫 주 데이터로.
> 3모드: care "신상품 초기 반응 이상 감지" / insight "상품·매장의 트렌드 적합도 분석" / agent "발주·상품 구성·바이어 의사결정 제안"

### store pop (판촉물) — Wave 1
> (eyebrow) 기능 · store pop
> (H) **붙여둔 POP이, 정말 보이고 있나요?**
> (sub) 게시된 판촉물의 노출·주목·전환을 재고 — 문구·위치·교체 시점까지 제안합니다.
> 3모드: care "게시된 POP 노출·훼손 상태 감지" / insight "POP별 주목·전환 효과 분석" / agent "POP 문구·위치·교체 주기 제안·생성"

## 7. 콘텐츠 소싱

- 각 기능의 3모드 문구는 이미 **매트릭스(functions_국문화_카피_v1.md)**에 존재 → 개별 페이지는 이를 확장.
- 만들기 계열(fit·pop)은 agent의 "생성"(POP 문구·발주서)과 강하게 연결 → 데모 여지 큼.

## 8. 결정 필요

1. **12 vs 10** — motion·tide 통폐합 여부(기술 검증).
2. **URL 규칙** — /products/store-{name} 채택 + count 정리.
3. **Wave 1 착수** — fit·pop 개별 페이지 카피 작성 진행 여부.
