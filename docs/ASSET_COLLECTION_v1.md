# ASSET_COLLECTION_v1 — Phase 2 와 병렬 자산 수집 계획 (D10)

> 마스터 플랜 `docs/PLAN_v1.1.md` §0.3 D10 · §5.2 · §5.3 의 *자산 수집 = Phase 2 와 병렬 진행* 결정을 실행 가능한 표·일정·결재 흐름으로 풀어쓴 문서. 본 문서는 Phase 2 시작 시점에 `assets/new/README.md` 색인의 부모 문서로 활용한다.

## 0. 한 페이지 요약

본 자산 수집은 새 회사 사이트가 *코드로 박을 수 있는 상태* 의 자산을 Phase 4 (빌드) 이전에 모두 확보하는 것을 목표로 한다. 인벤토리 4종을 교차 확인한 결과 *임원·오피스 사진은 0건*, *매체 로고 카드는 라이브에서 placeholder*, *파트너 로고 그리드는 부재* — 세 카테고리가 자산 격차의 핵심이다. 본 문서는 이를 **수집 대상 4 영역** (임원·오피스·매체 로고·파트너 로고) 과 **부수 영역 3종** (인포그래픽 신규 제작·영문 OG·새 Hero Key Visual) 으로 분리해 우선순위·발주·일정·결재를 정리한다.

| 일정 축 | 작업 | 결과물 |
|---|---|---|
| **Phase 2 시작 (D-0)** | 자리 식별 — 본 문서 §1·§2 확정 | 발주 가능한 spec 표 |
| **Phase 2 W1** | 외부 발주 — 사진 작가·인포그래픽 디자이너·비주얼 디자이너 섭외 | 계약서·일정표 |
| **Phase 2 W1-W2** | 매체·파트너 로고 raw 수집 | 30-40 매체 + N 파트너 로고 raw |
| **Phase 2 W2-W3** | 임원·오피스 촬영 | raw 컷 |
| **Phase 3 W1-W2** | 가공 — 흑백 통일·SVG 표준화·인포그래픽 재제작 | 박을 수 있는 자산 |
| **Phase 3 W2-W3** | 새 Hero Key Visual 제작 | 회사 사이트 hero 1-2종 |
| **Phase 4** | 새 사이트 코드에 자산 박음 | 라이브 |

예산·외부 발주가 필요한 영역은 §1.1 임원 사진 (사진 작가) · §1.2 오피스 사진 (동일 작가, 동시 진행) · §2.1 인포그래픽 신규 제작 (인포그래픽 디자이너) · §2.3 새 Hero Key Visual (비주얼 디자이너) — 총 외부 발주 4 트랙이다. 매체·파트너 로고는 *수집·정리* 작업이며 외부 발주가 아닌 SA Team 내부 작업으로 가정한다 (담당 결정 필요).

## 1. 수집 대상 4 영역 상세

### 1.1 임원 사진 (P0)

**자리 (사이트 IA 매핑)**

- `/company/about` — Vision/Mission 옆 임원 카드 그리드
- `/company/career` — 채용 페이지 상단 *우리 팀* 섹션
- `/company/investors` — D5 결과 신설된 IR 페이지의 경영진 소개

**필요 인물**

- 김태훈 (대표이사·CEO) — 라이브 deepingsource.io 의 About 페이지에서 확정
- 핵심 임원 N명 — CTO·COO·CMO 등 *실제 인물 구성은 Jamin 결재 필요* (현 인벤토리에는 임원진 구성표 부재)
- (추정) 임원 4-5명 규모로 가정

**촬영 spec**

| 컷 | 용도 | 비율 |
|---|---|---|
| 정면 1컷 | About 카드 메인 | 1:1 (320×320 / 640×640) |
| 정면-1/4 1컷 | About 카드 호버·디테일 | 3:4 (480×640) |
| 환경 1컷 | Career·Investors 보조 | 16:9 (1280×720) |
| 풀샷 1컷 | 보도자료·SNS 보조 | 4:5 (1080×1350) |
| 흑백 1컷 | 모노톤 톤·언론 인용용 | 1:1 |

- 백그라운드 — 회사 컬러(흑/회/브랜드 액센트) 단색 또는 사무실 환경 (Phase 3 디자인 시스템과 톤 통일)
- 광원 — 자연광 + 보조 1등 / 인공광 키 1등 + 필 1등 (작가 선택)

**라이선스**

- 회사 소유 — 회사 사이트 (영문·한글·일문 동시) · 보도 자료 · 채용 자료 · SNS · IR 자료 사용권 명문화
- 임원 본인 동의 — 게시 동의서 별도 (§4 결재 흐름 참조)

**외부 발주**

- 사진 작가 1명 — 반일 (4시간) · 임원 5명 가정 시 1명당 약 30분 셋업 + 15분 촬영

**산출**

- 임원당 4컷 컬러 + 1컷 흑백 = 5컷
- crop variants — 1:1·3:4·16:9·4:5 자동 생성 (작가가 raw 제출, SA Team 이 변환)
- 저장 — `assets/new/executives/{name}/{ratio}.{ext}` 구조 (§5 참조)

### 1.3 매체 로고 (P1) — News 페이지 핵심

**자리**

- `/company/news` — 88건+ 외부 매체 노출 카드 그리드 (현 라이브 deepingsource.io 의 News 페이지는 *모든 카드가 placeholder 아이콘* — 매체 로고 자산 0건)
- 보도자료 PDF·세일즈 자료 보조 (재사용 가능)

**필요**

- News 88건의 매체별 로고 raw — 중복 제거 후 *약 30-40 매체* 추정
- 한국 주요 IT 매체·일반지·방송 등 (구체적 매체 리스트는 News 인벤토리에서 추출 필요 — `assets/inventory/deepingsource.io/news_pages_inventory.json` 등 교차 확인)

**수집 방법**

| 우선 | 출처 | 가공 |
|---|---|---|
| 1순위 | 매체 공식 사이트의 *Press Kit* / Brand Center | SVG raw 다운로드 |
| 2순위 | Wikipedia Commons (Fair use 영역) | PNG → SVG 트레이싱 |
| 3순위 | 매체 PR 담당 컨택 | 이메일 요청 |

**가공 룰**

- 흑백 단일 톤 통일 (브랜드 컬러 무시 — 그리드에서 *덜 시끄러운* 톤 우선)
- SVG 우선 (스케일링 자유) · 부재 시 PNG 2x·3x
- 고정 높이 — 24px 또는 32px (Phase 3 디자인 시스템 토큰 결정 시 확정)
- 여백 정돈 — 모든 로고가 같은 *시각 무게* 로 보이도록 padding 조정

**라이선스**

- 보도 인용 용도 (Fair use) — *법무 검토 필요* (별도 트랙)
- 매체별 사용 정책 — 표 작성 후 위반 매체는 제외 또는 텍스트 캡션으로 대체

**산출**

- SVG/PNG 두 포맷 + 흑백 변형 1종 = 매체당 3 파일
- 그리드 박을 단일 sprite (SVG sprite 또는 CSS sprite — Phase 3 결정)
- 저장 — `assets/new/media-logos/{slug}/{name}.{svg|png|bw.svg}`

## 4. 결재 흐름 & 라이선스 정리

| 자산 | 결재자 | 동의서·계약서 | 사용 범위 명문화 |
