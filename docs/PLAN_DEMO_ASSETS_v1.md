## 1. 트랙 A — 실제 화면 확장 (소개서 + 직접 캡처)
실제 화면은 목업보다 신뢰도 높음. 제품별로 si-guide 패턴(제품 UI 스크린샷 폴더)을 확장.

**두 가지 소스 (직접 캡처 우선)**
- **A-1. 직접 스크린샷 확보 (Jamin, 우선)** — 라이브 대시보드에서 직접 캡처. 소개서 PDF 추출본보다 해상도·최신성 우수. 캡처 규격 통일: 동일 해상도·브라우저 크롬 제거·2x(레티나)·동일 매장/기간. 기능소개 v7의 "라이브 캡처(mock 없음)"와 같은 소스.
- **A-2. 소개서 PDF 캡처 (보조)** — 직접 캡처가 어려운 화면만 소개서에서 추출.

| 소스 (자료 모음 1) | 대상 위치 | 비고 |
|---|---|---|
| StoreAgent 기능소개 v7 (30 캡처) | saai-agent 페이지 | **2-6에서 별도 처리**(독립) — 여기선 링크만 |
| 딥핑소스_솔루션_소개_store insight (28p) | `si-guide/` 확장 + saai-insight 페이지 | 신규/갱신 화면만 선별. 가능하면 직접 재캡처 |
| 스토어케어 소개서 (12p) | 신규 `sc-guide/` + saai-care 페이지 | StoreCare 실제 화면이 현재 목업 위주 → 실캡처 보강 |
| StoreCount 설치사용설명서 (14p) | saai-count 페이지·Docs | 설치 플로우 화면 |
- **작업 방식**: 직접 캡처/PDF 선별 → 마스킹 판정(매장 실명·수치, J-3 원칙) → webp 최적화(quality 86) → `public/images/*-guide/` 배치 → 페이지 연결.
- **캡처 체크리스트(직접 캡처 시)**: 로그인·개인정보 노출 제거 · 동일 뷰포트 · 동일 데모 매장/기간 · 다크모드 여부 통일 · KO 우선(EN/JP는 언어 전환 후 동일 화면).
- **주의**: si-guide는 asset-prompts 재생성 **제외 대상**(실제 UI). 실캡처는 AI로 다시 만들지 않음.

## 2. 트랙 B — AI 생성 에셋 확장 (asset-prompts 파이프라인, **GPT-image 확정**)
기존 파이프라인에 **신규 프롬프트 추가**로 갭 메움. **생성 도구 = GPT-image (7/16 확정)** — 자연어 서술형 영어 프롬프트, `--ar` 등 파라미터 미사용(asset-prompts 기준과 일치). 반드시 `00-style-guide.md` STYLE ANCHOR를 앞에 붙여 생성.

**QA-REPORT가 남긴 갭·후속**
- 업종 히어로 일부에 원치 않는 CCTV 크롬 → 마케팅용 클린 매장 히어로로 재생성(편의점·뷰티·패션·창고).
- korea-map은 SVG 유지(래스터 금지).

**신규 프롬프트 후보 (asset-prompts에 12~14번 파일 추가)**
- 이벤트/박람회용 키비주얼 (EV-2 커버, 창업자 눈높이 — 상권·관리·POP 3-흐름 이소메트릭)
- 리드 마그넷/BOFU용 인포그래픽 (G-4 사례 모음집 표지, G-5 효과 추정 도식)
- 업종 확장 히어로 (리플렛 타깃: 무인매장·창업 맥락)
- 퍼널/파이프라인 다이어그램 (G-1 리드 퍼널 시각화)
- **작업 방식**: 프롬프트 작성(Claude) → 생성(GPT-image, Jamin/디자이너) → QA(스타일 앵커 정합·기능색·익명화) → webp 변환·배치 → QA-REPORT 갱신.
