# Docs 편집 가이드 (비개발자용)

작성일: 2026-07-24 · 근거: 페이지_수정_숨김_개발계획_2026-08 §3-4 (MASTER Phase 2)

`/resources/docs`에 노출되는 제품 문서(`content/docs/*.mdx`)를 Keystatic CMS로 편집하는 방법을 정리합니다.

## 1. 접속

1. 사이트 주소 뒤에 `/keystatic`을 붙여 접속합니다. (예: `https://www.deepingsource.io/keystatic`)
2. GitHub 계정으로 로그인합니다. (GitHub App `deepingsource-hp-cms` 권한 필요 — 권한이 없으면 개발팀에 요청)
3. 좌측 목록에서 **「docs」 컬렉션**을 선택합니다.

## 2. 문서 편집·새 문서

- 기존 문서를 클릭해 수정 → 우상단 **Save**. 저장하면 GitHub에 커밋(또는 PR)이 생성됩니다.
- 새 문서는 **Create** 버튼 → 아래 필드를 채웁니다.

| 필드 | 의미 |
|------|------|
| `title` / `slug` | 제목 / URL 경로 (`/resources/docs/<slug>`) |
| `excerpt` | 목록 카드에 보이는 한 줄 요약 |
| `section` | 사이드바 분류 — 선택지는 사이트 사이드바 순서와 일치 |
| `order` | 같은 섹션 안에서의 정렬 순서(숫자 작을수록 위) |
| `lang` | `ko` / `en` / `jp` |
| `draft` | **체크하면 사이트에서 숨김** (아래 3장) |
| `access` | `public`(공개) / `gated`(이메일 게이트) |

- 본문은 MDX — 표·콜아웃 등 커스텀 블록은 에디터의 블록 삽입 메뉴를 사용합니다.
- 이미지는 에디터에서 업로드하면 `public/images/docs/`에 저장됩니다.

## 3. 노출 토글 (숨김/복귀)

- `draft` 체크박스가 **켜져 있으면**: 사이드바·문서 상세·sitemap 어디에도 나오지 않습니다.
- 2026-08 개편으로 **편의점 운영 가이드(`storeagent-` 로 시작하는 문서 22종 × 3로케일)는 전부 draft 상태**입니다. 다시 노출하려면 해당 문서의 draft를 끄면 됩니다.
- ⚠️ 문서를 숨기거나 복귀할 때는 **ko/en/jp 3개 파일을 함께** 토글하세요. (파일명: `<slug>.mdx` / `<slug>-en.mdx` / `<slug>-jp.mdx`)

## 4. 번역 규칙

- 한국어 원본: `<slug>.mdx` (`lang: ko`)
- 영어/일본어 번역본: 같은 slug 뒤에 `-en` / `-jp`를 붙인 별도 문서 (`lang`도 맞춰 변경)
- 번역본이 없으면 en/jp 페이지에는 한국어 원본이 폴백으로 노출됩니다. 번역본을 만들면 자동으로 교체됩니다.

## 5. 저장 후 사이트 반영 시점

- 저장 = GitHub 커밋. **사이트에 보이려면 배포가 한 번 돌아야 합니다.**
- Vercel이 main 브랜치 커밋을 감지해 자동으로 재배포합니다(수 분 소요). 반영이 안 보이면 개발팀에 재배포를 요청하세요.

## 6. 문제가 생기면

- 저장이 안 될 때: GitHub 로그인 만료 → 로그아웃 후 재로그인.
- 필드를 잘못 넣었을 때: `npm run lint:frontmatter`가 CI에서 잡아줍니다 — 에러 메시지의 필드명을 확인해 수정.
- 그 외: 개발팀(SAAI@deepingsource.io)으로 문의.
