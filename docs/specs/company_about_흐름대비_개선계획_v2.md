문제(사용자): ① 한 흐름/호흡에 안 읽힘 ② 시각적 대비 부족 · 겸 **카피 정돈**
**v2 변경점:** 서사를 **미괄식 → 두괄식 선언문**으로 전환. Mission을 맨 앞에서 선언하고 "왜냐하면"으로 뒷받침. **확정 결정 2건 반영**(아래).
참조: 현행 소스(`AboutView.tsx`·`about.yaml`·`MasterPair`/`VisionDiagram`/`OriginStoryTimeline`/`VisionCoordinatesMockup`) + `company_about_재설계_v1.md`

## 0. 한 줄 요약

두 문제는 한 뿌리다 — 페이지가 *같은 볼륨의 블록 9개*다.
- **흐름:** 도입부(히어로)가 과적이고, 같은 메시지가 3~4번 반복되며, 서사에 결론이 없다.
- **대비:** 본문이 흰색↔`slate-50`(명도차 2%)으로만 교대해 경계가 안 보이고, 제목·컬러가 전 섹션 동일해 강약이 없다.

**처방:** (A) **두괄식 선언문**으로 재편 — "우리는 오프라인을 다시 만든다"를 먼저 선언하고 → 비전 → 5년 계획 → **"왜냐하면"(기원·증거)** → 누가 → 함께하자. (B) 다크 앵커로 명암 리듬을 끊는다. (C) 제목 위계 3단 + 파랑 앵커로 볼륨 차이를 만든다.

## 1. 문제 진단 (요약 — 상세는 v1 §1)

**"한 흐름에 안 읽힌다"**
- **히어로 과적 + 자기 중복:** "온라인은 최적화됐다 / 현장은 그대로였다"를 대비 카드로 한 번, 바로 아래 `companyIntro` 문장으로 또 한 번.
- **핵심 문구 과다 반복:** "모든 공간을, 완벽하게"가 히어로 kicker·V/M H2·Vision 카드 3곳.
- **결론이 없는 서사:** 현행은 미괄식(설명 후 결론)이라 "이 회사가 누구인지"가 끝까지 안 박힌다. → **두괄식으로 전환**(v2 핵심).
- **같은 개념 이중 시각화:** `VisionDiagram`(3층)과 How we work 3스텝이 같은 구조를 두 번, 다른 섹션에서.

**"시각적 대비가 부족하다"**
- 본문 5섹션 흰색↔`slate-50` 교대 → 경계 안 보임.
- `VisionDiagram`이 **다크용인데 흰 배경**에 얹혀 씻김·톤충돌(버그성).
- 9섹션 제목 전부 `text-3xl/4xl` 동일 → 위계 없음.
- 파랑(#376AE2)이 점·아이콘 미량으로만 → 색 앵커 부재.
- 카드가 전부 같은 처리(`rounded` + `border-gray-100`) → 도형/배경 분리 실패, 하단 소카드 난립.

## 2. 개선 원칙 — 두괄식 선언문

1. **결론부터.** "우리는 오프라인을 다시 만드는 회사"라고 **먼저 선언**하고, 뒤에서 근거("왜냐하면")로 증명한다. 독자가 첫 화면에서 "누구인지"를 잡는다.
2. **하나의 서사선.** 블록 9개가 아니라 **"선언 → 비전 → 5년 계획 → 왜냐하면 → 누가 → 함께하자"** 한 이야기.
3. **볼륨 차이를 설계.** 다크 앵커 3개로 3막, 제목 3단계 + 파랑 앵커로 강약.

## 3. 정보 구조(IA) — 두괄식 재배열

같은 자산을 순서만 바꿔 선언문으로 재편. (MasterPair 제거 · 마일스톤은 타임라인 흡수)

| # | 장(章) | 역할 | 현행 자산 | 배경 |
|---|---|---|---|---|
| ① | **선언 — 우리는 오프라인을 다시 만든다** | Mission 두괄 선언 + 온라인↔현장 훅 **1줄** | 히어로(정리) | 다크 |
| ② | **우리가 가려는 곳 — 모든 공간을, 완벽하게** | Vision | 비전/미션 섹션 | 밝음-강조 |
| ③ | **향후 5년, 이렇게 합니다** | **목적지(2031) 먼저 → 방법(3층) 나중** | Vision 2031 + How we work + VisionDiagram 흡수 | **다크** |
| ④ | **왜냐하면 — 이건 중요한 문제였습니다** | 기원·창업자·연혁을 **"자격의 증거"**로 | Our Story + OriginStoryTimeline + 네이밍 | 밝음 |
| ⑤ | **누가 만드나** | 리더십 · 파트너·인증 | Leadership + Partners | 밝음(분리) |
| ⑥ | **함께하자** | CTA | CTA | 다크 |
| — | ~~MasterPair(본부/매장)~~ | **제거** | — | — |

**핵심 이동**
- **Mission을 히어로 선언으로 승격**(두괄식 진입).
- **③에서 미래(2031)를 먼저, 방법을 뒤로** — 섹션 안에서도 두괄식.
- **연혁·기원을 ④ "왜냐하면"으로** — 시간 역행이 아니라 *증거*로 프레이밍(선언 → 근거).
- **MasterPair 삭제, 마일스톤 4카드는 ④ 타임라인에 흡수.**

> **④의 프레이밍 주의:** 과거(2018)가 미래(2031)보다 뒤에 오지만, ④는 "우리의 역사"가 아니라 **"우리가 이 미래를 말할 자격 — 2018년부터 이 문제를 풀어왔다"는 근거**로 쓴다. 연혁 = 연대기가 아니라 증거.

## 4. 섹션별 실행안 (카피 초안 + 디자인)

> 카피는 `content/site/about.yaml`(Keystatic). 아래 국문은 **초안** — 대화로 다듬음.

### ① 선언 — 우리는 오프라인을 다시 만든다 (다크)
**카피(초안):**
- (eyebrow) About DEEPINGSOURCE
- (H1·대형) **REINVENT OFFLINE**
- (국문) 우리는 오프라인을, 다시 만듭니다.
- (훅 1줄) "온라인은 20년간 측정으로 진화했지만, 현장은 그대로였습니다. 우리는 그 현장을 다시 만듭니다."
**정돈:** 현행 히어로의 대비 카드(온라인은/현장은)와 `companyIntro` 문장이 **같은 말 반복** → **훅 1줄로 압축**. `companyLine`·`companyIntro2`·`perfectSpace` 화살표는 ②로 이관.
**디자인:** 선언(H1) + 국문 + 훅 1줄까지만. 여백을 크게. (현행 6요소 → 3요소)

### ② 우리가 가려는 곳 (밝음-강조)
**카피:**
- (eyebrow) Vision
- (H2) **모든 공간을, 완벽하게** — 당신의 공간에서, 모든 공간으로.
- (본문) 측정할 수 있으면, 나아질 수 있습니다. 온라인을 바꾼 그 힘을, 이제 물리 세계로.
**정돈:** "모든 공간을 완벽하게"는 **About 안에서 여기 1회만.** 미션은 ①에서 선언했으니 여기선 **비전 전용**(중복 제거).
**디자인:** 파랑 앵커 섹션(§5-3) — 카드 하나 파랑 채움. `VisionDiagram`은 여기서 빼서 ③으로.

### ③ 향후 5년, 이렇게 합니다 (다크) ★두괄식 핵심
**구성(확정): 목적지 먼저 → 방법 나중.**
- (상단) **Vision 2031 좌표** — "5년 뒤, 우리 자리": 익명화·공간·운영 교집합의 거의 유일한 위치.
- (하단) **어떻게 도달하나 — 3층:** `VisionDiagram`(개요 그림) → ①익명화 ②공간지능 ③에이전트 3스텝(상세).
**정돈:** VisionDiagram과 3스텝의 **이중 시각화**를 "개요 그림 → 상세 카드" 관계로 통합. closing은 ②와 겹치지 않게 **"권고는 AI가, 결정은 사람이."**(안전선).
**디자인:** **다크 배경** → `VisionDiagram`이 제 옷을 입어 대비·톤충돌 해소. 다크 위 `ProcessStepper` 카드 톤 반전 필요(§5-4).

### ④ 왜냐하면 — 이건 중요한 문제였습니다 (밝음)
**카피(초안):**
- (eyebrow) Why we exist
- (H2) **카메라는 결국, 사람을 봅니다.**
- (본문) 이건 그냥 넘길 문제가 아니었습니다. 개인정보 규제가 강해지던 2018년, 우리는 그 딜레마를 정면으로 풀기로 했습니다 — 촬영하는 그 순간 신원을 지우되, 분석에 필요한 특징은 남기는 '익명화된 AI'로. (이하 현행 `storySub`·창업자 여정·네이밍 유지)
- (타임라인) 2018 데이터 장벽 → … → 2025 고객과 함께 만든 답. **이 미래를 말할 자격의 증거.**
**정돈:** 현행 자산 대부분 유지. 마일스톤 4카드는 타임라인에 흡수(저대비 카드 난립 완화). 프레이밍만 "역사 → 증거"로.

### ⑤ 누가 만드나 (밝음, 리더십 ↔ 파트너 시각 분리)
리더십·파트너·인증 유지. §5-1 리듬으로 파트너를 리더십과 분리. 파트너 통계·인증 초점화(§5-4).

### ⑥ 함께하자 (다크)
CTA 유지 — "데이터센터가 아니라, 사람이 사는 세상을" + 문의하기.

## 5. 디자인 처방

### 5-1. 배경 명암 리듬 (다크 앵커 3개 = 3막)
`다크`(①선언) → `밝음-강조`(②비전) → **`다크`(③5년계획)** → `밝음`(④왜냐하면) → `밝음`(⑤누가) → `다크`(⑥CTA).
- 다크 3개(①·③·⑥)가 페이지를 3막으로 끊는다. MasterPair를 빼도 중앙 다크(③)가 리듬을 잡음.
- 밝은 구간끼리는 `slate-50` 대신 **얇은 구분선(hairline)** 또는 `slate-100` 이상으로 명도차 확대.

### 5-2. 타이포 위계 3단계
- **L1(선언·전환):** ①히어로, ③5년계획. `text-4xl sm:text-5xl+`.
- **L2(일반):** ②비전, ④왜냐하면, ⑤리더십. `text-3xl sm:text-4xl`.
- **L3(보조):** 파트너·인증. `text-2xl` + eyebrow 강조.

### 5-3. 파랑 앵커 (면·강조, 섹션당 1곳)
- ②비전 카드 하나 **파랑 채움** → "이게 우리의 약속"을 색으로.
- 각 장 eyebrow에 **파랑 룰/숫자 인덱스(①②③)**로 장 감각.
- 다크 섹션(③·⑥)에서 파랑을 **발광색**으로 크게.
- 남용 금지 — 도처에 있으면 다시 무앵커.

### 5-4. 카드 위계
- **primary 카드 / neutral 카드** 2톤 분리.
- 다크(③) 위 `ProcessStepper` 활성 카드 → `bg-white/10 border-primary/40`로 반전.
- 하단 소카드 난립 완화: 마일스톤→타임라인 흡수, 파트너 통계는 **큰 숫자+파랑** 4칸, 인증은 로고 스트립.

### 5-5. 여백
히어로처럼 요소 많은 곳은 여백 키워 하나씩 읽히게. 관련 정보(2031+방법, 기원+연혁)는 붙여서 한 덩어리로.

## 6. 카피 정돈 원칙
1. **한 섹션 = 한 메시지** (히어로 과적 해소 1순위).
2. **같은 문장 반복 금지** — "모든 공간을 완벽하게"·온라인↔현장 각 1회.
3. **중복이면 새 정보로 교체** (삭제만 하지 말 것).
4. **closing 차별화** — ②본문과 ③closing이 같은 "측정하면 나아진다"를 안 쓰게.
5. 나머지 자산(스토리 본문·3스텝·연혁·리더십·파트너) 품질 양호 → **유지.** 작업의 90%는 덜어내기와 순서.
