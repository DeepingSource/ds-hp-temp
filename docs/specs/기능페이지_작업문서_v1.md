# 기능 페이지 작업 문서 v1

각 페이지를 **"이렇게 만들자(구성)"** + **"카피는 이렇다"** 형식으로 정리한 단일 작업 문서.
작업일: 2026-07-20 · 이전 기능 문서(v1 waves·확장계획·국문화·v2 지도·정본)는 `_archive/기능문서_v1/`로 이관, 본 문서가 최신 기준.

## 0. 구조 & 공통 규칙

**기능 라이브러리(3모드 가로지르는 도구) = count · queue · pop · fit**
**재배치:** gaze → saai ads insight(신규 제품) / keep·shelf → store care 감지 항목 / trail·census → 기술(Spatial AI·Vision Models) / talk → 숨김

**공통 규칙**
- URL: `/products/store-{name}`  · 표기: 본문·페이지 `store {name}`, hub 매트릭스는 소문자 단독.
- 라벨: **DETECT**(care·지금) · **ANALYZE**(insight·어제) · **ACT**(agent·다음).
- 데이터엔 항상 "예시" 주석. 상단엔 항상 hub 회귀 링크. 프라이버시 안심 한 줄("개인 식별 없이, ~만").

**개별 도구 페이지 표준 골격(이렇게 만들자)**
① 히어로 → ② 지금까지의 한계 → ③ 해결 → ④ 무엇을 알 수 있나 → ⑤ 3모드 통과 → ⑥ 목업·데모 → ⑦ (해당 시)정확도·설치 → ⑧ 연결·CTA

## 1. 기능 라이브러리 hub  ( /products/functions )

**이렇게 만들자:** 히어로 + 3모드 정의 카드(care·insight·agent) + **4×3 매트릭스** + "count는 기능" 문구 + 각 도구로 링크.

**카피는 이렇다**
> (H1) 기능은 세 모드를 가로지릅니다
> (sub) 기능은 한 제품의 것이 아닙니다. 같은 기능을 care는 지금으로, insight는 어제로, agent는 다음으로 통과시킵니다.

| 기능 | care · DETECT·지금 | insight · ANALYZE·어제 | agent · ACT·다음 |
|---|---|---|---|
| **count** — 유입·재실 | 지금 재실·혼잡도 감지, 임계 초과 알림 | 일·주·월 방문 시계열·추세 | 방문 예측 기반 인력·영업시간·프로모션 제안 |
| **queue** — 대기·혼잡 | 대기열·혼잡 임계 초과 실시간 알림 | 대기시간 피크·혼잡 주기 분석 | 인력 재배치·카운터 증설·셀프 유도 제안 |
| **pop** — 판촉물 | 게시된 POP 노출·훼손 감지 | POP별 주목·전환 효과 분석 | POP 문구·위치·교체 주기 제안·생성 |
| **fit** — 트렌드 적합 | 신상품 초기 반응 이상 감지 | 상품·매장 트렌드 적합도 분석 | 발주·상품 구성·바이어 의사결정 제안 |

> (하단) count는 기능입니다 — 네 번째 제품이 아닙니다. → store count

## 2. store count — 유입·상권  ( /products/store-count )  ※ 라이브

**이렇게 만들자:** 현행 페이지 구조 유지(히어로→기능×3모드→문 밖/문 안→둘 중 하나였다→해결→무엇을 알 수 있나→차별점→설치 6단계→본사·확장→정확도 95~96%→프라이버시→클로징).

**카피는 이렇다 — 반영사항만**
- 표기 **store count**로 통일(eyebrow의 saai count → store count).
- 차별점 "상시 데이터로 **다음 한 수**를 정합니다" → "…다음을 정합니다".
- **OBSERVE · 입문 도구** 라벨 + "saai insight로 가는 첫 걸음" 한 줄 명시.
- URL `/products/saai-count` → `/products/store-count`.
- 정확도·설치 섹션은 현행 유지.

## 3. store queue — 대기·혼잡  ( /products/store-queue )

**이렇게 만들자:** 표준 골격. ⑥ 목업은 **대기 인원·대기 시간 실시간 게이지 + 혼잡 피크 차트**. (정확도·설치 섹션은 생략 — 하드웨어 특화 아님.)

**카피는 이렇다**
> (eyebrow) 기능 · store queue · 대기·혼잡
> (H1) **줄이 길어지는 순간, 손님은 말없이 나갑니다**
> (sub) 계산대 대기(시간·인원)와 매장 혼잡을 실시간으로 재 시각화 — 붐비기 전에 인력을 옮길 수 있게.
> (안심) 개인 식별 없이, 대기·혼잡만
>
> **② 한계** — 줄은 길어진 뒤에야 눈에 띕니다. 언제 붐빌지 미리 알면 인력을 옮길 수 있는데, 그 예측이 없었습니다.
> **③ 해결** — 대기열(시간·인원)과 매장 혼잡을 실시간으로 재, 임계를 넘기 전에 알립니다.
> **④ 무엇을** — 대기시간(지금 몇 분) · 대기 인원(몇 명) · 혼잡 주기(밀물·썰물) · 피크(요일·시간대)
> **⑤ 3모드** — care: 대기열·혼잡 임계 초과 실시간 알림 / insight: 대기시간 피크·혼잡 주기 분석 / agent: 인력 재배치·카운터 증설·셀프 유도·혼잡 완화 제안
> **⑧ 연결** — store count · saai care · saai agent → 도입 상담

## 4. store pop — 판촉물(POP)  ( /products/store-pop )

**이렇게 만들자:** 표준 골격. ⑥ 목업은 **POP 효과 카드(주목·전환) + 게시 상태 체크리스트(전 매장)**.

**카피는 이렇다**
> (eyebrow) 기능 · store pop · 판촉물(POP)
> (H1) **붙여둔 POP이, 정말 보이고 있나요?**
> (sub) 게시된 판촉물의 노출·주목·전환을 재고 — 문구·위치·교체 시점까지 제안합니다.
> (안심) 개인 식별 없이, 반응만
>
> **② 한계** — POP은 붙이고 나면 효과를 모릅니다. 제대로 붙었는지, 눈에 들었는지, 매출로 이어졌는지 확인할 방법이 없었습니다.
> **③ 해결** — 게시 상태(노출·훼손)부터 시선·전환까지 재, POP 한 장의 실제 효과를 숫자로.
> **④ 무엇을** — 게시 상태(붙었나·가려졌나·훼손) · 주목(본 비율) · 전환(구매로 이어진 비율)
> **⑤ 3모드** — care: 게시된 POP 노출·훼손 상태 감지 / insight: POP별 주목·전환 효과 분석 / agent: POP 문구·위치·교체 주기 제안·생성
> **⑧ 연결** — saai ads insight · saai agent(생성) → 도입 상담

## 5. store fit — 트렌드 적합  ( /products/store-fit )

**이렇게 만들자:** 표준 골격. ⑥ 목업은 **적합도 스코어 카드 + gaze→pick 퍼널**.

**카피는 이렇다**
> (eyebrow) 기능 · store fit · 트렌드 적합
> (H1) **잘 나갈 상품인지, 진열 첫 주에 압니다**
> (sub) 신상품이 이 매장에 맞는지 — 시선·집어듦·재방문으로 트렌드 적합도를. 감이 아니라 첫 주 데이터로.
> (안심) 개인 식별 없이, 상품 반응만
>
> **② 한계** — 신상품의 운명은 보통 한 달 뒤에 압니다. 왜 안 팔렸는지(안 보여서·안 끌려서·집었다 놓아서)는 POS에 없습니다.
> **③ 해결** — 진열 첫날부터 상품 앞 시선·집어듦·재방문을 익명으로 재, 이 매장·손님층에 맞는지 점수로.
> **④ 무엇을** — 관심도(본 비율) · gaze→pick(집은 비율) · 매장 적합(또래 대비)
> **⑤ 3모드** — care: 신상품 초기 반응 이상 감지 / insight: 상품·매장 트렌드 적합도 분석 / agent: 발주·상품 구성·바이어 의사결정 제안
> **⑧ 연결** — saai ads insight · store shelf · saai agent → 도입 상담

## 6. 재배치 목적지 (도구 페이지 아님 — 별도 작업, 소스 보존)

**→ saai ads insight (신규 제품)** · gaze 승격
- 대상: 사이니지·전시물 + 매대 상품 / 지표: 통행·접근·주목(시선)·행동(interaction, opt)
- 소스 3모드: care "진열·POP·사이니지 앞 시선 반응 실시간 확인" / insight "대상물별 주목도(gaze→pick·접근 전환) 분석" / agent "골든존·게시 위치·노출 우선순위 제안"
- → 별도 제품 페이지 신규 설계 필요.

**→ store care 감지 항목** · keep·shelf
- keep(손실 방지): 이상 행동·이탈·분실 위험 실시간 감지 (신원 아닌 위험 행동만)
- shelf(매대·재고): 결품(out-of-stock)·흐트러짐·가격표 오류·오염 실시간 감지
- → store care 페이지 '감지 항목' 섹션(감지 카탈로그)으로 편입.

**→ 기술** · trail·census
- trail → Spatial AI/MTMC: 구역별 통과·체류 히트맵, 대표 경로, 이상 동선
- census → Vision Models: person attribute(연령대·성비 추정 · 신원 식별 아님)

**→ 숨김** · talk
- 다국어 응대(감지·분석·문구 생성). 페이지 비노출, 카피 보관.
