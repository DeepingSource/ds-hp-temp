# DEEPINGSOURCE 웹사이트 카피·체계 재설계 — 마스터 v1

이 문서는 세션에서 확정한 결정을 한곳에 모은 **인덱스 + 결정 로그 + 통합 액션 리스트**입니다. 영역별 상세는 개별 문서 참조(§8).

## 1. 전체를 관통하는 핵심 결정

**브랜드 위계 (brand-system 정본 + 이번 결정)**
- DeepingSource(회사) → **SAAI**(플랫폼·우산, 익명화 공간 AI) → 비전 **Perfect Every Space**(모든 공간을 완벽하게).
- **saai suite** ★신규★ = 엔터프라이즈 SOLUTION 패키지 = insight·care·agent 3모드가 하나의 익명화 루프.
- **SOURCE 도구**(store count·기능 라이브러리 12개) = 입문 다리. **SEED**(SEAL·anonymizer·spatial AI·vision models) = 기술 기반.
- solution-first: 가치(saai {value})로 열고, 구현체(store {value})로 계약, 도구로 입문.

**약속 / 보이스**
- 랜딩·회사 공통 약속: **"당신의 공간을, 완벽하게 → 모든 공간을, 완벽하게"** (북엔드).
- Mission = **REINVENT OFFLINE** / Vision = **Perfect Every Space**.
- 안전선: **"권고는 AI가, 결정은 사람이"** (전 페이지 공통).
- 프라이버시: **"영상은 남기지 않습니다 · 누구인지 남기지 않습니다"** — 비식별(흐림)이 아니라 원본에 가까운 정확도의 **익명화**.

**라벨·루프 (제품)**
- 운영 루프: **OBSERVE(count 입문) → ANALYZE(insight·어제) → DETECT(care·지금) → ACT(agent·다음) → LEARN(학습·다시)**.
- "다음 한 수" 표현 전면 제거 → "무엇을 바꿀지 / 운영을 바꾸는".
- store count = 4번째 제품 아님, **입문 도구**.

## 2. 랜딩(/ko) — 확정 요약

전 10개 섹션 + MTMC 재설계 완료. 상세: `랜딩_카피_개정안_v1.md`

- **히어로:** H1 "당신의 공간을, 완벽하게" / 서브 "개인정보 걱정 없이 공간을 해석해, 운영을 바꾸는 지능 — SAAI." / 본문 "쓰던 CCTV 그대로, 진열된 물건부터 오가는 사람까지 — 공간 안의 모든 것을 읽습니다."
- **문제 제기:** "어제 들어온 손님 10명 중 8명은, 결제 없이 나갑니다 — 어디서 놓쳤을까요?" + 카드 3각도(POS 한계→CCTV 한계→손실 반복).
- **SAAI 네 글자:** Anonymized "누구인지 남기지 않습니다" · Agentic "무엇을 바꿀지 짚어줍니다" · Intelligence "쓸수록 정확해집니다".
- **제품 루프:** OBSERVE·ANALYZE·DETECT·ACT (count는 입문 도구로 재편 — 파급 반영 필요).
- **현장 사례:** 과거형 → 현재형(정직한 예시).
- **SEAL:** "누구인지가 아니라, 무슨 일인지를 봅니다."
- **세 store:** 루프·조합 역할로 정리, 영문 라벨 통일.
- **Beyond retail / 숫자·파트너 / 클로징:** "본사와 매장, 사장과 손님 — 보이지 않던 사이를 메웁니다. 그게, SAAI." + 북엔드 클로징.

## 7. 통합 액션 아이템

### A. 개발 / QA (버그·구현)
1. **functions 페이지 국문화** — 본문 전체 영문 폴백. 최우선.
2. **제품 카테고리 라벨** — care=DETECT, agent=ACT로 통일(기능 페이지 "Advise"→ACT 포함).
3. **/products 루프 라벨·순서** — care=DETECT·agent=ACT, 순서 어제·지금·다음·다시.
4. **/products·랜딩 SAAI 네 글자 섹션 구 카피 교체** — 누구인지 / 무엇을 바꿀지 / 쓸수록 정확.
5. **랜딩 섹션2 루프 칩 순서** — 지금·어제·다음 → 어제·지금·다음·다시.
6. **SEAL eyebrow 렌더링** — "익명화 신뢰 헌장"(憲章) 확인.
7. **"다음 한 수" 잔존 제거** — 전 페이지.
8. **REINVENT OFFLINE 히어로 승격** — /company/about Vision&Mission 중복 제거.
9. **investors Traction 카운터** — 파트너·업종 숫자 노출(랜딩 8·7과 일치).

### B. 정합성 / 확인 (사실·표기)
10. **본사 마스터 카피 반영** — /company/about 본부 "모든 공간을 완벽하게" → "모든 매장을 한 매장처럼".
11. **창업자 표기 통일** — 'Pete Kim'(블로그) = 'CEO 김태훈'(페이지) 동일인. 영문/국문 통일.
12. **Series A 연도·리드 확인** — 2019(구 about) vs 2020·KDDI 리드(신규). A/A1, KDDI 리드 여부 검증.
13. **네이밍 표기 규칙** — 가치 브랜드 `saai {value}` / 구현체·도구 `store {value}`. count·insight 페이지 혼용 정리.
14. **파트너·업종·특허 수** — 8 파트너·7 업종·103 특허(등록 66·출원 37) 전 페이지 통일.
15. **"먼저 도입" 표현** — 로고가 운영 도입일 때만; 파트너십·PoC면 "함께합니다"로.

### C. 미결 결정 (담당자 확인 권장)
16. **saai suite 정식 도입** — brand-system 위계에 신규 층 추가(문서화).
17. **agent 라벨 ACT vs Advise** — 사이트는 Advise, 결정은 ACT. 최종 크로스체크.
18. **현장 사례 실측화** — 실제 숫자 확보 시 "예시" 해제 + 수치 삽입.
