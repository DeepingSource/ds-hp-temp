## 1. 문제 진단 (요약 — 상세 v1 §1)

**흐름:** 히어로 과적·자기중복 / "모든 공간을 완벽하게" 3회 반복 / 미괄식이라 결론이 안 박힘 / VisionDiagram↔3스텝 이중 시각화.
**대비:** 본문 5섹션 흰색↔`slate-50` 교대(명도차 2%) / VisionDiagram이 다크용인데 흰 배경 / 제목 전부 동일 크기 / 파랑이 미량 / 카드 전부 같은 처리.
**연결(신규 진단):** 현재 About의 유일한 외부 링크는 CTA→/contact 뿐. SAAI(제품)·technology(기술)·enterprise(기업도입)로 가는 **다리가 없다.** About이 "읽고 끝"나는 막다른 페이지.

## 2. 개선 원칙
1. **결론부터** (두괄식 선언 → 근거).
2. **하나의 서사선** ("선언 → 비전 → 5년 → 왜냐하면 → 누가 → **SAAI 브릿지** → 함께하자").

4. **허브화(v3)** — 각 섹션이 다음 행동(제품 보기·기술 보기·문의)으로 나가는 문을 하나씩 연다.

## 4. 섹션별 실행안 (신규·변경분 중심 — ①~⑤ 카피는 v2 §4 유지)

### ③ 향후 5년 — 끝에 SAAI 핸드오프 추가
- 목적지(2031) → 방법(3층, VisionDiagram+3스텝 통합)까지는 v2대로.
- **추가:** 3층 각 pillar에 **기술 딥링크**(익명화→/technology/seal · 공간→/technology/spatial-ai · 에이전트→/technology/models). `saaiPromiseLayer`에 이미 URL 존재 → 그대로 활용.
- **섹션 마무리 + 연결 그림 상단 절반:** 여기에 **회사(모든 공간) → SAAI(당신의 공간) + S·A·A·I** 위계를 배치(§10 그림 상단). "이 방법을, 하나의 플랫폼으로 — **SAAI란 →**"(`/products/saai`, 정체성 소개). ③에서 first 핸드오프.

### ⑤ 누가 만드나 — 파트너에 두 개의 문
- 파트너·인증 유지. **추가 링크 2개:**
 - **→ /enterprise** — "본사·프랜차이즈 도입" (About에서 뺀 본부/매장 약속의 **제 집**). 여기로 보내면 제거가 손실이 아니라 정리.
 - **→ /contact** — "파트너십·리셀러 문의" (협업 경로).

### ⑥ 이제, 당신의 공간부터 — SAAI (신규 섹션) ★
**목적:** ③에서 소개한 SAAI를 **제품(3모드)으로 펼쳐** 문의로 넘긴다. (그림 **하단 절반** — SAAI→제품→Contact)
**카피(초안):**
- (eyebrow) From every space — to yours
- (H2) **모든 공간을 완벽하게 하려면 — 당신의 공간부터.**
- (본문) SAAI는 세 모드로 일합니다 — **insight(어제를 읽다) · care(지금을 알리다) · agent(다음을 실행하다).** 익명화 위에서, 하나의 루프로.
- (CTA 메인) **제품 전체 보기 →** (`/products`) · (보조) SAAI 허브 (`/products/saai`)
**디자인:** §10 연결 그림 **하단 절반**(SAAI → 3모드 → 문의)을 이 섹션 시각물로. 상단 절반(회사→SAAI)은 ③이 이미 보여줬으므로 SAAI 노드가 ③↔⑥의 연결 고리.

### ⑦ 함께하자 — Contact (메인 전환)
CTA 유지 강화. "데이터센터가 아니라, 사람이 사는 세상을" + **문의하기 →**(`/contact`). ⑥(제품 탐색)과 역할 분리: ⑥ = 둘러보기, ⑦ = 대화 시작.

## 9. 크로스링크 맵 — About 허브화

About의 각 섹션이 여는 문(아웃바운드). 목적지는 실제 존재 라우트.

| From (About 섹션) | To (라우트) | 문구(초안) | 성격 |
|---|---|---|---|
| ③ 3층 pillar | `/technology/seal · spatial-ai · models` | (pillar명 클릭) | 기술 심화 |
| ③ 마무리 | `/products/saai` | "SAAI 보기 →" | 제품 1차 핸드오프 |
| ⑤ 파트너 | `/enterprise` | "기업·프랜차이즈 도입 →" | 본부/매장(구 MasterPair)의 집 |
| ⑤ 파트너 | `/contact` | "파트너십 문의" | 협업 경로 |
| ⑥ SAAI 브릿지 | `/products/saai` · `/products` | "SAAI 알아보기 / 제품 전체" | 제품 메인 전환 |
| ⑦ CTA | `/contact` | "문의하기 →" | 도입 메인 전환 |

**원칙:** 모든 경로가 결국 **/contact(도입·협업)** 로 수렴. 제품 관심은 `/products/saai`로, 기술 관심은 `/technology`로, 기업 관심은 `/enterprise`로 분기 후 다시 문의로.

**참고 — 인접 페이지 역할(연결 시 혼동 방지):**
`/products/saai`=SAAI 허브(당신의 공간) · `/products`=제품 목록(모든 공간) · `/solutions`=업종별 문제-해결 · `/enterprise`=기업·프랜차이즈 도입(본부/매장) · `/technology`=익명화 공간 AI 기술.

## 10. About → SAAI → Contact 연결 그림 (⑥ 섹션 시각물)

브랜드 상수 `perfectSpace`(your=SAAI · every=DeepingSource)를 그대로 시각화한 **수직 funnel**. (시안: `about_saai_bridge.html`)

```
┌──────────────────────────────────────────┐
│ 회사 · DeepingSource         [About]      │   ← 다크
│ "우리는 오프라인을 다시 만듭니다"          │
│  모든 공간을, 완벽하게                     │
└──────────────────────────────────────────┘
        │  모든 공간을 완벽하게 하려면 — 당신의 공간부터
        ▼
┌──────────────────────────────────────────┐
│ 플랫폼 · SAAI — 익명화 공간 AI  [primary] │   ← 파랑 강조
│  당신의 공간을, 완벽하게                   │
│  S · A · A · I  →(각) /technology/*        │
└──────────────────────────────────────────┘
        │  네 근거가 하나의 루프로 — 제품으로
        ▼
  saai insight(어제) · saai care(지금) · saai agent(다음)
        │  도입 문의 · 협업 제안
        ▼
┌──────────────────────────────────────────┐
│ 전환 · Contact  "사람이 사는 세상을, 함께" │   ← 파랑 채움
│  [문의하기 →]  /contact                    │
└──────────────────────────────────────────┘
```

**핵심:** 억지 연결이 아니라 **브랜드 위계 그대로** — 회사(every) → 플랫폼(your) → 제품(3모드) → 문의. About 방문자가 "그래서 내가 뭘 쓰지?"의 답(SAAI)과 "어떻게 시작하지?"의 답(Contact)을 페이지 끝에서 자연히 만난다.

**배치(확정):** 그림을 **둘로 나눠** — 상단 절반(**회사→SAAI + S·A·A·I**)은 **③**에, 하단 절반(**SAAI→제품 3모드→Contact**)은 **⑥**에. 가운데 **SAAI 노드**가 ③과 ⑥을 잇는 연결 고리. ⑥ 메인 전환은 **`/products`**(전체).
