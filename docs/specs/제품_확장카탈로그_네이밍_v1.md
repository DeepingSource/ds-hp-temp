## 0. 한 줄

**saai suite(코어 3모드)는 고정된 뼈대. 그 위에 확장(Extensions) — saai 확장 제품(ads insight…)과 store 기능(count·queue·pop·fit) — 이 당신의 공간에 맞춰 계속 늘어난다.** 그게 customizable.

## 1. 확정 아키텍처

```
SAAI (익명화 공간 AI)
├ saai suite   — 코어·엔터프라이즈 · insight·care·agent 한 루프        [고정된 뼈대]
├ 확장 (Extensions) — 당신 공간에 맞춰 더하는 것들 · 계속 늘어남
│   ├ saai 확장 제품 — saai ads insight … (saai 그래머)
│   └ store 기능     — store count·queue·pop·fit … (store 그래머 · 3모드 가로지름)
├ SEED         — SEAL·anonymizer·spatial AI·vision models              [기술 기반]
└ B2C          — saai.store · storecare.ai                             [별도 사이트]
```

**네이밍 규칙:** 제품 = `saai` 그래머 · 기능/도구 = `store` 그래머. `saai tool` 같은 세 번째 브랜드는 **안 만든다**(위계만 흐려짐). '**확장**'은 위계의 층 라벨이지 브랜드가 아니다.

## 2. 네이밍 결론 (메모의 '고민' 해소)

- **"saai suite vs saai tool?"** → suite만 브랜드 층, 나머지는 전부 '확장'. saai tool 폐기.
- **"saai가 아닌 것은 add-on service?"** → store 기능·saai 확장 제품 모두 '확장'으로 묶음. **add-on(추가 구성·추가 비용)은 가격/상담의 표현**이지 제품 카테고리명이 아님.
- **"계속 늘어나야 한다 + 당신 공간에 딱 맞는 AI(customizable)"** → 확장 층 자체가 세일즈 포인트: **"코어는 그대로, 당신의 공간에 맞게 더한다. 그리고 계속 늘어난다."** 브랜드를 안 늘리고 서사로 해결.

## 3. /products IA — 확장 카탈로그 섹션 신설

현행 IA(히어로 → saai suite 3모드 → 루프 → 카테고리 → SEED → B2C → CTA)에 **확장 카탈로그**를 삽입. **위치: suite 3모드 다음, SEED 앞**(코어→확장→기반 순).

> (eyebrow) 확장 · Extensions

## 4. saai ads insight — 확장 제품 1호 (번역 완료 · 콘텐츠 보강)

**현황:** KO/EN/JP 번역 **완료**(메모의 '번역 완수'는 사실상 끝). 페이지 구조 양호(히어로→한계→해결→무엇을 재나→3모드→목업→연결). **부족한 것:** suite와의 관계(확장 제품임)·대상 독자·시나리오·표준 8파트 중 ⑦(정확도·설치).

**보강안**
1. **확장 제품 명시** — 히어로/브레드크럼에 `saai 확장 제품` 뱃지 + "saai suite에 더하는 확장" 한 줄. (아키텍처 §1과 정합)
2. **대상 독자** — 브랜드·리테일 광고/VMD 담당, 사이니지·매대 운영자. "누구를 위한가" 한 줄 신설.
3. **시나리오** — 드럭스토어 테스터존 · 편의점 엔드캡 · 전시 부스(솔루션 문서의 `ads insight` 태그와 정합).
4. **⑦ 무설치 한 줄** — 별도 하드웨어 없이 쓰던 CCTV로(정확도 수치 확보 시 추가, 아니면 무설치만).
5. **연결 서사** — `store pop`(기능)·`saai insight`(코어)와의 관계를 카드 나열이 아니라 문장으로("pop이 게시 상태를, ads insight가 시선 반응을").
6. **그라운딩 톤** — "범용이 아니라, 이 대상물 앞을 직접 봅니다" 한 줄(랜딩 포지셔닝과 정합).
