## 0. 한 줄

SAAI는 하나의 플랫폼. **모듈(Modules)** — saai 제품(insight·care·agent·ads insight…)과 store 기능(count·queue·pop·fit…) — 을 **필요한 것부터 골라 쓰고, 계속 늘어난다.** `saai suite`는 세 모드를 한 번에 쓰는 본사용 묶음. 하나로 시작하든 전체를 쓰든, 당신 선택.

## 1. 확정 아키텍처 (골라 쓰는 모듈 · 종속 없음)

```
SAAI (익명화 공간 AI · 하나의 플랫폼)
│  골라 쓰는 모듈 (Modules) — 필요한 것부터, 전부까지 · 계속 늘어남
│
├ saai 제품 모듈
│   insight(어제) · care(지금) · agent(다음) · ads insight(대상물 반응) …
│   └ saai suite = insight·care·agent 세 모드를 한 번에 쓰는 본사용 묶음(대표 조합)
│
├ store 기능 모듈
│   count(입문·유입) · queue · pop · fit … — 3모드를 가로지르는 맞춤 도구
│
├ SEED — SEAL·anonymizer·spatial AI·vision models        [기술 기반]
└ B2C  — saai.store · storecare.ai                       [별도 사이트]
```

**핵심(사용자 지적 반영):** 모듈은 **코어를 사야 붙는 게 아니라 각자 입구가 된다.** 기능 하나로 입문(count)하거나 타깃 도입(ads insight)하고, 필요하면 suite로 전체. **suite는 "전부 다"의 이름이지 전제조건이 아님.** → store count의 '입문 다리' 포지션 회복.

**네이밍 규칙:** 제품 = `saai` 그래머 · 기능 = `store` 그래머. `saai tool` **미신설**. '모듈'은 위계의 층 라벨(브랜드 아님).

## 2. 네이밍/구조 결론 (메모 '고민' 해소)

- **"saai suite vs saai tool?"** → suite = 본사용 묶음, 나머지는 '모듈'. saai tool 폐기.
- **"saai 아닌 것 = add-on?"** → store 기능·saai 제품 모두 '모듈'(골라 쓰기). **add-on(추가 구성·비용)은 가격/상담의 표현**이지 카테고리명이 아님.
- **"계속 늘어남 + customizable"** → 플랫폼이 커지는 것 + 골라 조합: **"필요한 것부터, 전부까지 — 그리고 계속 늘어납니다."**
- **종속 제거** → '확장'이 깔던 "코어를 먼저 사야" 뉘앙스 제거. 모듈은 각자 입구.
