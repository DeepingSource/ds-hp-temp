## 1. [P0] solutions 진입 시 영문 전환 버그

**현상:** 한국어 상태에서 solutions를 거쳐 진입하면 일부 페이지가 영문으로 바뀜.

**원인 (코드 확인됨):**
`src/components/corporate/views/SolutionsView.tsx` L160-165 의 업종 링크가 `localeHref`로 감싸지지 않은 **비-locale 링크**다.

```tsx
// SolutionsView.tsx L161
<Link href={`/industries/${industry.slug}`}>   // ❌ 접두어 없음 → en(기본) 로케일로 이동
```

- `en`은 접두어가 없는 루트(`/...`)이므로, `/ko/solutions`에서 `/industries/...`로 가면 로케일 컨텍스트가 영문으로 리셋된다.
- 추가로 `/app/industries` 라우트 자체가 존재하지 않음 → 잘못된 목적지일 가능성도 있음(라우트 확인 필요).
- 참고: `SolutionDetailView.tsx` L227은 `${prefix}/industries/${sol.industry}`로 **올바르게** 처리 → 두 컴포넌트 간 불일치.

**작업:**
1. `SolutionsView.tsx`의 모든 `<Link href>`를 `localeHref(locale, path)`로 통일. 특히 L161.
2. `/industries/[slug]` 라우트 존재 여부 확인 → 없으면 올바른 목적지(예: `#industry-{slug}` 앵커 또는 실제 라우트)로 교정.
3. **전역 감사(audit):** 하드코딩된 비-locale 내부 링크 전수 점검.
   - 점검 커맨드: `grep -rn 'href={\`/' src/components src/app | grep -v localeHref`
   - `<Link>` 뿐 아니라 `router.push`, `redirect`, breadcrumb `path`도 포함.
4. (권장) 재발 방지: 내부 링크를 항상 로케일 인지하도록 감싸는 `LocaleLink` 래퍼 도입 또는 ESLint 룰(raw 내부 href 금지).

**검증:** `/ko/solutions` → 각 업종/솔루션/CTA 클릭 시 URL이 `/ko/...` 유지되는지 확인. `jp`도 동일 검증.

**피드백:** 무엇을 하는 제품/회사인지 소개가 부족. 서브카피를 줄이고 한 번에 파악 가능하게.

카피 소스: `content/site/home.yaml` → `src/data/generated/site-content.json` (`homeCopy`). **카피는 yaml에서 수정.**

**작업:**
1. 헤드라인을 "익명화 공간 AI로 매장을 운영한다" 급의 **한 문장 가치 제안**으로 재작성 (회사/제품 정체성 명확화).

3. `homeCopy`의 `heroSub` 길이 기준 en/ko/jp 3개 로케일 동시 수정.
4. (선택) 히어로 하단에 "무엇을 하는가"를 1줄로 보여주는 서브헤더/제품 라인업 프리뷰 추가.

**주의:** 카피 3개 로케일 일관성. 카피라이팅은 `brand-copywriting` 스킬 활용 가능.
