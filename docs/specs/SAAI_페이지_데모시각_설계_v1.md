## 2. ★ 운영 루프 데모 (핵심 시각 — 히어로 직후)

"뭘 하는지 모르겠다"의 해법. 정적 다이어그램이 아니라 **각 단계를 누르면 실제 화면이 뜨는** 인터랙티브 데모.

| 단계 | 라벨 | 제품 | 화면(예시) |
|---|---|---|---|
| 관찰 | OBSERVE · 입문 | store count | 문 밖 통행 1,160 · 입장 382 · 유입률 33% |
| 분석 | ANALYZE · 어제 | saai insight | 퍼널 1,160→382→317→65 + "음료 매대 체류 2배" 인사이트 |
| 감지 | DETECT · 지금 | saai care | 알림 1,247건 → 손쓸 3건(냉장고 문·바닥 오염·빈 매대) |
| 실행 | ACT · 다음 | saai agent | 액션카드 "삼각김밥 40개, 오픈 전 입고" [승인] |

- 하단: "결과가 다시 입력으로 — 한 바퀴 돌 때마다, 사람의 손은 가벼워집니다."(LEARN 루프 닫힘)
- **구현:** 서브 페이지(count·insight·care·agent)의 실제 목업 스크린을 각 단계에 연결. (인터랙티브 목업 시안은 대화에서 시연함)

## 3. 페이지 구조 (시각 요소 삽입)

1. 히어로 (SAAI 정체성 + 약속)
2. **★ 운영 루프 데모** (§2) — 주인공
3. who/what — "보는 데서 멈추지 않습니다 — 운영까지"
4. 약속 3단 (감으로 알던 매장을 데이터로 / 종일 지켜보던 매장을 필요한 순간만 / 사람이 못 하던 운영을 매일 대신)
5. saai suite = 3모드 카드 — 각 서브 페이지 목업 썸네일 얹기
6. **익명화 토글(C)** — SEAL 원본↔익명화 슬라이더 소형 삽입("얼굴 없이"를 즉시 체감)
7. SOURCE 기능 라이브러리 → SEED 기술 → use case·solution 연결 → CTA

## 4. 시각 원칙

- 글 사이사이에 **'작동하는 화면'**을 배치 — 루프 데모(주) · 익명화 토글(조) · 제품 목업 썸네일.
- 목업은 신규 제작보다 **서브 페이지 자산 재활용**(insight 대시보드·care 알림·agent 액션카드).
- 데이터엔 항상 "예시" 주석(정직성).
