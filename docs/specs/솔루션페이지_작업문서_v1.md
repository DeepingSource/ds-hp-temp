# 솔루션 페이지 작업 문서 v1

각 페이지를 **"이렇게 만들자(구성)"** + **"카피/수정은 이렇다"** 형식으로 정리.
작업일: 2026-07-20 · 대상: /ko/solutions 및 카테고리 4종

---

## 0. 구조 & 공통 규칙

**역할 분리(확정)**
- **/solutions** = 문제별 탐색 **가이드**(업종 파인더). 7개 업종을 한눈에 → 내 업종 찾기 → 문제·해결 → CTA.
- **/solutions/{retail·food-beverage·drug-store·large-space}** = **업종 종합** 페이지(4). 깊은 문제·before/after·사례.
- 둘은 서로 링크(파인더의 "이 업종 종합 보기 →" ↔ 카테고리 상단 "다른 업종 문제 찾기 →").

**공통 규칙**
- **지표 전부 '예시' 표기**(A): 뱃지에 `예시` + 섹션 하단 `* 지표는 대표 예시 — 매장 조건에 따라 다릅니다.`
- **가벼운 제품 태그**(C): 각 해결에 제품 표기 — 도난·이상·위생·안전→`saai care` / 진열·동선·전환·체류→`saai insight` / 대기·혼잡→`store queue` / 부스·전시·테스터 주목→`saai ads insight`. 문제 우선 톤 유지, 태그 수준.
- **톤:** "같은 기준으로 / 한 매장처럼"(본사 마스터 카피 호응) 유지.
- **도입 사례 넘버 시리즈**(01 발견→02 검증→03 번역→04 전파→05 재측정): 페이지를 가로지르는 서사 장치 — 유지.

## 1. /solutions 전체 — 업종 파인더 (재설계)

**이렇게 만들자**
1. 히어로 — "현장 문제에 검증된 해결책" + "내 업종을 골라 확인하세요."
2. **업종 파인더** (핵심 인터랙션): 7업종 그리드(한눈에) → 클릭 시 그 업종 문제·해결 3개 인라인 노출(예시 지표 + 제품 태그) + "이 업종 종합 보기 →"(카테고리 링크) + 상시 CTA.
3. 핵심 용어 정의 (유지).
4. 클로징 CTA.

**카피/데이터 — 7업종 × 해결(예시 지표 · 제품 태그)**

| 업종 | 해결 (예시 지표 · 제품) |
|---|---|
| 편의점 | 야간 도난 방지 −68% `care` · 결품·재고 손실 −42% `care` · 진열 최적화 +18% `insight` |
| 카페·음식점 | 좌석 회전율 +22% `insight` · 혼잡 시간대 −28% `queue` · 대기 전환 +19% `care` |
| 무인매장 | 도난 방지 −35% `care` · 이상 감지 −82% `care` · 원격 관리 −60% `care` |
| 드럭스토어 | VMD 최적화 +12% `insight` · 구역별 성과 +20% `insight` · 테스터 상호작용 +16% `ads insight` |
| 대형마트 | 계산대 혼잡 −32% `queue` · 카트 동선·체류 +20% `insight` · 구역 전환 +17% `insight` |
| 전시 공간 | 방문객 체류 +28% `insight` · 부스 성과 3배 `ads insight` · 인파 흐름 −25% `care` |
| 물류·창고 | 작업자 안전 −45% `care` · 구역 작업 효율 +15% `insight` · 보호구 준수 −70% `care` |

- 모든 수치에 `예시` 표기. 각 업종 카드 → 문제 상세 페이지(예: convenience-night-theft) 링크 유지.
- **히어로 "검증된"**은 유지(접근은 검증, 수치는 예시). 하단 CTA "우리 매장의 문제도 해결할 수 있을까요? → 무료 상담 신청" 상시.

## 2. /solutions/retail — 리테일·편의점

**이렇게 만들자:** 현행 구조 유지 — 히어로 → 문제 3카드 → 대시보드 → **before/after**(유일 보유, 유지) → 고객 인용+지표 → 도입 사례(01·02) → CTA.

**카피/수정**

- 제품 태그(C): 결품 신호 `saai care` / 진열 최적화 `saai insight` / 이상 감지 `saai care`.
- 지표 "예시" 주석 유지(이미 있음). H1 "결품도 이상도, 놓치지 않는 매장" 유지 — 좋음.
- 상단에 "다른 업종 문제 찾기 → /solutions" 링크 추가.

## 6. 비주얼 에셋 사양 (5종)

| # | 에셋 | 위치 | 구성 | 상태 |
|---|---|---|---|---|
| **V1** | 업종 파인더 | /solutions 허브 | 7업종 그리드 → 클릭 시 문제·해결 3개(예시 지표 + 제품 태그) + "이 업종 종합 보기" + 상시 CTA | ✅ 시연 완료 |
| **V2** | before/after 토글 | 4 카테고리 공통 | "지켜보던 운영 ↔ 먼저 아는 운영" 토글, After에 제품 태그 | ✅ 시연 완료 |
| **V3** | 카테고리 히어로 데모 | 4 카테고리 히어로 | 업종별 실제 화면(아래 사양) | ✅ retail 시연 · 나머지 3 사양 |
| **V4** | 문제 3카드 시각화 | 전 카테고리 문제 섹션 | 각 문제 카드에 아이콘/미니 시각 + 상태 | 사양 |
| **V5** | 도입 사례 지표 시각화 | 도입 사례 섹션 | 01~05 넘버 타임라인 + 지표 그래픽 | 사양 |

**V3 — 카테고리 히어로 데모 (업종별)**
- **retail**(✅): 전 매장 결품·이상 현황 보드 — 결품 감지·미해결 알림·이상 감지 카드 + 실시간 알림 리스트(강남역점 결품 등).
- **food-beverage:** 좌석·대기 게이지 — 좌석 점유율 / 대기 인원·시간 / 회전율. (`store queue`·`saai insight`)
- **drug-store:** 구역 관심 히트맵 — 카테고리별 관심 + HOT ZONE 표시. (`saai insight`·`saai ads insight`)
- **large-space:** MTMC 멀티캠 — 여러 카메라 → 하나의 공간 좌표 + 층·구역 혼잡. (`Spatial AI/MTMC`·`saai care`)

**V4 — 문제 3카드 시각화**
- 각 카드에 문제성 아이콘 + 해결 상태를 미니 시각으로(예: 결품=빈 매대 아이콘, 혼잡=대기 게이지). 전 카테고리 공통·가벼움.

**V5 — 도입 사례 지표 시각화**
- 01 발견(30분→8분)·02 검증(100개 매장)·03 번역(권고 실행률 87%)·04 전파(KPI 정합 64%→100%)·05 재측정(22중 18)을 넘버 타임라인 + 각 지표 그래픽으로.

**비주얼 공통 원칙**
- 모든 데이터에 "예시 화면" 주석. 서브 페이지 자산(care 대시보드·insight 히트맵·agent 액션카드) 재활용. CSS 변수·다크모드 대응. 과밀 금지(핵심 지표 2~3).
