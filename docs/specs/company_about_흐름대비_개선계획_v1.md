## 0. 한 장 요약

두 문제는 사실 **한 뿌리**다 — 페이지가 *같은 볼륨의 블록 9개*로 되어 있다.

- **흐름이 안 읽히는 이유:** 서사에 기승전결이 없다. 도입부(히어로)가 과적이고, 같은 메시지("온라인↔현장", "모든 공간을 완벽하게")가 3~4번 반복되며, **과거(기원) → 미래(비전) → 방법**의 시간축이 뒤엉켜 있다. 읽는 사람이 "지금 어느 대목인지"를 잡지 못한다.
- **대비가 부족한 이유:** 본문 5개 섹션이 흰색↔`slate-50`(거의 같은 흰색)으로만 번갈아, 경계가 눈에 안 보인다. 제목 크기·컬러·카드가 전 섹션 동일해 **"이게 중요한 대목"이라는 신호**가 없다. 파랑(primary)은 아이콘·점 등 미량으로만 쓰여 화면을 잡아주지 못한다.

처방은 세 가지다: **(A) 서사를 시간·논리 순으로 재배열하고 중복을 걷어낸다 → (B) 배경을 명암 리듬(다크 앵커 2~3개)으로 끊어준다 → (C) 제목 위계와 파랑 앵커로 "볼륨 차이"를 만든다.** 아래는 그 실행안이다.

## 1. 문제 진단

### 1-1. "한 흐름에 안 읽힌다" — 원인 4가지

**① 히어로 과적 + 자기 중복.**
현재 히어로 한 섹션이 짊어진 메시지: `REINVENT OFFLINE`(H1) + "오프라인을 다시 만든다"(sub) + **온라인↔현장 대비 카드 2장** + 회사 한 줄(`companyLine`) + `companyIntro`("온라인은 지난 20년…현장은 거의 그대로") + `companyIntro2`("딥핑소스는 그 현장을 다시 만듭니다") + `perfectSpace` 화살표("당신의 공간 → 모든 공간"). 
문제는 **"온라인은 최적화됐다 / 현장은 그대로였다"가 카드로 한 번, 바로 아래 `companyIntro` 문장으로 또 한 번** — 같은 말을 두 형식으로 반복한다. 히어로에서 이미 힘이 빠진다.

**② 핵심 문구의 과다 반복.**
"모든 공간을, 완벽하게"(`perfectSpace.every`)가 **히어로 kicker → V/M 섹션 H2 → Vision 카드** 세 곳에 등장한다. 하위 페이지들까지 이 비전 문장을 공유(마스터 문서 §6)하다 보니, About 안에서도 같은 문장이 계속 되돌아온다. 반복은 각인이 아니라 **정체(停滯)** 로 읽힌다.

**③ 시간축이 거꾸로/뒤엉킴.** 현재 순서:
히어로(미션) → 비전/미션 → **MasterPair(본부·매장 약속)** → **Vision 2031(미래)** → How we work(방법) → **Our Story·연혁(기원·과거)** → 리더십 → 파트너 → CTA.
- 회사의 **"왜 시작했나"(익명화 기원·창업자 여정·연혁)** 가 방법과 미래비전보다 **뒤**에 온다. 감정적 도입이 페이지 중반 이후에 묻힌다.
- **미래(2031)가 과거(연혁 2018~)보다 앞**에 나와 시간이 역행한다.
- **MasterPair(본부/매장 세일즈 약속)** 가 회사소개 서사 한복판에 끼어든다 — 이건 랜딩/제품의 영업 메시지지 "회사가 누구인가"의 흐름이 아니다.

**④ 같은 개념의 이중 시각화.**
`VisionDiagram`(Physical World → Privacy Shield → Spatial/Agentic AI)과 `How we work`의 3스텝(익명화 → 공간지능 → 에이전트)은 **완전히 같은 3층 구조**를 서로 다른 그림으로 두 번 보여준다. 게다가 둘은 다른 섹션에 떨어져 있어, 독자는 같은 개념을 두 번 처리하며 흐름이 끊긴다.

### 1-2. "시각적 대비가 부족하다" — 원인 5가지

**① 배경 리듬이 밋밋하다.** 섹션 배경 순서:
`surface-dark`(히어로) → `white` → `dark`(MasterPair) → `slate-50` → `white` → `slate-50` → `white` → `slate-50` → `slate-950`(CTA).
문제 구간은 **4~8번(Vision 2031·How we work·Our Story·리더십·파트너)** — 다섯 섹션이 흰색↔`slate-50`으로만 교대한다. `slate-50`은 흰색과 명도차 약 2%라 **경계선이 사실상 안 보인다.** 본문 전체가 한 덩어리 밝은 벽처럼 읽힌다.

**② 다크용 컴포넌트가 흰 배경에 얹혀 있다.** `VisionDiagram`은 `slate-800/60`·`slate-700`·`slate-400`·`primary-light` 등 **어두운 배경 전제로 설계**됐는데, 실제로는 **흰색 V/M 섹션 안**에 렌더된다. 흰 바탕 위의 `slate-400` 텍스트, `primary/10` 위의 `primary-light`(#5B86EA) 글씨는 대비가 낮아 **씻긴 듯** 보이고 톤도 충돌한다. (대비 문제이자 버그성 이슈.)

**③ 타이포 위계가 없다.** 9개 섹션 제목이 전부 `text-3xl sm:text-4xl font-bold`로 동일하다. **미션(가장 중요)과 파트너(보조)가 같은 크기**다. "어디에 힘을 줄지"를 활자가 말해주지 않으니 전부 같은 목소리로 웅얼거린다.

**④ 파랑(primary)이 앵커로 안 쓰인다.** #376AE2는 eyebrow·작은 아이콘·점(dot)·1.5px 라인 등 **미량**으로만 등장한다. 화면의 90%가 흰 바탕 위 회색 텍스트라, 시선을 붙드는 색 앵커가 없어 **소심하고 흐릿**하게 느껴진다.

**⑤ 카드가 전부 같은 처리.** `rounded-2xl/3xl` + `border-gray-100` + `bg-slate-50`/`white`. 히어로 미러 카드, V/M 카드, 스텝 카드, 연혁 마일스톤, 리더십, 파트너 통계, 인증 — **모두 같은 톤**이라 카드가 도형(figure)/배경(ground)을 분리하지 못한다. 특히 하단(연혁 마일스톤·파트너 통계·인증)은 작은 저대비 카드가 잔뜩 쌓여 밀도만 높고 초점이 없다.

## 2. 개선 방향 — 두 원칙

1. **하나의 서사선(one throughline).** 페이지를 *블록 9개*가 아니라 **"왜 → 무엇을 믿나 → 어떻게 → 어디로 → 누가 → 함께하자"** 6장(章)의 한 이야기로 다시 짠다. 각 장은 앞 장의 결론을 받아 다음으로 넘긴다.
2. **볼륨 차이를 설계한다.** 모든 섹션을 같은 세기로 두지 않는다. **다크 앵커 2~3개**로 이야기를 끊고, **제목 크기 3단계**와 **파랑 앵커**로 "중요한 대목"을 물리적으로 크고 진하게 만든다.

같은 자산을 **순서만** 바꿔 시간축과 논리를 바로 세운다. (배경은 §5-1의 리듬을 적용)

- **VisionDiagram을 How we work로 흡수**: 같은 3층 개념을 한 자리에서 한 번만.

## 4. 섹션별 실행안 (카피 정돈 + 디자인)

> 카피는 `content/site/about.yaml`(Keystatic)에서 편집. before는 현행 문구.

### 4-1. ① 히어로 — 덜어내서 명제 하나로

**문제:** 메시지 7개가 경쟁. 온라인↔현장 대비가 카드+문장으로 이중.
**카피 정돈:**
- `companyIntro`(before) "온라인은 지난 20년, 끊임없이 측정하고 개선해 왔습니다. 그동안 현장은 사람의 손과 눈에 기대어, 거의 그대로였습니다."
 → **대비 카드(온라인은/현장은)가 이미 이 말을 하므로, `companyIntro`는 삭제하거나** 다음 한 줄로 대체: **"온라인을 바꾼 건 측정이었습니다. 현장에는, 그 측정이 없었습니다."** (카드와 겹치지 않는 새 정보로.)
- `companyLine` + `companyIntro2` + `perfectSpace` 화살표가 히어로 하단에 3연속 → **`companyIntro2`("딥핑소스는 그 현장을 다시 만듭니다") 한 줄만 남기고** 나머지(회사 한 줄·화살표)는 ③ Mission/Vision으로 이관.
**디자인:** 히어로는 **H1 + sub + 대비 카드 2장 + 클로징 한 줄**까지만. 아래 여백을 늘려 "숨"을 준다. (현재 6요소 → 4요소)

### 4-2. MasterPair(본부/매장) 재검토 — 흐름의 최대 걸림돌

**진단:** `variant="dark"`의 강한 영업 카드(도입 상담 / storecare.ai 첫 달 무료). **회사소개 서사에 낀 랜딩 메시지.** 흐름을 가장 크게 끊는 구간.
**권고(택1, §8 열린 결정):**
- **(A·추천) 제거** — 본부/매장 약속은 랜딩·`/products`의 몫. About에선 덜어내면 흐름이 크게 산다.
- **(B) 축소 이관** — 꼭 남긴다면 CTA 바로 앞에 **텍스트 한 쌍**으로 줄여 "우리가 누구를 위해 일하나"의 마무리로. (다크 카드+버튼은 해제.)

### 4-3. ③ Mission & Vision — 반복 문구 걷어내기

**문제:** "모든 공간을, 완벽하게"가 H2·Vision카드에 중복. 히어로 kicker와도 겹침.
**카피 정돈:**
- H2(before) = `perfectSpace.every`("모든 공간을, 완벽하게") → **H2를 미션 문장 "REINVENT OFFLINE · 오프라인을, 다시 만듭니다"로** 바꾸고, "모든 공간을 완벽하게"는 **Vision 카드에서만 1회** 사용(히어로에서 이관해온 `perfectSpace` 화살표를 이 카드로).
- `mission`(before) "측정할 수 있으면, 나아질 수 있습니다. 온라인을 바꾼 그 힘을, 이제 물리 세계로." → 유지(좋음). 단 히어로에서 유사 표현을 뺐으니 여기서 처음 등장하게.
**디자인:** `VisionDiagram`은 이 섹션에서 **빼서 ④로 이동**(§4-4). 대신 Vision/Mission **두 카드에 파랑 앵커** 부여(§5-3).

### 4-4. ④ 어떻게 — VisionDiagram + 3스텝 통합, 다크로

**문제:** 같은 3층 구조를 두 번. `VisionDiagram`은 다크용인데 흰 배경.
**실행:**
- 이 섹션을 **다크 배경**으로 만들고(§5-1 앵커) → `VisionDiagram`이 **제 옷(어두운 배경)을 입는다.** 대비 문제 자동 해소.
- 상단에 `VisionDiagram`(Physical World→Shield→AI 한 눈 그림) → 그 아래 3스텝(①익명화 ②공간지능 ③에이전트) **상세**. "그림으로 개요 → 카드로 상세"의 자연스러운 확대. 개념 중복이 **개요/상세 관계**로 정리된다.
- 다크 위 `ProcessStepper` 카드 톤 반전 필요(§5-4).
**카피:** `methodIntro`·3스텝 promise는 현행 유지(품질 좋음). closing "측정할 수 있으면, 나아질 수 있습니다"는 ③ mission과 겹치니 **다른 마무리**로: "권고는 AI가, 결정은 사람이." (안전선, 마스터 문서 공통).

### 4-5. ⑤ 어디로 — Vision 2031 + 연혁 = 하나의 타임라인

**실행:** `OriginStoryTimeline`(2018~2025)과 `VisionCoordinatesMockup`(2031)을 **인접 배치**해 "2018 → 오늘 → 2031" 한 시간선으로. 연혁의 마지막(2025)이 미래 좌표(2031)로 이어지는 구성.
**카피 정돈:** 마일스톤 4장 카드 그리드는 타임라인과 정보가 겹침 → **연혁 타임라인으로 일원화**하고 마일스톤 카드는 축소/삭제 검토(저대비 카드 난립 완화, §1-2⑤).

### 4-6. ⑥ 누가 — Leadership + Partners

**실행:** 유지하되 §5-1 배경 리듬으로 파트너 섹션을 리더십과 시각 분리. 파트너 통계·인증의 **작은 카드 난립**은 §5-4 처리로 초점화.
**카피:** 현행 양호. `leadershipSub`·`partnersSub` 유지.

### 4-7. ⑦ CTA — 유지

다크 유지. "데이터센터가 아니라, 사람이 사는 세상을" + 문의하기. 좋음.

## 5. 디자인 시스템 차원의 처방

### 5-1. 배경 명암 리듬 — 다크 앵커로 끊기

흰색↔`slate-50`의 무의미한 교대를 버리고, **밝은 구간 사이에 다크 앵커**를 넣어 이야기를 단락 짓는다.

제안 리듬(§3 순서 기준):
`다크`(①히어로) → `밝음`(②Why) → `밝음-강조`(③M&V) → **`다크`(④어떻게)** → `밝음`(⑤Vision2031·연혁) → `밝음`(⑥리더십·파트너) → `다크`(⑦CTA)

- 다크 앵커 **3개(히어로·④·CTA)** 가 페이지를 3막으로 끊는다. MasterPair를 빼도(§4-2) 중앙 다크(④)가 생겨 리듬 유지.
- 밝은 구간끼리는 `slate-50`이 아니라 **경계가 보이도록** 처리: 섹션 사이 **얇은 구분선(hairline)** 또는 한쪽을 **흰색+상단 그라데이션**, 다른 쪽을 **`slate-100` 이상**으로 명도차를 키운다. (현 `slate-50`은 너무 옅음.)

### 5-2. 타이포 위계 — 제목 3단계

전 섹션 동일 `text-3xl/4xl`을 **3단계**로:
- **L1(대문·서사 전환):** 히어로 H1(현행 `7xl`)과 ④어떻게/③M&V 같은 핵심 장. 예 `text-4xl sm:text-5xl`.
- **L2(일반 섹션):** Why·Vision2031·리더십. `text-3xl sm:text-4xl`.
- **L3(보조):** 파트너·인증 소제목. `text-2xl` + eyebrow 강조.
"미션과 파트너가 같은 크기" 문제(§1-2③) 해소. 활자만으로 중요도가 읽힌다.

### 5-3. 컬러 앵커 — 파랑을 "면"으로

파랑을 미량(점·라인)에서 **면·강조**로 승격:
- ③ Mission/Vision 카드 중 **하나는 파랑 채움**(`bg-primary` 또는 `primary/10`+두꺼운 좌측 바) → "이게 우리의 약속"을 색으로 못박음.
- 각 섹션 eyebrow 위/H2 옆에 **짧은 파랑 룰(rule)** 또는 숫자 인덱스(①②③…)로 장(章) 감각 부여.
- 다크 섹션(④·CTA)에서 **파랑을 발광색**으로 크게 써 밝은 구간과 대비.
- 남용 금지: 앵커는 **섹션당 1곳**. 파랑이 도처에 있으면 다시 무앵커가 된다.

### 5-4. 카드 위계 — "전부 같은 카드" 탈피

- **primary 카드 / neutral 카드** 2톤으로 분리해 중요 카드(미션·핵심 스텝)를 도드라지게.
- ④가 다크가 되면 `ProcessStepper`의 활성 카드는 `bg-white`→**`bg-white/10 border-primary/40`** 등 다크 대응으로 반전(대비 확보).
- 하단 **저대비 소카드 난립**(마일스톤·파트너 통계·인증): 마일스톤은 타임라인으로 흡수(§4-5), 파트너 통계는 **큰 숫자+파랑**으로 4칸을 굵게, 인증은 로고 스트립으로 가볍게 — 밀도를 낮추고 초점을 준다.

### 5-5. 여백(리듬) — 섹션 내부 호흡

히어로처럼 요소가 많은 곳은 **요소 사이 여백을 키워** 한 번에 하나씩 읽히게. 반대로 관련 정보(연혁+2031)는 **붙여서** 한 덩어리로. "덜어내기 + 묶기"가 흐름의 핵심.

## 6. 카피 정돈 원칙 (요약)

1. **한 섹션 = 한 메시지.** 히어로 과적 해소가 1순위(§4-1).
2. **같은 문장 반복 금지.** "모든 공간을 완벽하게"는 About 안에서 **1회**(Vision 카드)로. 온라인↔현장 대비도 **1회**(히어로 카드)로.
3. **중복이면 새 정보로 교체**(삭제만 하지 말고, 그 자리에 다음 이야기를).
4. **closing 문구 차별화**: ③mission과 ④closing이 같은 "측정하면 나아진다"를 쓰지 않게(§4-4).
