# 팀 페이지 컨셉 계획 — "구성원의 입을 빌린 자랑" (Team Voices)

작성일: 2026-07-23 · 대상: `/company/team` (`TeamView.tsx`, `src/data/teamMembers.ts`)

## 0. 목표 재정의

팀 페이지를 조직도가 아니라 **증언대**로 만든다. 회사의 철학·문화·일하는 방식을 회사가 직접 선언하는 게 아니라, 각 구성원이 자기 언어로 자랑하게 해서 전달한다.

- 1순위 독자: **투자자** ("이 팀은 진짜다"), **예비 입사자** ("여기서 일하고 싶다")
- 2순위 독자: 고객사/소비자 ("이런 사람들이 만들면 믿을 만하다")
- 핵심 원칙: **회사가 말하면 주장이고, 구성원이 말하면 증거다.**

## 1. 현재 상태 진단

구조는 이미 절반 준비되어 있다. 41명 전원이 `quoteKo/quoteEn`을 갖고 있고, 컴팩트 카드 → 스포트라이트 모달(대형 초상 + 인용구) 동선도 방금 정리했다. 문제는 콘텐츠의 질이다:

1. **모든 인용구가 같은 목소리다.** "단 한 사람의 개인정보도 침해하지 않으면서…", "글로벌 리테일 현장의 진정한 혁신은…" — 41명이 전부 회사 소개서 문장을 말한다. '각자의 입을 빌린' 게 아니라 '회사의 입을 41개로 복제'한 상태. 이대로면 독자는 3개 읽고 스크롤을 내린다.
2. **자랑의 대상이 분류돼 있지 않다.** 철학 자랑인지, 동료 자랑인지, 기술 자랑인지 구조가 없어서 페이지 전체가 하나의 인상으로 수렴하지 않는다.
3. **문화 섹션(Beat 4)이 인물과 분리되어 있다.** '프라이버시 First' 등 3개 카드가 추상 선언으로만 존재 — 바로 위에 그 문화를 증언할 41명이 있는데 연결이 없다.
4. 모달에 인용구 한 줄뿐이라 클릭 보상이 약하다. `quoteJp`도 없다 (jp 로케일은 현재 ko 인용구 노출).

## 2. 콘텐츠 전략 — "자랑"의 설계

### 2.1 자랑의 3종 분류 (theme)

각 구성원의 증언에 theme을 1개 부여한다. 페이지 전체에서 3종이 고르게 섞이도록 편집 단계에서 배분 관리 (약 1/3씩).

| theme | 무엇을 자랑하나 | 주 타깃 | 예시 각도 |
|---|---|---|---|
| `craft` (제품·기술) | 내가 만든 것, 우리 기술의 어려운 지점 | 투자자, 고객사 | "카메라 한 대로 3D 동선을 풀 때 무엇이 가장 어려웠는가" |
| `culture` (팀·동료) | 함께 일하는 방식, 동료 | 예비 입사자 | "여기서는 인턴의 반론이 대표의 가설을 이긴다" |
| `philosophy` (철학·소신) | 왜 이 문제를, 왜 이 방식으로 | 투자자, 예비 입사자 | "익명화를 먼저 하는 건 규제 대응이 아니라 소신이다" |

### 2.2 목소리 원칙 (편집 가이드)

1. **1인칭 + 구체적 장면.** "우리는 세계 최고 수준의…"(회사 화법) 금지. "입사 첫 주에 내 코드 리뷰에 대표가 들어왔다"(장면 화법)로.
2. **서로 자랑 (cross-bragging).** 자기 자랑보다 강한 것이 동료 자랑이다. 엔지니어가 디자이너를, 리더가 주니어를 자랑하게 한다. 최소 1/3은 타인·타팀에 대한 증언으로.
3. **숫자와 고유명사.** "빠르게 성장"(X) → "PR 51개가 한 주에 머지되는 팀"(O). 검증 가능한 자랑만.
4. **대필 금지, 편집만.** 실제 인터뷰 답변에서 문장을 고르고 다듬는다. 말투의 개성(반말투 섞임, 웃음, 사투리)은 지운다 ↔ 남긴다 중 **남긴다**를 기본값으로.
5. 브랜드 캐넌과 충돌 금지: "권고는 AI, 결정은 사람" 계열 표현은 자유롭게, "자율 운영·대체" 계열은 회피.

### 2.3 수집 방법 — 3질문 인터뷰

전 구성원에게 서면/슬랙으로 3개 질문 (10분 분량). 답변에서 short(한 줄)와 story(2~3문단)를 편집 후 **본인 컨펌 필수**:

1. 여기서 일하면서 "이건 밖에 자랑하고 싶다" 싶었던 순간 하나를 장면으로 말해달라.
2. 동료나 다른 팀이 한 일 중 감탄한 것 하나는?
3. 우리 회사가 다른 회사와 다르게 일하는 방식 하나를 꼽는다면?

(1→craft/philosophy, 2→culture, 3→philosophy/culture 소스가 된다. 41명 중 초기엔 리더십 6 + 지원자 10~15명으로 시작해도 충분 — 전원 강제하지 않는다. 미제출자는 현행 인용구 유지.)

## 3. 페이지 구성안 (Beat 재설계)

현재 5-Beat 구조를 유지하되 콘텐츠 흐름을 "증언"으로 재배선한다. 레이아웃 대공사보다 **연결**이 핵심.

### Beat 1 — Hero: 첫 증언
- 기존 face pile 아래(또는 대체)에 **로테이팅 인용 한 줄**: 작은 아바타 + "…" + 이름·직함. 5초 간격 페이드 교체 (reduced-motion 시 정적).
- 헤드라인 방향: "저희가 왜 여기서 일하는지, 직접 들어보세요" 류 — 페이지의 화법 선언.

### Beat 2 — Leadership: 방향의 증언
- 현행 유지 + 인용구를 새 원칙으로 교체. 리더십 6인은 philosophy theme 고정 (투자자 독자용). 각자 "왜 이 문제인가"를 다른 각도에서 — 6명이 같은 말을 하면 실패.

### Beat 3 — Voices Wall (현 Portrait Wall 개편)
- 컴팩트 카드에 **한 줄 자랑(short)을 항상 노출** (현재 sm 카드는 이름·직함만). 카드가 '사람 목록'이 아니라 '말풍선 벽'이 된다.
- 필터에 **theme 축 추가**: 기존 그룹(직군) 필터 유지 + "무엇에 대한 이야기인가" 토글 (제품·기술 / 팀·문화 / 철학). 예비 입사자는 문화만, 투자자는 철학·기술만 골라 읽는 동선.
- 정렬은 theme이 섞이도록 인터리브 (같은 theme 3연속 방지).

### Beat 4 — Culture: 선언 → 증언 구조로 전환 (가장 중요한 변경)
- 현행 3개 추상 카드(프라이버시 First / 권고는 AI 결정은 사람 / 자율과 책임)를 **각 원칙 + 그 원칙을 증언하는 구성원 인용 1~2개**의 묶음으로 재구성.
- 구조: 원칙 제목(회사의 문장) 아래에 실제 구성원의 반증 불가능한 일화(구성원의 문장) + 아바타·이름. 인용 클릭 시 해당 인물 스포트라이트 오픈.
- 이 순간 문화 섹션이 "우리는 이렇습니다"에서 "이 사람들이 이렇게 말합니다"로 바뀐다.

### Beat 5 — CTA: 독자별 분기
- 현행 채용 CTA 유지 + 보조 링크를 독자별로: 예비 입사자 → career, 투자자 → company/investors, 고객사 → contact. 한 줄 카피로 "방금 읽은 사람들과 일하게 됩니다" 류 연결.

### Spotlight 모달: 클릭 보상 강화
- 현행: 대형 초상 + 인용 한 줄. 개편: **short(제목 역할) + story(2~3문단 일화)** + theme 뱃지 + (선택) "Ask me about" 키워드 칩 2~3개 (채용 문의 시 대화 소재).
- story가 없는 멤버는 현행처럼 인용구만 — 데이터 유무로 UI가 자연히 분기되게 (optional 필드).

## 4. 데이터 모델 변경 (`teamMembers.ts`)

```ts
export type VoiceTheme = 'craft' | 'culture' | 'philosophy';

export interface TeamMember {
  // …기존 필드 유지 (quoteKo/En는 하위호환용으로 유지)…
  quoteJp?: string;          // 신규 — jp 공백 해소
  voice?: {
    theme: VoiceTheme;
    shortKo: string; shortEn: string; shortJp: string;   // 한 줄 자랑 (wall·hero)
    storyKo?: string; storyEn?: string; storyJp?: string; // 모달용 2~3문단
    askMeAbout?: string[];   // 선택 — 모달 칩
  };
  endorses?: string;         // 선택 — cross-bragging: 이 증언이 자랑하는 대상 멤버 id (모달에서 상호 링크)
}
```

- `voice` 없는 멤버는 기존 quote 폴백 → 점진 전환 가능, 일괄 마이그레이션 불필요.
- 문화 섹션 연결: `CULTURE_ITEMS`에 `voiceOf: string[]` (멤버 id)를 추가해 원칙별 증언 인용.

## 5. 예시 — 현행 vs 목표 (편집 방향 감각용, 실제 문구는 인터뷰에서)

| | 현행 (회사의 입) | 목표 (구성원의 입) |
|---|---|---|
| CEO | "단 한 사람의 개인정보도 침해하지 않으면서, 오프라인 공간의 모든 흐름을 데이터로 전환합니다." | "창업 때 정한 규칙이 하나 있습니다. 원본 영상은 우리도 못 봅니다. 이 제약 때문에 8년이 더 걸렸고, 이 제약 덕분에 아무도 우리를 못 따라옵니다." *(philosophy)* |
| 엔지니어 | "세계 최고 수준의 엔진을 구축합니다." | "카메라 한 대로 3D 동선을 푸는 문제는 논문에도 답이 없었어요. 특허 문서에 제 이름이 들어간 날, 부모님께 링크를 보냈습니다." *(craft)* |
| 신입/주니어 | (유사 슬로건) | "입사 2주 차에 낸 반론으로 스프린트 방향이 바뀌었어요. 여기선 연차가 아니라 근거가 이깁니다." *(culture)* |

## 6. 구현 단계 및 공수

| Phase | 내용 | 공수 | 비고 |
|---|---|---|---|
| A. 콘텐츠 (개발 아님) | 3질문 인터뷰 발송 → 수집 → 편집 → 본인 컨펌. 리더십 6인 + 자원자 우선 | 1~2주 (엘랩스) | 페이지 개발과 병렬. **이게 진짜 공수** |
| B. 데이터 모델 + 폴백 | `voice`/`quoteJp`/`endorses` 타입 추가, 폴백 로직 | 0.5일 | |
| C. Wall·모달 개편 | sm 카드에 short 노출, theme 필터, 모달 story·칩 | 1일 | 방금 만든 컴팩트 카드 위에 증분 |
| D. 문화 섹션 증언화 + Hero 로테이터 + CTA 분기 | | 1일 | |
| E. QA·번역 | 3로케일, reduced-motion, 접근성(로테이터 aria-live 등) | 0.5일 | |

개발 총 **3일**. 단, 콘텐츠(Phase A)가 없으면 껍데기이므로 **A를 먼저 발사**하고 B~D를 그 사이에 진행하는 순서를 권장.

## 7. 리스크 / 결정 포인트

- **진정성이 전부다.** 인터뷰 없이 카피라이터가 41개를 "개성 있게" 써내면 지금보다 나쁘다(가짜 개성). 초기엔 진짜 증언 10~15개가 균질한 41개보다 낫다 — 수가 아니라 밀도로.
- **본인 컨펌 + 퇴사 프로세스**: 증언은 실명 발화이므로 게시 전 본인 승인, 퇴사 시 비노출 처리 규칙을 데이터에 함께 (`published: boolean` 고려).
- **jp 번역 부채**: 현재도 jp 인용구가 없다. voice 도입 시 jp까지 한 번에 갚을지, en까지만 먼저 갈지 결정 필요.
- 로테이팅 인용의 접근성: 자동 교체는 pause 가능해야 함 (WCAG 2.2.2) — reduced-motion 시 정적 + 수동 넘김.
