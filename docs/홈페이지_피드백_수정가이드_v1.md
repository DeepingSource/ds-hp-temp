## 0. 시작 전: 배포본과 로컬 소스가 다르다 (가장 먼저 확인)

피드백 대상인 vercel 배포본은 로컬 HEAD보다 뒤처져 있다. 특히 커밋 `4ca31f93` *"fix(hero): 랜딩 동적 회전 문구 반응형 wrapping 구속"* 이 이미 로컬에 있고, 히어로 CTA도 현재 코드에선 모바일 세로 스택(`flex flex-col sm:sm:flex-row`)이 적용돼 있다. 또 고정 하단 바(`LandingStickyCta`)도 현재 코드는 "히어로 통과 후에만 노출"이 이미 구현돼 있다(§3 참고).

**→ Action 0: 로컬 HEAD를 먼저 재배포하고, 390px 뷰포트에서 피드백 항목을 재검증한 뒤 아래 잔여 항목을 적용한다.** 배포본 기준으로 이미 고쳐진 걸 다시 고치는 낭비를 막는 게 첫 단계다.

또한 배포본에 있다는 "Field Scenarios" 섹션 라벨은 현재 소스에 존재하지 않는다(개편 과정에서 제거/개명됨). 피드백 중 해당 섹션 관련 내용은 현 구조의 대응 섹션(`CaseBand`, `SpacesShowcase`)에 매핑해 아래에 반영했다.

## P0-1. 모바일 히어로 오버플로우 + 로테이팅 단어 공백 (한 뿌리 문제)

### 현상
390px에서 헤드라인/CTA 우측 잘림(배포본), 데스크톱에서도 "당신의 매장을 ␣␣␣ 알고 있나요?"처럼 회전 단어 앞뒤 공백으로 문장 리듬이 끊김.

### 코드상 원인 — `src/components/ui/RotatingNoun.tsx`
두 증상의 뿌리가 같다. RotatingNoun은 **가장 긴 토큰("물류센터를")을 invisible sizer로 깔아 셀 폭을 고정**한다:

```tsx
// L44 — 최장 토큰이 폭을 예약: "매장을"이 표시될 때도 "물류센터를" 폭을 차지
const sizer = [fixed, ...words].reduce((a, b) => (b.length > a.length ? b : a), '');
```

- 짧은 단어가 나올 때 남는 폭 = 피드백의 "큰 공백".
- 최장 단어 기준 고정폭 + `CorporateHero.tsx` L71·L76의 `whitespace-nowrap` 스팬 + 고정 `text-4xl`(36px)이 겹치면 좁은 뷰포트에서 줄바꿈이 막혀 오버플로우가 난다.
- **버그 발견**: L49 `className={'inline-inline-flex items-baseline ...'}` — `inline-inline-flex`는 존재하지 않는 클래스다(오타). `inline-flex`로 의도한 정렬이 실제로는 적용되지 않고 있다.

### 수정 방법

**(a) 오타 수정 — 1분짜리**
```tsx
// RotatingNoun.tsx L49
- <span className={`inline-inline-flex items-baseline text-primary ${className}`}
+ <span className={`inline-flex items-baseline text-primary ${className}`}
```

**(b) 회전 컨테이너를 현재 단어 폭으로 애니메이트 (공백 제거의 본질 수정)**
sizer 방식을 버리고, 현재 단어의 실측 폭으로 `width`를 트랜지션한다. 폭이 줄었다 늘며 문장이 항상 붙어 보인다:

```tsx
'use client';
import { useLayoutEffect, useRef, useState } from 'react';

// inline-grid sizer 3중 스팬 → 측정 기반 width 트랜지션으로 교체
const measureRef = useRef<HTMLSpanElement>(null);
const [w, setW] = useState<number>();

useLayoutEffect(() => {
  if (measureRef.current) setW(measureRef.current.offsetWidth);
}, [current]);

return (
  <span className={`inline-flex items-baseline text-primary ${className}`} …>
    <span className="sr-only">{fixed}</span>
    <span
      aria-hidden="true"
      className="inline-block overflow-hidden whitespace-nowrap align-baseline
                 transition-[width] duration-300 ease-out"
      style={{ width: w === undefined ? 'auto' : `${w}px` }}
    >
      {/* 실측용 + 표시용을 같은 노드로: key 교체 시 애니메이션 유지 */}
      <span key={current} ref={measureRef} className="rotating-noun inline-block">
        {current}
      </span>
    </span>
    {suffix && <span className="text-gray-900 font-bold">{suffix}</span>}
  </span>
);
```
주의점: `prefers-reduced-motion`일 땐 기존처럼 `fixed` 고정 표시(이미 구현됨)라 폭 애니메이션도 자연히 발생하지 않는다. SSR 첫 페인트는 `width:auto`라 CLS 없이 안전하다.

**(c) 폭 변동 자체를 줄이는 카피 보조 수정 (선택, 효과 큼)**
`src/lib/brand-canon.ts` L89의 단어 세트에서 길이 편차가 문제의 절반이다:
`['매장을','현장을','전시장을','물류센터를','카페를','무인매장을']` → 3~5자로 편차 축소를 검토(예: '물류센터를' → '창고를', 또는 긴 단어를 뒤쪽 순번으로 몰아 첫인상 구간의 점프 최소화). 카피 결정 사안이므로 (b)와 병행 권장.

**(d) 헤드라인 clamp 타이포 스케일 — 오버플로우 안전망**
`CorporateHero.tsx` L67의 `text-4xl sm:text-5xl`은 360~640px 사이 구간이 36px 고정이라 여유가 없다:
```tsx
- className="text-4xl sm:text-5xl lg:text-5xl font-bold …"
+ className="text-[clamp(1.75rem,8vw,3rem)] font-bold …"
```
히어로 섹션 루트에 이미 `overflow-hidden`(L53)이 있어 잘림은 가려지지만, 가림은 해결이 아니므로 clamp로 애초에 넘치지 않게 한다.

### 검증
Playwright/DevTools 390×844, 360×800에서: H1 줄바꿈 정상 / 가로 스크롤바 없음(`document.documentElement.scrollWidth === innerWidth`) / 회전 6단어 모두 순환하며 레이아웃 점프 없는지 확인.

## P0-2. 스크롤 리빌 빈 프레임 — 원인은 "리빌 시스템 이원화"

### 현상
카드 그리드 섹션(현 구조: ProblemBeat, CaseBand, SpacesShowcase 등)에서 빠른 스크롤 시 카드가 한동안 안 나타남.

### 코드상 원인
사이트에 **리빌 시스템이 3개 공존**하고, 그중 1개만 빠른-스크롤 대응이 돼 있다:

| 시스템 | 사용처 | 빠른 스크롤 대응 |
|---|---|---|
| `Reveal`/`RevealStagger` (framer `whileInView`) | 신규 섹션 | ✅ `margin: '0px 0px 15% 0px'` 선트리거 |
| `AnimatedSection` + `useScrollAnimation` | 섹션 래퍼 | ✅ 마운트 시 통과분 즉시 표시 + 3초 세이프티 타임아웃 + 15% rootMargin |
| **`StaggerContainer`/`StaggerItem`** | **ProblemBeat L147, CaseBand L108, SpacesShowcase L87** | ❌ **`margin: '-80px'`** — 뷰포트를 80px *축소*해 오히려 트리거를 늦춤. 세이프티 없음 |

즉 피드백이 본 "빈 카드"는 정확히 `StaggerContainer`를 쓰는 그리드들이다. 카드가 `opacity:0; y:22`로 시작해 `useInView`가 늦게 발화하는 동안 텍스트/이미지 전부가 비어 보인다.

### 수정 방법

**(a) StaggerContainer를 Reveal 계열과 같은 정책으로 정렬:**
```tsx
// src/components/ui/StaggerContainer.tsx
- margin = '-80px',
+ margin = '0px 0px 15% 0px',   // 뷰포트 하단 15% 확장: 진입 전 선트리거
```
그리고 `useScrollAnimation`과 동일한 세이프티를 추가:
```tsx
const isInView = useInView(ref, { once: true, margin });
const [forced, setForced] = useState(false);
useEffect(() => {
  // 마운트 시 이미 지나쳤거나 3초 내 미발화 시 강제 표시 (R5와 동일 정책)
  if (ref.current && ref.current.getBoundingClientRect().top <= window.innerHeight) {
    setForced(true); return;
  }
  const t = setTimeout(() => setForced(true), 3000);
  return () => clearTimeout(t);
}, []);
…
animate={isInView || forced ? 'show' : 'hidden'}
```
**장기적으로는 `StaggerContainer`/`StaggerItem`을 `RevealStagger`/`RevealItem`으로 수렴**시켜 시스템을 하나로 줄이는 걸 권장한다(Reveal.tsx 주석에도 이미 그 의도가 명시돼 있다).

**(b) 이미지 "빈 회색" 구간 제거 — 로딩 정책**
카드 이미지는 `next/image` 기본 lazy라 리빌 직후에야 요청이 나간다. 애니메이션 지연 + 로딩 지연이 겹쳐 빈 순간이 길어진다:
- 각 그리드의 **첫 화면행 이미지(모바일 기준 1~2장)에 `loading="eager"`** 지정 (예: SpacesShowcase 카드 index < 2).
- 나머지는 lazy 유지하되 placeholder를 현행 `bg-slate-100` 단색 대신 `placeholder="blur"` + blurDataURL(빌드타임 생성) 적용하면 "비어 있음"이 아니라 "로딩 중"으로 읽힌다.
- 홈 하단 `SpatialTrajectoryMockup`은 dynamic import 스피너(`h-[400px] animate-pulse`)가 이미 있어 유지.

### 검증
크롬 성능탭 CPU 4x + Fast 3G에서 휠 연속 스크롤/스크롤바 드래그로 최하단까지 이동 → 어떤 시점에도 "텍스트만 있는 카드 프레임"이 300ms 이상 보이지 않아야 함. `prefers-reduced-motion` 켠 상태에서 전 섹션 즉시 표시 확인(회귀 방지).

## P0-3. CTA 문구·위치 통일

### 현상/원인 — 같은 액션이 4곳, 문구는 2종
현재 코드 기준 `/contact`로 가는 CTA:

| 위치 | 파일 | 문구 |
|---|---|---|
| 헤더 우측 (상시) | `Header.tsx` L354 | "도입 상담" (`homeCopy.ctaPrimary`) |
| 히어로 | `CorporateHero.tsx` L82 | "도입 상담" + 보조 "본사·다점포 도입 보기"(→ /enterprise) |
| 고정 하단 바 | `LandingStickyCta.tsx` L18 | **"무료 상담 신청"** ← 유일하게 다른 문구 |
| 마지막 다크 CTA | `HomeView.tsx` L147 | "도입 상담" |

참고: "고정 바가 항상 떠 있다"는 피드백은 배포본 기준이고, 현재 코드는 오가닉 홈에서 `scrollY > innerHeight * 0.85` 이후에만 노출된다(L37). 재배포로 해소. `?lp=1` 광고 랜딩 모드의 즉시 노출은 의도된 동작이므로 유지.

### 수정 방법

**(a) 문구 단일화**: 표준 문구를 "도입 상담"으로 통일하고, "무료"는 문구가 아니라 보조 라인(reassure)으로만 쓴다 — 이미 `무료 상담 · 영업일 1–2일 내 회신` 패턴이 있으므로 일관적이다.
```tsx
// LandingStickyCta.tsx L18
- ko: { tagline: 'AI 매장 관리, 지금 바로 시작하세요', button: '무료 상담 신청', aria: '상담 신청' },
+ ko: { tagline: '무료 도입 상담 · 영업일 1–2일 내 회신', button: '도입 상담', aria: '도입 상담 신청 — 하단 고정 바' },
```
(en/jp도 `ctaPrimary`와 동일 문구로 정렬: "Talk to us" / jp 표준 문구.)

**(b) 마지막 CTA 섹션과의 중복 노출 제거**: 페이지 끝에서 다크 CTA와 고정 바가 동시에 보이면 같은 버튼이 화면에 2개다. 마지막 CTA 섹션에 `id="closing-cta"`를 주고 sticky에서 관찰해 숨긴다:
```tsx
// LandingStickyCta.tsx — visible 계산에 추가
const [atClosing, setAtClosing] = useState(false);
useEffect(() => {
  const el = document.getElementById('closing-cta');
  if (!el) return;
  const io = new IntersectionObserver(([e]) => setAtClosing(e.isIntersecting), { threshold: 0.2 });
  io.observe(el);
  return () => io.disconnect();
}, [pathname]);
const visible = (lpMode || (isHome && scrolledPastHero)) && !atClosing;
```

**(c) 스크린리더 맥락 구분**: 동일 텍스트 링크 반복은 `aria-label`로 구분한다.
```tsx
// Header.tsx L354: aria-label="도입 상담 — 헤더"
// CorporateHero.tsx L82: aria-label="도입 상담 — 지금 바로 시작"
// HomeView.tsx L147: aria-label="도입 상담 — 무료, 영업일 1–2일 내 회신"
```

**(d) 헤더 CTA vs 히어로 CTA**: 히어로가 뷰포트에 있는 동안은 헤더 CTA를 텍스트 링크로 강등(또는 투명 배경)하고 스크롤 후 채움 버튼으로 승격하는 패턴도 검토 — 첫 화면에서 "도입 상담" 채움 버튼이 2개 보이는 경쟁을 없앤다. (선택 사항, 디자인 판단 필요)

## P1-1. 섹션 리듬 — "또 카드 세 개" 완화 + 다크 구간 빈 화면

### 수정 방법
1. **다크 CTA 직전 빈 화면 해소(우선)**: Beat 8은 `AnimatedSection`이 섹션 *전체*를 `scroll-hidden`(opacity 0)으로 잡아, 다크 배경+워터마크만 보이고 콘텐츠는 관찰자 발화 후 한꺼번에 등장한다. 섹션 래퍼의 리빌을 제거하고(배경·워터마크는 즉시 렌더), 내부 텍스트 블록만 `Reveal`로 감싸 `amount: 0.1` 수준으로 일찍 띄운다:
```tsx
- <AnimatedSection className="relative py-20 lg:py-28 section-dark …">
+ <section id="closing-cta" className="relative py-20 lg:py-28 section-dark …">
    … 배경/워터마크는 그대로 …
    <Reveal amount={0.1}><div className="relative max-w-3xl …">…</div></Reveal>
```

2. **리듬 변화 삽입 (택 1–2, 전부 넣지 말 것)**
   - CaseBand와 SpacesShowcase 사이 또는 HomeEnterpriseBeat 자리에 **전체 폭 실제 대시보드 스크린샷 밴드**(saai insight 화면 + 캡션 1줄). 신뢰 소재로도 겸용되므로 우선 추천.

   - SpacesShowcase 6카드는 모바일에서 세로 6장 스크롤이 길다 — 모바일만 가로 스냅 캐러셀(2열×3 or overflow-x snap) 검토.

## P1-2. TRUSTED BY — 로고 교체 + 당장의 대비 수정

### 현상/원인 — `GuideIntroBeat.tsx` L120–132
파트너가 회색 **텍스트**(`text-gray-400`)로만 나열. 확인 결과 **로고 이미지 에셋이 저장소에 아직 없다**(`PartnerGrid.tsx` 주석에 "PR-17 will swap to logo sprite"로 예정만 있음). 그리고 실측 대비가 **gray-400(#9CA3AF) on white = 2.54:1로 WCAG AA(4.5:1) 명확 미달** — 피드백의 "눈에 잘 안 들어옴"이 수치로 확인된다.

### 수정 방법
1. **(즉시, 에셋 불요)** 대비 수정: 라벨·브랜드명 `text-gray-400` → `text-gray-500`(4.83:1, 통과) 이상. 브랜드명은 `text-gray-600 font-semibold` 권장.

```tsx
<ul className="flex flex-wrap items-center gap-x-10 gap-y-4">
```

## P2. 접근성 — 이미 잘돼 있는 것 vs 남은 것

**이미 구현돼 있어 재작업 불필요(피드백의 우려 3건 중 2건은 해소 상태):**
- `prefers-reduced-motion`: 전역 CSS 킬스위치(globals.css L825~) + `usePrefersReducedMotion` 훅 + MotionProvider 3중 방어. RotatingNoun도 reduced 시 고정 토큰 표시.
- SEAL 슬라이더 키보드 접근: `BeforeAfterSlider`에 range input 오버레이(포커스 + 방향키 제어)가 이미 있음.
- 회전 헤드라인 SR 대응: sr-only 고정 토큰 + `aria-hidden` 회전 레이어 — 올바른 패턴.

**남은 것:**
1. **대비 미달 수정 (실측 기준)**: white 배경 위 `text-gray-400` 전부 (P1-2 포함; `text-2xs` kicker류도 동일). 참고로 다크 섹션의 `text-gray-400`은 7.37:1로 **통과**라 손댈 필요 없음 — 피드백의 "다크 섹션 서브텍스트" 우려는 기우로 확인.
2. 반복 CTA `aria-label` 구분 (P0-3-c에 포함).
3. 회전 단어 정지 수단이 현재 hover뿐 — 터치 사용자는 정지 불가(WCAG 2.2.2). reduced-motion 외 노출이 5초 미만 주기 반복이므로, 간단하게는 `onPointerDown` 토글로 탭-일시정지 추가.
4. 마지막 CTA 섹션의 `reassure`(`text-gray-400` on dark)는 통과하나, 배경 사진(opacity-40) 위 텍스트는 사진 밝은 영역과 겹칠 수 있음 — 현행 `bg-surface-dark/80` 스크림 유지 필수(제거 금지) 정도만 명심.
