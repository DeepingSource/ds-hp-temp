1. **캐노니컬 데이터 SOT부터 만들 것.** `src/data/mockup-scenarios/canonical.ts` — 디폴트 시연점은 **잠실광적점(G001)**(데이터시나리오 §1-1), 본부 뷰는 200점 분포·일일 알림 1,247건·응답률 89%·일일 2.4h 절감(§2·§4). 모든 신규 목업의 숫자는 이 파일에서만 파생.
2. **오픈 결정 2건을 브랜드 오너와 확정할 것.**
   - **OD-1 표기**: 목업 헤더 `STORECARE` vs `saai care` vs `store care` (Architecture v4 §1 기준).
   - **OD-2 색 이원화**: 제품 UI 목업 = 기존 3색 테마(emerald/violet/blue) 유지, **다이어그램류 = 회색조 + brand-blue(#376AE2) 1색** — 이 이원화 룰의 공식 승인.
3. **공통 제작 규칙** (전 항목 적용):
   - 다이어그램: 그라데이션·그림자·3D 금지, 회색조+블루 1색, 모바일 세로 stack, SVG `<title>`·`<desc>`.

   - 자동화 데모에는 [승인] 단계 노출 — 결정과 책임은 사람의 자리.
