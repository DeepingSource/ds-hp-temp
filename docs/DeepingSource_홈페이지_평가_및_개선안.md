# DeepingSource 홈페이지 평가 및 개선안

- **평가 대상:** https://ds-hp-temp.vercel.app/ (임시/프리뷰 배포)
- **평가일:** 2026-07-01
- **평가 범위:** UX·디자인·정보구조 / 카피·메시지·설득력 / 기술·성능·SEO·접근성 / 전환·비즈니스
- **제품:** Anonymized Spatial AI (SAAI) — store count · insight · care · agent

---

## 1. 종합 요약

DeepingSource 랜딩페이지는 "얼굴을 남기지 않는 공간 AI"라는 명확한 차별점을 강한 스토리텔링으로 풀어낸 **완성도 높은 B2B 마케팅 사이트**다. POS의 사각지대(382명 입장 → 65명 결제, −317명의 행동이 사라짐)라는 구체적 문제 제기로 시작해 Observe→Analyze→Suggest→Learn의 운영 루프와 4개 제품으로 자연스럽게 연결되는 서사가 특히 우수하다. 성능도 빠르고(로드 완료 약 0.73초) 기본적인 접근성·SEO 위생도 대체로 갖춰져 있다.

개선 여지는 크게 네 가지다. (1) 임시 도메인이 `index, follow`로 노출되어 **검색 중복 색인 위험**이 있고, (2) `lang="en"` 페이지에 한국어가 섞여 있으며 **파트너 "로고"가 실제 로고가 아닌 텍스트**라 신뢰도·접근성이 약화된다. (3) 히어로 우측 여백과 느린 등장 애니메이션으로 **첫인상이 비어 보이고**, (4) 핵심 실적이 "예시(Illustrative)" 수치라 **설득의 마지막 근거(실제 레퍼런스)**가 부족하다.

| 영역 | 평가 | 핵심 이슈 |
|---|---|---|
| UX·디자인·정보구조 | 양호 | 히어로 우측 공백, 과도한 스크롤 등장 애니메이션 |
| 카피·메시지·설득력 | 우수하나 추상적 | 시적이나 모호함, 실적이 예시 수치 |
| 기술·성능·SEO | 양호, 리스크 1건 | 임시 도메인 색인 허용(중복 색인), 언어 혼용 |
| 접근성 | 보통 | alt 누락 7건, 언어 태깅, 모바일 검증 필요 |
| 전환·비즈니스 | 강함 | 단일 전환 경로, 실제 사회적 증거 부족 |

## 2. 강점 (유지할 것)

- **문제 정의가 구체적이고 설득력 있음.** "입장 382 → 결제 65, POS에 없는 −317"이라는 시각화는 추상적 가치 제안을 즉시 체감시킨다.
- **일관된 서사 구조.** Yesterday·Now·Next / Observe·Analyze·Suggest·Learn 루프가 4개 제품(store count/insight/care/agent)과 1:1로 연결되어 정보구조가 명료하다.
- **제품을 "보여주는" 인터랙션.** SEAL 원본↔익명화 비교 슬라이더, MTMC 다중 카메라 추적 데모가 기술을 말이 아닌 시연으로 전달한다.
- **빠른 성능.** DOMContentLoaded 약 302ms, load 약 733ms. 이미지·JS 번들이 무겁지 않다.
- **SEO·시맨틱 기본기.** 단일 `<h1>`, 논리적 h2/h3 위계, `header/main/nav/footer/section` 랜드마크, title·description·OG image·Twitter card·JSON-LD 2건 구비.
- **접근성 배려 흔적.** 스킵 링크 존재, `prefers-reduced-motion` 미디어쿼리 지원, 외부 링크에 `rel="noopener"` 적용, 레이블 없는 버튼 0건.
- **전환 설계.** 스크롤 시 하단 고정 CTA 바("AI store operations — start today"), 반복되는 "Talk to us", "영업일 1–2일 내 회신" 안심 문구로 리드 유입 동선이 촘촘하다.

## 3. 개선점 (우선순위별)

우선순위: **P0 = 즉시(공개 전 필수) · P1 = 단기(2주 내) · P2 = 중기(개선 백로그)**

### 3.1 기술·성능·SEO·접근성

**[P0] 임시 도메인의 검색엔진 색인 허용 → 중복 색인 위험**
- 현상: `robots` 메타가 `index, follow`. `canonical`은 운영 도메인(`https://deepingsource.io/`)을 가리키지만, 임시 도메인(`ds-hp-temp.vercel.app`) 자체가 크롤링·색인될 수 있다.
- 리스크: 운영 사이트와 임시 사이트가 동시에 색인되면 중복 콘텐츠·순위 분산이 발생하고, 미완성 빌드가 검색 결과에 노출될 수 있다.
- 권장: 프리뷰/스테이징 배포에는 `noindex, nofollow` 적용 + `robots.txt`에서 `Disallow: /` + (가능하면) Vercel 배포 비밀번호 보호. 운영 전환 시에만 색인 허용.

**[P0] 언어 속성과 실제 콘텐츠 불일치 (`lang="en"` + 한국어 혼용)**
- 현상: 문서 `lang="en"`인데 "예시 / Illustrative", 파트너명(올리브영·BGF CU·코리아세븐 등) 등 한국어 텍스트가 포함됨.
- 영향: 스크린리더가 한국어를 영어 발음으로 오독, 다국어 SEO 신호 혼선.
- 권장: EN/KO/JP 언어 스위처가 이미 있으므로 **각 언어 버전에서 완전 로컬라이즈**하거나, 불가피한 혼용 구간은 `lang="ko"` 스팬으로 명시. "예시 / Illustrative" 같은 이중 라벨은 언어별로 하나만 노출.

**[P1] 이미지 대체 텍스트(alt) 누락 7건**
- 현상: `<img>` 16개 중 7개가 alt 미지정/공백.
- 권장: 의미 있는 이미지(제품 대시보드, 시나리오 화면 등)에는 설명형 alt, 순수 장식 이미지에는 `alt=""` 명시로 구분.

**[P1] 파트너 신뢰 섹션이 실제 로고가 아닌 텍스트**
- 현상: "TRUSTED PARTNERS"의 올리브영·현대자동차·롯데월드 등이 **텍스트로만** 렌더링됨(실제 로고 이미지 아님).
- 영향: 프리미엄 디자인 대비 신뢰 섹션만 완성도가 떨어지고, 로고의 즉각적 인지 효과를 놓침. 언어 혼용 문제도 유발.
- 권장: 사용 승인된 실제 브랜드 로고(단색/그레이스케일 처리)로 교체. 각 로고 `img`에 브랜드명 alt 부여.

**[P1] 모바일 반응형 검증 필요**
- 현상: CSS에서 감지된 브레이크포인트가 `1024px` 단일이며, 데스크톱 DOM에서 모바일 내비게이션 토글(햄버거 메뉴)이 확인되지 않음. (평가 환경에서 실제 모바일 뷰포트 강제가 불가해 육안 확인은 미완료.)
- 리스크: 7개 상위 메뉴 + 언어 스위처 + CTA가 좁은 화면에서 어떻게 접히는지 불명확.
- 권장: 실제 모바일(360·390·414px)에서 내비게이션·히어로·좌우 2단 레이아웃·시나리오 카드 가로 스크롤을 QA. 햄버거 메뉴 미구현 시 추가.

### 3.2 UX·디자인·정보구조

**[P1] 히어로 첫인상 — 우측 대형 여백 + 느린 헤드라인 등장**
- 현상: 데스크톱 히어로 우측 절반이 비어 보이고, "Perfect your space." 헤드라인이 단어 단위로 천천히 나타나 초기 화면이 미완성처럼 느껴짐. 의도된 "Anonymous tracking · no ID(ID·01/02/03)" 시각화가 첫 화면에서 즉시 보이지 않음.
- 영향: 가장 중요한 3초 첫인상에서 밀도·완성도 손실, LCP 텍스트가 늦게 확정.
- 권장: 우측 히어로 시각화를 초기 뷰포트에 즉시 노출(또는 여백 축소로 좌측 정렬 강화). 헤드라인 등장 속도를 높이거나 첫 줄은 즉시 표시 후 나머지만 애니메이션.

**[P2] 전 섹션에 걸친 과도한 스크롤 등장(reveal) 애니메이션**
- 현상: 대부분 섹션이 `opacity:0`에서 스크롤 진입 시 페이드인. 빠르게 스크롤하면 콘텐츠가 순간적으로 비어 보이는 구간 발생.
- 완화 요소: `prefers-reduced-motion` 지원은 이미 되어 있음(양호).
- 권장: 등장 트리거 임계값·지속시간을 짧게, 접힘선(above-the-fold) 콘텐츠는 애니메이션 제외해 항상 즉시 표시.

**[P2] 카운트업 애니메이션 중 잘못된 수치 노출**
- 현상: 통계 카운터가 최종값 도달 전 중간값을 표시(예: "103 특허"가 "24"로, "−317"이 "−58"로 잠깐 보임).
- 권장: 최종 수치를 마크업에 두고 애니메이션은 시각 효과로만. 화면 캡처·저사양 기기에서 오해 소지 제거.

### 3.3 카피·메시지·설득력

**[P1] 핵심 실적이 "예시(Illustrative)" 수치**
- 현상: 현장 시나리오의 "매출 +10%", "체류시간 +25%", "회전율 +15%"가 모두 "예시 / Illustrative"로 표기됨.
- 평가: 정직한 표기는 신뢰에 긍정적이나, B2B 구매자에게는 **가공 예시가 실적처럼 배치**되어 설득의 마지막 근거가 약하다.
- 권장: 익명화된 실제 레퍼런스("국내 대형 편의점 프랜차이즈, 3개월" 형태)로 최소 1~2건 교체. 불가 시 "예상 효과 모델"임을 더 명확히 프레이밍.

**[P2] 시적이지만 모호한 상위 카피**
- 현상: "We weave the in-between", "The force that remade online — now for every space" 등은 감성적이나 처음 방문한 구매자에게 **무엇을 하는 제품인지**를 즉시 전달하지 못함.
- 권장: 상징적 헤드라인 바로 아래에 1문장의 구체적 서브카피(무엇을·누구에게·어떤 결과) 병기. 브랜드 감성과 명료성의 균형.

**[P2] 한·영 이중 라벨의 산발적 노출**
- 현상: "예시 / Illustrative"처럼 두 언어를 동시에 노출하는 라벨이 영어 페이지의 톤을 흐림.
- 권장: 언어별 단일 표기로 통일(3.1의 로컬라이즈 항목과 연동).

### 3.4 전환·비즈니스

**[P1] 실제 사회적 증거(Social Proof) 부족 — 특히 프라이버시 인증**
- 현상: 고객 인용문·실제 사례 로고·수치 기반 ROI·보안/개인정보 인증 배지가 없음. 핵심 가치가 "프라이버시"인데 이를 **제3자 검증으로 뒷받침하는 근거**가 비어 있다.
- 권장: 고객 한줄 후기, 실제 케이스 스터디 링크, ISO 27001·개인정보보호(예: 개인정보 비식별 관련 인증/심의 결과) 배지, NVIDIA Inception 외 검증 로고를 신뢰 섹션에 추가.

**[P2] 단일 전환 경로(상담 문의)로 수렴**
- 현상: 거의 모든 CTA가 "Talk to us / 무료 상담"으로 귀결. 자기주도 학습(제품 데모 영상, 실제 대시보드 스크린샷 갤러리, 제품 상세 페이지 미리보기)이 상대적으로 약함.
- 권장: "상담 요청"과 병행해 **저마찰 2차 CTA**(데모 영상 보기, 제품 원페이저 다운로드, 케이스 스터디 읽기)를 제공해 아직 상담 준비가 안 된 리드도 육성.

## 4. 실행 로드맵

### 4.1 즉시 (공개 전 필수, P0)
1. 임시 도메인 `noindex` + `robots.txt Disallow` (+ 배포 비밀번호). 운영 도메인 전환 시에만 색인 허용.
2. 언어 속성 정합성: 완전 로컬라이즈 또는 한국어 구간 `lang="ko"` 태깅, 이중 라벨 단일화.

### 4.2 단기 (2주 내, P1)
3. 파트너 섹션을 실제 로고(그레이스케일)로 교체 + alt 부여.
4. 이미지 alt 7건 보강(의미형 vs 장식형 구분).
5. 히어로 우측 시각화 즉시 노출 + 헤드라인 등장 속도 개선.
6. 현장 시나리오를 실제 익명화 레퍼런스로 최소 1~2건 교체.
7. 프라이버시·보안 인증 배지 및 고객 후기 추가.
8. 실제 모바일 기기 반응형 QA(내비게이션 토글 포함).

### 4.3 중기 (개선 백로그, P2)
9. 스크롤 등장·카운트업 애니메이션 최적화(above-the-fold 제외, 최종값 마크업).
10. 상위 카피에 구체적 서브카피 병기.
11. 저마찰 2차 CTA(데모 영상·원페이저·케이스 스터디) 도입.

## 부록 B. 평가 방법
- 실제 브라우저 렌더링 기반(스크롤 전체 캡처), DOM/CSS 스크립트 점검, 콘솔·성능 API 계측.
- 한계: 평가 환경에서 실제 모바일 뷰포트 강제가 불가하여 모바일 반응형은 **육안 검증 미완료**(권장 QA 항목으로 분류). 외부 스타일시트 CORS 제약으로 일부 브레이크포인트가 누락 집계되었을 수 있음.
