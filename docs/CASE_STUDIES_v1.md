## 0. 한 페이지 요약

5건의 case-study 는 새 `deepingsource.io` 의 **신뢰의 척추** 다. 사이트의 다른 모든 페이지 — `/products/*`, `/technology/*`, `/solutions/*`, `/enterprise`, `/pricing` — 가 *약속* 의 자리라면, `/resources/case-studies` 는 *증명* 의 자리다. 동시에 영업팀이 *미팅 후속* 자료로 즉시 꺼낼 수 있는 직거리 자산이며, `SAAI_HQ_Sales_Deck_v1` 슬라이드 11 (*동료의 결정으로 안심*) 의 가상 인용을 *실명·실측* 으로 갈아끼우는 자리다.

**5건의 전략적 자리 한 줄** — 53명 실측 통계 (Case 1) 를 *척추*, Golden Case 5단계 (발견·검증·번역·동기화·재측정) 의 *각 단계를 대표하는 4건* 을 *근육* 으로 묶어, 본사 임원·점주·기술 의사결정자 *3 청자* 모두에게 *자기 자리* 가 보이게 한다.

## 1. 5건의 큐레이션 원칙

5건은 *우연한 선택* 이 아니라, 회사 사이트가 동시에 응대해야 하는 *3 청자* 와 *4 제품* 의 매트릭스를 *최소 비용으로 메우는* 큐레이션이다. 본 §1 의 5가지 원칙이 동시에 성립하도록 잡혔다.

**원칙 1 — 3 청자를 *모두* 1건 이상 다룬다.** 본사 임원 (Case 2·3·4), 점주 (Case 1·4), 기술 의사결정자 (Case 5). Case 4 는 의도적으로 *본사 + 점주 양시각* 으로 짜여 있어, 한 페이지에서 두 청자가 *자기 자리* 를 동시에 본다.

**원칙 2 — 4 도메인 중 *3개 이상* 의 자산이 등장한다.** `deepingsource.io` (Case 2·3·5 의 Store Insight × StoreAgent), `storecare.ai` (Case 1·4 의 StoreCare 4 모듈), `app.deepingsource.io` (Case 2·3·5 의 본사 대시보드 캡처 자리). `saai.store` 는 5건 중에는 *언급되지 않는다* — 그 자리는 회사 사이트가 *B2C 콘텐츠 분기* 로 외부 이관(D2)했기 때문에 case-study 의 자리가 아님 (별도 saai.store/archive 에서 운영).

**원칙 3 — 4 제품을 *모두* 1번 이상 등장.** Store Insight (Case 2·3·4), Store Agent (Case 2·3), Store Care (Case 1·4), SAAI (Case 5 의 *익명화 공간 AI* 카테고리어 자리에서). 한 케이스가 *2 제품* 의 결합을 보이면 자동으로 `/products` 진열대 (D1) 의 *4 카드 → 결합* 의 증명이 된다.

**원칙 4 — Golden Case 5단계 의 *각 단계를 대표* 하도록 매핑.** 5건이 *발견 → 검증 → 번역 → 동기화 → 재측정* 의 5단계와 *1대1 정렬* 된다. 회사 사이트 방문자가 5건을 *순서대로* 읽으면 본사 운영 사이클이 머리에 그려진다.

| Case | 대표 Golden Case 단계 | 단계 약식 명칭 (사용자 spec) |
|---|---|---|
| 1 | 01 발견 (Discover) | 발견 |
| 2 | 02 검증 (Verify) | 검증 |
| 3 | 03 번역 (Translate) | 표준화 |
| 4 | 04 동기화 (Sync) | 전파 |
| 5 | 05 재측정 (Re-measure) | 운영자산화 |

**원칙 5 — 산업 분포 *4개 이상*.** 편의점 (Case 1·2) · 드럭스토어 (Case 3) · 카페 (Case 4) · 대형 공간 — 전시·박물관 (Case 5). Upgrade Spec A3 의 *대형 공간 신설* 자리가 Case 5 의 등장으로 회사 사이트의 첫 *대형 공간 사례* 가 된다. *무인매장* 은 Case 1 안에 21곳 표본으로 포함 (별도 케이스 신설 없음 — Phase 5 이후 운영 큐에서 분기).

## 4. Phase 2 작업 분해

본 §4 가 v1 발행 *직후* 의 작업 큐. Phase 2 의 2주 내에 v1 골격을 *카피 완성도* 로 끌어올린 v1.1 을 만든다.

**1주차 — 척추 확정 (현재 v1 가 커버하는 자리).**
- 5건 헤드라인 + 서브헤드라인 결정 (✅ v1 에서 완료).
- 5건 고객 가명 결정 (✅ v1 에서 A·B·C·D + 53명 표본).
- 5건 Golden Case 5단계 1대1 매핑 (✅ v1 에서 완료).
- 산출물 — *본 문서 v1*.

**2주차 — 본문 깊이.**
- 5건 스토리 구조 4-step 본문 카피 (각 케이스 200-300자) — v1 의 *Seam·Thread·Knot·Weave* 한 줄을 *3-4 문장 prose* 로 풀어 쓴다.
- 5건 인용 카피 다듬기 — v1 의 가상 인용을 *Brand Voice Guidelines v2* 의 *형용사 X · 사실로* 룰로 정제. 15단어 이내 유지.
- 5건 요청 사진·데이터 spec 의 *수량·자리·해상도* 를 디자인 시스템 차트 토큰과 정렬.
- 본사 *실명·익명* 결정 — 5건 중 *몇 건을 실명으로* 갈지 영업팀과 미리 협의.
- 산출물 — `docs/CASE_STUDIES_v1.1.md` (본문 prose 박힘).

**Phase 3 진입 시 — 디자인 시스템 정렬.**
- 5건 페이지의 *시각 자산 박음 spec* 을 디자인 시스템 dist CSS 에 박는다.
- 차트 토큰 5종 (막대·도넛·라인·매트릭스·타임라인) 이 본 문서의 *요청 자산 spec* 과 1대1 정렬.
- 인용 카드·KPI 카드·Before/After 비교 카드의 *3 컴포넌트 정의* (디자인 시스템 ui_kits/corporate/ 에 추가).

**Phase 4 진입 시 — 발행.**
- 5건 페이지 발행 (ko 우선, en 동시).
- *(가상·차후 확정)* 자리를 *실측 데이터* 로 교체. Phase 2-3 동안 진행된 영업·도입 결과를 박음.
- D10 자산 수집(임원 사진·매장 사진·매체 로고) 결과를 박음.
- `/resources/case-studies` 인덱스 (§3 표) + 5개 상세 페이지 = *총 6 라우트* 발행.

## 5. 발행 후 운영 룰 (v1.2 후순위)

5건 발행 후의 *상시 발행 큐* 는 본 v1 의 범위를 *넘는다* . PLAN v1.1 §8 *2차 Q* 의 *D9 의 case-study 5건 이후 상시 발행 큐 운영* 자리. 본 §5 는 그 자리의 *밑그림* 만 박는다.

- **발행 주기.** 도입 매장이 *6개월 시점* 에 첫 측정 데이터를 갖춘다 (`SAAI_Page_Company_Group_v1` §4.5 *향후 케이스 그릇* 의 *2026 Q4·2027 Q1·2027 Q2* 일정과 정합). 초기 6개월은 *분기 1건*, 정착 후 *월 1건* 을 목표.
- **다음 5건의 후보 그릇.** (1) 대형마트 (2026 Q4) (2) 다른 드럭스토어 본사 (2027 Q1) (3) 다른 카페 체인 (2027 Q1) (4) 무인매장 단독 케이스 (2027 Q1) (5) 공장·물류·병원 등 *Beyond Retail* 신규 시장 (2027 Q2+).
- **포맷의 일관성.** 새 5건도 본 v1 의 *골격 카드 12 항목* 을 *그대로* 따른다. 디자인은 한 번에 박힌 자산을 *상속* .
- **운영 거버넌스.** v1.2 에서 *케이스 발행 트리거 (도입 후 6개월 + 합의 + KPI 측정 완료)* · *본사 실명 협의 절차* · *번역 표준 (5단계 매핑 룰)* 을 명문화.

본 §5 의 운영 룰은 v1.2 의 *작업 자리* 로 넘긴다.

## 6. 신설 필요 자산 — Phase 2-3 자산 수집(D10) 에 박을 항목

본 §6 의 자산 목록을 `docs/ASSET_COLLECTION_v1.md` (TaskList #12) 에 *직접 박는다* — 자산 수집(D10) 의 *case-study 트랙* 이 본 §6 그대로.

**케이스별 매장·시설 외관·내부 사진** — 5건 × 평균 2-3컷 = 약 13컷. 모두 흑백 또는 회사 사이트 디자인 시스템에 맞춘 *저채도 tinting* 처리. 사람 등장 시 *익명화 처리* 또는 *부감 컷* . 외부 발주 (Case 5 의 전시 공간) 또는 도입 매장에서 자체 촬영 (Case 1·2·3·4).

**대시보드·모바일 UI 캡처** — 5건 × 평균 2-3장 = 약 13장. Store Insight 본사 대시보드 (Case 2·3·4), StoreCare 모바일 알림 (Case 1·4), StoreAgent 권고 카드 (Case 2·3), MTMC 시각화 (Case 5). *실측 데이터* 가 박힌 화면이어야 함 — Phase 4 발행 시점에 *진짜 화면* 으로 갈아끼움.

**KPI 그래프** — 5건 × 평균 2장 = 약 10장. 디자인 시스템의 *차트 토큰 5종* (막대·도넛·라인·매트릭스·타임라인) 을 사용. v1 의 *대표 숫자 1개* 가 각 차트의 *제목 자리* 가 됨.

**인터뷰 인용 영상·사진 (선택, Phase 5)** — 5건 × 1-2 인용자 = 약 7-10명. 본사 실명 협의가 끝난 케이스만. 인용자 본인 *상반신 컷* + 익명 인용 카드. 영상은 *15초 이내* 의 *한 줄 인용 컷* .

**인포그래픽 — Golden Case 5단계 다이어그램** — `/resources/case-studies` 인덱스 페이지에 *공통 자산* 1장. 5단계가 *5건의 케이스* 와 *1대1 매핑* 된 인포그래픽. `SAAI_Website_Upgrade_Spec_v1` B5 의 카피 5줄 그대로 박음.

**자산 수집 일정.** PLAN v1.1 §5.3 *자산 수집 일정* 의 D10 트랙과 정합 — Phase 2 시작 시 자리 식별 (본 §6 이 자리 식별) → Phase 2 + 1주 외부 발주 → Phase 3 시작 전 촬영·수집 완료 → Phase 4 시작 전 가공·정돈 → Phase 4 박음.
