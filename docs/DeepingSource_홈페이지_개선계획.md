## P1 — 메시지·카피·IA 정비

### 1-A. 홈 히어로 재편 ★메모 핵심
현재 히어로("당신의 공간을, 완벽하게")는 감성 카피만 있고 **무엇을 하는 회사/제품인지** 첫 화면에서 설명되지 않음.
- 히어로에 한 줄 정의 추가: "기존 CCTV 위에 올리는 익명화 매장 분석 AI" 수준의 구체적 서술
- 대상(프랜차이즈 본사/개인 매장)과 제공 가치(유입→동선→이상감지→실행 제안)를 3초 안에 파악 가능하게
- 서브카피에 SAAI Suite 개념(1-B)과 연결

### 1-B. "SAAI Suite" 개념 도입 ★메모 핵심
/products 및 전 사이트에 4개 제품을 묶는 **"SAAI Suite"** 브랜드 프레임 신설.
- /products 히어로: "SAAI Suite — 하나의 운영 루프, 네 개의 제품"
- 홈·기술·솔루션·요금 페이지의 제품 언급을 Suite 프레임으로 통일
- 부수 효과: "Observe·Analyze·Suggest·Learn" 라벨과 제품 성격 미스매치(분석 지적)도 Suite 재정의 시 함께 해소
- saai care의 B2B/B2C 혼선(storecare.ai CTA 이탈)도 Suite 내 역할 정의로 정리

### 1-C. 솔루션 히어로 카피 전면 재작성 ★메모 핵심
| 페이지 | 현재 | 방향 (메모) |
|---|---|---|
| /solutions/retail | "업종이 달라도 한 매장처럼" | "한 매장처럼" 메시지가 애매 — hero copy 재고민 |
| /solutions/food-beverage | 동일 템플릿 | **"매장 운영의 표준화"** 관점으로 히어로 변경 |
| /solutions/drug-store | 동일 템플릿 | retail과 동반 재작성 (템플릿 반복 해소) |
| /solutions/large-space | "모든 공간을, 한 매장처럼" | **"대형 공간을 한눈에 관리"** 관점으로 hero부터 재구성 |

분석에서 지적된 "3개 업종 페이지 템플릿 반복" 문제를 히어로 재작성과 함께 업종별 고유 페인포인트(허브의 21개 카드에는 이미 존재) 반영으로 해소.

### 1-D. /solutions 허브 탐색 flow 재설계 ★메모 핵심
현재 21개 카드 나열로 "읽을 게 너무 많음". depth와 flow를 고려해 **사용자가 자기 솔루션을 찾아가는 구조**로:
- 1단계: 업종 선택(대형 카드 4~7개) → 2단계: 해당 업종의 문제-해결 가이드 노출
- 또는 "우리 매장은 어떤 유형인가요?" 질문형 진입(매장 수·업종 2문항)
- 무인매장·전시공간·물류창고의 전용 페이지 부재(분석 지적)도 이 재설계에서 함께 결정

### 1-E. 기타 카피·메시지
- Anonymizer ↔ SEAL 관계(엔진 vs SDK)를 양 페이지 상단에 명시, 중복 문구 축소 (분석)
- 요금 ↔ 엔터프라이즈 페이지 관계 설명 추가: "몇 개 매장부터는 엔터프라이즈 상담" 기준 제시 (분석)
- Spatial AI의 CES/KDDI/NVIDIA 레퍼런스에 근거(링크·연도) 추가 또는 삭제 (분석)
- FAQ saai count 섹션 신설, 섹션 간 문체(격식체/구어체) 통일 (분석)

## P2 — 디자인·목업·애니메이션 고도화

### 2-A. 제품 페이지 목업 개선 ★메모 핵심
| 페이지 | 작업 |
|---|---|
| saai insight | ① "감으로 보던 매장, 데이터로 봅니다" 이미지를 목업으로 대체 검토 ② "전체 기능 17가지" 나열식 → 카테고리 그룹핑+시각화로 재디자인 ③ "매출을 끌어다 놓으면, 체류와 매출이 한 축에서 만납니다" — 기본 기능이 아니므로 **표현 강도 낮추기**(옵션 기능임을 명시) |
| saai care | "필요한 순간만, 손안으로" 영역 디자인·목업 개선 |
| saai agent | ① **웹 목업 중심으로 재구성, 모바일 화면도 웹 목업 기준에 맞춰 통일** ② "사장님이 보는 화면" Interface 섹션 제거 또는 축소 (B2B 톤 유지) ③ (분석) 채팅/푸시 탭 콘텐츠 빈약 — 웹 목업 집중 방침에 맞춰 정리 |

### 2-B. FAQ 컴포넌트 전면 재디자인 ★메모 핵심
- anonymizer·seal 페이지 FAQ가 "AI slop" 느낌 — **디자인 시스템(design-system/ 참조)에 맞춰 재설계**
- /resources/faq 페이지 디자인 재검토 포함
- 사이트 전체 FAQ 컴포넌트를 하나로 통일 (기술 페이지·리소스 페이지 공용)
- Spatial AI 페이지에 FAQ 없음(분석) — 통일 컴포넌트 도입 시 신설

### 2-C. 기술 페이지 시각 표현
| 페이지 | 작업 |
|---|---|
| /technology | "세 개의 기술 축" 섹션 우측 여백 과다 — 레이아웃 조정 |
| spatial-ai | "카메라가 여러 대여도…" MTMC 섹션 애니메이션 재고 — **카메라 뷰 + 지도(평면도) 매핑 화면**으로 표현 |
| models | 비전 모델 카탈로그 애니메이션·표현이 애매 — 시각적으로 발전 (상태 배지 정리 포함) |

## P3 — 콘텐츠 자산 보강

1. **고객사 실명·로고 노출** (분석 최대 리스크): 올리브영·CU·코리아세븐·CJ푸드빌·롯데GRS·현대차·롯데월드·국립박물관재단 — 사용 허가 확보 후 솔루션·케이스스터디·엔터프라이즈 페이지에 배치. 허가 불가 시 "업종+규모" 준실명 표기라도 격상
2. **케이스 스터디 강화**: "예시 수치" → 실측 사례 전환, 업종 필터, PDF 다운로드
3. **회사 카테고리 빈 페이지 해소**: Events 과거 아카이브 추가, Career ATS 연동 또는 "채용 예정 직군" 명시
4. **리소스 검색·필터**: docs/glossary/faq/case-studies 공통 검색 도입
5. **세그먼트별 문의 폼 분리**: IR/파트너/채용/미디어 전용 폼 (현재 전부 mailto/공용 contact)
6. **요금 페이지**: 총액 계산기(카메라 대수 기준), 30% 할인 기준 명시

## 실행 순서 제안

| 스프린트 | 범위 | 비고 |
|---|---|---|
| **Sprint 1** | P0 전체 (14건) | 대부분 코드 수정. drug→drug-store는 리다이렉트 포함. 1~2일 규모 |
| **Sprint 2** | P1-A/B (홈 히어로 + SAAI Suite) | 카피 확정이 선행 — 이후 모든 페이지 문구의 기준이 됨 |
| **Sprint 3** | P1-C/D/E (솔루션 카피·flow + 기타) | Suite 프레임 확정 후 진행 |
| **Sprint 4** | P2 (목업·FAQ·애니메이션) | 디자인 시스템 기준 컴포넌트 작업 |
| **Sprint 5+** | P3 (자산 보강) | 고객사 허가 등 외부 의존 — 병렬로 섭외 시작 권장 |

**의존 관계 주의**: SAAI Suite 네이밍(1-B)과 태그라인 정책(0-10)은 다른 카피 작업의 전제이므로 가장 먼저 확정 필요. 고객사 로고 허가(P3-1)는 리드타임이 길어 Sprint 1과 동시에 섭외 착수 권장.
