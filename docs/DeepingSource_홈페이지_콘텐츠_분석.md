## 1. 홈 (/ko)

히어로("당신의 공간을, 완벽하게") → 문제 제기(POS만으로는 매장 내 행동이 안 보임) → SAAI 4글자 의미 → 제품 4종 소개 → 현장 사례 3건 → SEAL 신뢰 섹션 → Spatial AI(MTMC) 데모 → 3-store 흐름 → 업종별 진입점 6개 → 신뢰 지표(특허/파트너/고객사 로고) → 최종 CTA. 구성 자체는 설득력 있는 내러티브지만, 신뢰 지표 섹션의 "국내외 특허 0개+", "파트너 브랜드 0개+"가 **숫자 카운터 애니메이션이 초기값(0)으로 노출**되는 QA 결함이 있고, 푸터에 "준비 중" placeholder 2건이 그대로 노출됨.

## 2. 제품 (5개 페이지)

| 페이지 | 핵심 메시지 | 비고 |
|---|---|---|
| /products | "네 개의 제품, 하나의 운영 루프" | 허브. 태그라인이 개별 페이지("당신의 공간을")와 불일치("모든 공간을") |
| saai count | "상권분석, 해야 하는데 못 하고 계셨죠?" | 헤드라인 띄어쓰기 오류, insight와 예시 수치 중복 |
| saai insight | "POS가 못 센 것을 봅니다" | 5개 중 콘텐츠 최다·최밀도. 대시보드 카운터가 "0.0"으로 렌더 |
| saai care | "누가 일해도, 언제나 완벽한 매장" | 유일하게 B2C 가격 노출 + CTA가 외부 사이트(storecare.ai)로 이탈 |
| saai agent | "보는 AI를 넘어, 운영하는 AI" | 인터랙티브 데모 강점, 채팅/알림 탭 콘텐츠 빈약 |

**공통 이슈**: (1) 제품별 가격 노출 정책이 제각각(무가격/하드웨어 대여료/소비자가/별도 페이지 유도)이라 "이게 얼마짜리 제품군인가"를 가늠하기 어려움. (2) care만 B2B 내비게이션 안에서 B2C 사이트로 CTA가 빠짐. (3) 소개 프레임("Observe·Analyze·Suggest·Learn")과 실제 제품 성격(care=보호, agent=제안·실행)이 라벨과 어긋남. (4) 전 페이지 푸터에 "준비 중" 반복.

## 3. 기술 (5개 페이지)

| 페이지 | 핵심 메시지 | 비고 |
|---|---|---|
| /technology | "비전 AI는 결국 사람을 봅니다" | 3대 축 카드 중 2개가 잘못된 하위페이지로 링크(Spatial AI→SEAL, Agentic AI→Models) |
| Anonymizer | "신원은 지우고, 분석은 그대로" | 결론부에 SEAL 브랜드명 등장해 두 기술 간 경계 혼선 |
| SEAL | 4원칙(Secure·Embeddable·Adaptable·Lightweight), 법조항 직접 인용 | Anonymizer와 문구 상당 부분 중복, 관계 설명 없음 |
| Spatial AI | "공간을 읽는 것이, 먼저입니다" | CES/KDDI/NVIDIA 레퍼런스에 링크·로고 등 근거 없음(검증 불가) |
| Vision Models | 모델 카탈로그 | 내비게이션 "20여 개"vs 실제 표 13개/카드 15개로 수치 불일치 |

**공통 이슈**: 법 조항·특허 등 정량적 신뢰 자료는 우수하나 페이지별 편차가 큼(Spatial AI만 근거 부실). Anonymizer/SEAL 관계가 어디에도 명시되지 않아 개념 혼동 유발. 개발자 대상 코드/기술 태그가 비개발자 의사결정자에게는 진입장벽. 링크·수치 QA 결함 다수.

## 8. 사이트 전반 공통 이슈 (Cross-cutting)

**A. QA/버그성 이슈 (선(先)수정 권장)**
1. 신뢰 지표 카운터가 "0개+"로 렌더링(홈 하단, saai insight 대시보드) — 애니메이션 초기값 노출 버그
2. 기술 개요 페이지 카드 링크 오배치(Spatial AI 카드→SEAL 페이지, Agentic AI 카드→Models 페이지)
3. 용어 사전 상세 링크 `/ko` 로케일 프리픽스 누락 의심(404 위험) — 솔루션 허브 카드 링크도 동일 의심
4. Vision Models "20여 개" 표기 vs 실제 13/15개 수치 불일치
5. 카페·음식점 페이지 breadcrumb("카페·외식")과 내비게이션 라벨("카페·음식점") 불일치
6. 회사소개 페이지 리더십 소개 문장이 중간에 잘려 노출
7. 제품·기술 페이지 전반의 "준비 중" placeholder, saai count 헤드라인 띄어쓰기 오류

**B. 메시지 일관성 이슈**
- 태그라인 불일치: "당신의 공간을" vs "모든 공간을"
- Anonymizer와 SEAL의 관계(엔진 vs SDK)가 어디에도 명시되지 않아 두 기술 개념이 혼동됨
- saai care만 B2B 내비게이션 안에서 B2C 사이트(storecare.ai)로 CTA 이탈 — 세그먼트 혼선
- "Observe·Analyze·Suggest·Learn" 운영 루프 라벨과 실제 제품 성격(care/agent)의 미스매치
- 요금(개인 매장 구체가) ↔ 엔터프라이즈(완전 비공개) 사이 가격 정보 격차, 두 페이지 관계 미설명

**C. 콘텐츠 깊이·차별화 이슈**
- 솔루션 3개 업종 페이지(리테일/카페/드럭)가 사실상 동일 템플릿 반복, large-space만 실질 차별화
- 회사 카테고리 내 Events(1건)·Career(공고 0건)가 사실상 빈 페이지 수준
- 전 사이트에 걸쳐 **실제 고객사 로고·실명 사례가 거의 없음** — 올리브영·CU·세븐일레븐·CJ푸드빌·롯데GRS·현대차·롯데월드·국립박물관재단 등 배경상 실존하는 대표 레퍼런스가 솔루션/케이스스터디 어디서도 로고·실명으로 노출되지 않고, 대부분 "예시 수치"·"익명 인터뷰"로 처리됨 → B2B 세일즈 사이트의 신뢰 구축 관점에서 가장 큰 공통 리스크

**D. 정보 탐색성 이슈**
- 리소스 4개 페이지(docs/glossary/faq/case-studies) 전부 검색·필터 기능 없음
- FAQ에 saai count 전용 섹션 누락, docs·glossary·FAQ 간 상호 링크 약함

## 9. 다음 단계 제안 (개선 계획 수립 전 우선순위 스케치)

3. **콘텐츠 자산 보강** (8-C, 8-D) — 실고객 로고/사례 확보, 업종 페이지 차별화, 검색 기능 등 상대적으로 리소스가 큰 작업
