## 0. v1 → v2에서 바뀐 것

v1은 "사이트의 빈 곳"에서 출발했다. v2는 **브랜드 스펙이 이미 발주해 놓은 시각물**에서 출발한다. 검토 결과:

1. **`03_web_specs/`가 시각화 5종 + 모킹 3종을 번호까지 붙여 명세해 놓았다** (각 페이지 §6). v1 제안 중 상당수가 이미 스펙 존재 — 발명할 게 아니라 구현하면 된다.
2. **Website Upgrade Spec의 Tier A "전략적 빈자리" 5개**가 v1의 직감("기술 레이어 공백")을 공식 확인하고, 카피·노드 구조·Acceptance Criteria까지 제공한다.
3. **Voice Variation Matrix(Voice v2.5 §6)의 "시각 hero" 행**이 제품별로 어떤 화면을 보여줘야 하는지 지정한다 — Insight `가설 카드·Funnel·KPI 사전` / Care `알림 모킹·Before/After·요금 카드` / Agent `1~3순위 카드·L0→L5·우선순위 엔진`.
4. 그 결과 **v1 제안 일부는 브랜드 룰과 충돌**함이 드러났다 (§4 오픈 결정).

## 2. 제안 목록 v2 — 브랜드 스펙 매핑 포함

### Tier 1 — 스펙이 이미 발주한 것 (카피까지 확정, 구현만 남음)

#### 2-1. IntegratedLoopDiagram — 세 제품 통합 신호 흐름 ★ v1의 SaaiPlatform + IntegrationMap 통합·승격

- **스펙 출처**: Upgrade Spec **A2** (카피·노드 구조·AC 확정)
- **배치**: 홈 Section 05 다음 + products/saai Hero
- **구성(스펙 그대로)**: 좌→우 — 입력(CCTV·POS·외부 컨텍스트) → SEAL+Anonymizer(블루 액센트) → **익명화 이벤트 허브** → Insight(어제)↔Care(지금)↔Agent(다음) 3분기 → 사람의 결정·행동 → 허브로 되먹임 점선. 노드 hover 툴팁. 가로 1.6:1, 모바일 세로 stack.
- **헤딩(확정 카피)**: "세 시간 축이 하나의 운영 루프로 닫힙니다"
- **v1 대비 업그레이드**: v1의 "3제품 위젯 대시보드"보다 이 다이어그램이 브랜드의 공식 답이다. 플랫폼 증명을 *UI 합성*이 아니라 *신호 흐름*으로 한다 — Architecture v4의 `SOLUTION = ∑ SOURCE` 합성 구조와 1:1. 위젯 대시보드 안은 보류(§4-3).
- **애니메이션**: 신호 펄스가 입력→허브→3분기→사람→되먹임으로 한 바퀴 도는 루프(useMockupLoop). 한 바퀴 = "관찰·분석·제안·학습" 사이클의 시각 번역.
- **난이도 중** / 회색조+블루라 기존 목업보다 제작 단순

#### 2-2. AutonomyLadderTimeline — L0→L5 자율 주행 진화 곡선

- **스펙 출처**: Upgrade Spec **A3** + Agent 페이지 스펙 시각화 #5 (6단계 카피 확정)
- **배치**: 홈 Beyond Retail 앞 + products/store-agent Z6
- **구성**: 가로 timeline L0(관찰만)→L5(자율 주행 매장). 노드가 갈수록 회색→블루로 밝아짐. 노드 클릭 시 대표 카테고리·진화 조건 카드 펼침. **진화 조건 footnote**(승인률 80%↑ · 검증 30회↑ · 명시적 합의 · 충돌 시 롤백)가 신뢰 장치의 핵심이므로 생략 금지.
- **포인트**: "결정과 책임은 사람의 자리"를 시각화하는 브랜드 차원의 자산. Agent의 L1~L4 어휘(Voice Matrix 숫자 단위 특화)가 사이트에 처음 등장하는 자리.
- **난이도 하~중** (인터랙션 포함 중)

#### 2-3. SpatialTrajectoryMockup — MTMC 멀티캠 궤적 (v1 1-2 유지, 스펙 정합 보강)

- **스펙 출처**: Upgrade Spec **A5** — v1 제안과 동일 방향, 단 스펙이 디테일을 고정한다:
  - 카메라 **3대**, 한 사람의 동선이 세 캠에 걸쳐 이어짐, 동선은 **점선**
  - 얼굴 자리는 **둥근 회색 도형(모자이크 톤)** — "얼굴 없이도 연속 추적"이 메시지의 심장
  - **회색조 + 블루 1색** — v1에서 구상한 실사 CCTV+컬러 궤적은 이 자리에선 과함. 실사 버전은 solutions 쪽 보조로.
- **헤딩(확정 카피)**: "카메라가 여러 대여도, 같은 사람을 연속 추적합니다 — 얼굴 없이"
- **보강**: 3 약속 카드(MTMC / Re-ID 얼굴 아님 / Any Environment) 동반. Re-ID 카드가 시각 강조 대상(AC 명시).
- **난이도 중** (v1의 '상'에서 하향 — 스펙이 단순한 형태를 지정)

#### 2-4. PriorityEngineDiagram — 우선순위 엔진 5단계

- **스펙 출처**: Agent 페이지 스펙 시각화 #4 (Z5.1)
- **구성**: Insight·Care·외부 컨텍스트·이력 (입력 4~5) → 우선순위 점수 → 권고 클래스(L1·L2·L3·L4) → 1~3순위 카드 → 사람 → 학습 되먹임.
- **포인트**: 기존 ActionCardMockup이 "결과"라면 이건 "원리". 둘을 같은 페이지에 세우면 *카드가 어디서 오는가*가 닫힌다. Agent 보이스 룰대로 라벨 **영문 우선**.
- **난이도 하** (정적 SVG + 펄스)

#### 2-5. FunnelDiagram — visit→stay→gaze→pick→buy

- **스펙 출처**: Insight 페이지 스펙 시각화 #2 (Z5.2)
- **구성**: 5단계 퍼널, 각 단계 이탈 가지. 단계별 수치는 캐노니컬 SOT에서. BLUF 룰 적용 — 제목은 "gaze→pick에서 절반이 떠납니다" 같은 takeaway.
- **포인트**: Insight의 고유 프레임(3 Decisions)과 "가설 한 줄" 보이스를 실을 수 있는 유일한 신규 자리 — 퍼널 아래 "가설: 음료 매대 전면 진열이 pick 전환을 막고 있다 → 검증 제안" 카드를 붙이면 Voice Matrix의 `가설 카드` hero 요구까지 충족.
- **난이도 하**

## 3. 기존 목업에도 소급되는 브랜드 정합 (보너스 발견)

신규 제작 전 짚고 갈 기존 목업의 브랜드 어긋남:

1. **차트 제목이 서술형** — "시간대별 매출 추이"는 BLUF 위반. takeaway 제목으로 교체(MOCKUP_REVIEW 1-2와 같은 사이클에서).
2. **Insight 목업에 '가설' 부재** — AI 인사이트 문구가 단정형. Voice는 "가설이 함께?"를 자기 점검 8문으로 명시. "…효과로 보입니다 → 검증하려면" 결로 1줄 추가.
3. **Care 알림이 30자 규격 초과** — StoreCareMockup·PushNotification 알림 본문 길이 재작성 대상.
4. **Agent 목업의 언어** — Voice는 Agent 영문 우선. 현재 ko 로케일 카드가 전부 한글 — 점주向 화면이라면 정당하나, 본사向(enterprise) 자리에서는 영문 카드가 브랜드 정합.
5. **목업 헤더 표기 "STORECARE"** — Architecture v4의 가치 브랜드 표기는 `saai care`(소문자, saai 동반). 구현체 표기(store care)와 혼재 중 — §4-1 오픈 결정.
