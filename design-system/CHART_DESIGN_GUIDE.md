## 0. 의사결정 트리 — 차트 만들기 전 5초

```
[질문 1] 보여주고 싶은 핵심 메시지가 한 문장으로 나오는가?
   ├─ NO  → 만들지 마라. 먼저 메시지부터 정의한다 (§1)
   └─ YES → ↓

[질문 2] 메시지가 "변화/비교/구성/관계/분포/순위/지리/밀도/흐름" 중 무엇인가?
   └─ §4 차트 유형 선택표로 이동

[질문 3] 독자가 직접 탐색하는가, 결론만 받는가?
   ├─ 탐색 → 인터랙티브 + 보조 데이터 노출
   └─ 결론 → 정적 + 강조 1개 + 직접 레이블

[질문 4] 색·범례·테두리 중 빼도 메시지가 살아남는 요소가 있는가?
   └─ YES → 빼라 (§7 ②)
```

## 1. 시각화 목적 정의

> **왜 중요한가** — 목적이 모호하면 차트는 "데이터를 그린 그림"에 그친다. Knaflic은 "차트는 결론을 보여주는 도구이지, 데이터를 보여주는 도구가 아니다"라고 말한다.

| 질문 | 선택지 | 결과 |
| --- | --- | --- |
| 무엇을 하는 시각화인가? | EDA(탐색) | 데이터의 모든 면을 본다. 미적 완성도 ↓, 정보 밀도 ↑ |
|  | 커뮤니케이션(전달) | 한 메시지에 집중. 미적 완성도 ↑, 정보 절제 ↑ |
| 누가 보는가? | 분석가 본인 | 인터랙티브 가능 |
|  | 의사결정자 | 정적 + 결론 명시 |
|  | 외부 청중 | 맥락 + 결론 + 비교 기준 모두 필요 |

**✅ 좋은 예** — "Q1 이탈률이 전 분기 대비 3.2pp 늘어난 원인이 신규 가입 코호트 때문임을 보여주려 한다."
**❌ 나쁜 예** — "분기별 이탈률을 보여주려 한다."

## 2. 시각 인지 기초 — 사람은 차트를 어떻게 읽는가

### 2.1 전주의적 속성 (Pre-attentive Attributes)

뇌가 **의식하기 전(<200ms)** 에 자동으로 처리하는 시각 요소들. 이 요소를 활용하면 독자가 "찾는" 게 아니라 "보이는" 차트가 된다 (Stephen Few).

| 속성 | 예시 | 정량 비교 정확도 |
| --- | --- | --- |
| 위치 (공통 척도) | 같은 축 위 점·막대 | ★★★★★ |
| 길이 | 막대 차트 | ★★★★ |
| 기울기·각도 | 선의 방향 | ★★★ |
| 면적 | 버블 크기 | ★★ |
| 색의 명도 | 열지도 | ★★ |
| 색상 (Hue) | 카테고리 구분 | ★ |
| 모양 | 산점도 마커 | ★ |

> Cleveland & McGill (1984) 의 정확도 순위. **"비교"가 목적이라면 길이·위치가 색·면적보다 항상 정확하다.** 그래서 파이 차트(각도)보다 막대 차트(길이)가 비교에 강하다.

**활용 규칙**

- **하나의 속성 = 하나의 의미** — 색은 카테고리, 크기는 수량처럼 분리
- 강조할 데이터에는 두 개 이상 속성을 동시에 적용 (색 + 굵기)
- 여러 속성을 동시에 다른 의미로 쓰면 인지 부하가 폭증한다

### 2.2 게슈탈트 원리 (Gestalt Principles)

사람이 시각 요소를 그룹으로 묶어서 인지하는 방식. 잘 쓰면 범례 없이도 의미가 전달된다.

| 원리 | 설명 | 차트 적용 |
| --- | --- | --- |
| 근접성 (Proximity) | 가까이 있는 것끼리 묶어 본다 | 그룹 막대 사이 간격 조정 |
| 유사성 (Similarity) | 같은 색·모양·크기는 같은 그룹 | 같은 카테고리에 같은 색 |
| 폐쇄성 (Closure) | 빈 곳을 메워 형태로 인식 | 점선·반투명으로 보조 표시 |
| 연속성 (Continuity) | 부드럽게 이어진 흐름을 따라간다 | 선 그래프, Sankey |
| 둘러쌈 (Enclosure) | 테두리·배경색으로 묶인 것은 같은 그룹 | 강조 영역, 주석 박스 |
| 공동 운명 (Common Fate) | 같은 방향·속도로 움직이는 것은 같은 그룹 | 애니메이션·스몰 멀티플스 |
| 초점 (Focal Point) | 다른 것과 구분되는 요소가 시선을 끈다 | 강조 1개에 다른 색 |

> **실전 적용** — 범례를 지우고 데이터 옆에 라벨을 붙이는 §6 ④ 원칙은 "근접성"의 활용이다. 강조 1개에 색을 입히는 §7 ① 원칙은 "초점"의 활용이다.

## 3. 차트 작성 4단계 프로세스

> 잘 만든 차트는 방해요소 없이 독자가 데이터를 비교·대조하여 결론을 이끌어낸다. — Tufte

1. **조사 (Investigate)** — 데이터의 범위·결측·이상치·단위를 먼저 파악한다
2. **편집 (Edit)** — 핵심 메시지를 찾고, 그 메시지를 드러낼 데이터만 선택한다
3. **구성 (Compose)** — §4 표를 따라 데이터 관계에 맞는 차트 유형을 결정한다
4. **검토 (Review)** — 독자 시점에서 보고, 특이값·오해 소지·접근성을 확인한다

**편집 규칙**

- 비교 대상(작년·목표·평균 등)을 설정해 맥락을 제공하라
- 반올림은 분석이 끝난 후 마지막에 한다 (계산 단계 반올림은 누적 오차)
- 각 차트는 핵심 메시지에 필요한 정보만 담는다 — "있으면 좋은" 데이터는 빼라

**검토 체크리스트**

- [ ] 차트 제목이 메시지인가, 주제인가? (메시지여야 함, §11)
- [ ] 색이 의미를 갖는가, 장식인가?
- [ ] 결측·이상치가 처리되어 있는가? (§9)
- [ ] 그레이스케일로 출력해도 구분되는가?
- [ ] 모바일/임베드에서 11pt 이상으로 읽히는가?

## 4. 차트 유형 선택

> 차트 유형은 데이터의 관계를 기반으로 선택한다. 미적 이유로 바꾸지 않는다.

| 보여주려는 관계 | 추천 차트 | 피해야 할 차트 | 이유 |
| --- | --- | --- | --- |
| 시간에 따른 변화 | 선 그래프, 영역 차트 | 파이 차트 | 파이는 시간 축 표현 불가 |
| 항목 간 비교 | 막대 차트 | 3D 차트, 도넛 | 3D는 길이·각도 왜곡 |
| 비율·구성 | 도넛, 누적 막대, 트리맵 | 파이(5개 초과) | 인간은 각도보다 길이를 잘 비교 |
| 두 변수 관계 | 산점도, 버블 차트 | 선 그래프 | 선은 인과·연속성을 암시 |
| 분포 확인 | 히스토그램, 박스플롯, 바이올린 | 막대 차트 | 막대는 이산값, 히스토그램은 연속값 |
| 순위 비교 | 수평 막대, 덤벨 차트 | 선 그래프 | 선은 순서가 아닌 흐름 |
| 지리 기반 | 등치역도, 버블맵 | 일반 차트 | 위치 정보가 핵심일 때 |
| 밀도·패턴 | 열지도(Heatmap) | 파이 차트 | 2차원 격자가 필요 |
| 누적 변화 | 워터폴 차트 | 일반 막대 | 시작값 → 변화 → 끝값 흐름 |
| 단계별 감소 | 퍼널 차트 | 막대(순서 무시) | 선형 프로세스 명시 |
| 흐름·전환 | 생키 다이어그램 | 퍼널(분기 못함) | 분기·합류가 있는 경로 |
| 부분-전체 계층 | 트리맵, 선버스트 | 파이 (3단계 이상) | 계층 구조 표시 |

### 4.1 선 그래프

- 한 차트에 선 4개 이하 — 초과 시 스몰 멀티플스(small multiples)로 분리
- 선 아래 음영은 Y축 기준선이 0일 때만 허용
- 데이터가 차트 높이의 약 2/3를 차지하도록 Y축 범위를 설정한다
- Y축은 0에서 시작하고 자연스러운 증가분(10, 25, 50 등)을 사용한다
- **이중축은 피한다** — 두 데이터의 상관을 인위적으로 만든다 (대안: 인덱스 100 재기준선, §8)
- 범례는 선 끝점의 순위 순으로 배열한다 (독자 시선이 멈추는 위치)

> **✅ 좋은 예** — 5년치 매출/이익 추이, Y축 0부터 시작, 매출 강조색·이익 회색, 우측 끝에 직접 레이블
> **❌ 나쁜 예** — Y축이 데이터 범위에서 시작해 변화가 과장됨, 범례가 알파벳순, 선 8개

### 4.2 막대 차트 (수직)

- **기준선은 반드시 0에서 시작한다** — 가장 흔한 왜곡 패턴 (Wilke ch. 25)
- 막대 너비는 막대 간 간격의 약 2배
- 3차원 막대는 쓰지 않는다
- 한 차트의 모든 막대는 색·음영을 통일한다 (예상값·강조값만 예외)
- 다중 막대 그룹 내 막대는 4개 이하
- 값이 0에 가까운 막대는 막대 위에 수치를 표시한다

### 4.3 수평 막대 차트

- 항목명이 길면(8자 이상) 수직 대신 수평을 쓴다
- 크기 순(내림차순 또는 오름차순)으로 정렬한다 — **무작위·알파벳 배열 금지**
- 시간 순 표현 시 최근 데이터를 위에 둔다
- 음수는 왼쪽, 양수는 오른쪽 — 양방향 수평 막대(diverging)는 설문/감성 점수에만 제한적으로 사용

### 4.4 파이 차트

- 조각은 5개 이내 — 그 이상은 누적 막대
- 가장 큰 조각은 12시 기준 오른쪽, 두 번째는 왼쪽에 배치
- 3차원·기울인 파이는 쓰지 않는다
- 조각을 또 세분해야 하면 분할 막대 차트(stacked bar)로 변경
- 데이터가 하나라도 빠지면(합이 100% 미만) 사용하지 않는다

> ⚠️ Few(2012)는 "파이 차트는 거의 항상 막대 차트로 대체 가능하다"고 본다. 비율 비교가 핵심이면 막대를 우선 검토하라.

### 4.5 히스토그램

- 연속적인 수치 데이터에만 사용한다 (이산 데이터는 막대 차트)
- 구간(bin)은 자연스러운 단위 (10, 25, 100 등)
- 구간 수는 √N 또는 Sturges 공식 참고

### 4.6 박스플롯 / 바이올린

- 여러 그룹의 분포를 비교할 때 나란히 배치
- 표본 수가 작으면(N<20) 점 분포(strip plot, beeswarm) 병행
- 분포 형태(이중 봉우리 등)가 중요하면 바이올린 플롯 사용

### 4.7 열지도 (Heatmap)

- 색상 스케일은 단방향 적용 (가장 옅은 색 → 가장 짙은 색)
- 발산(diverging) 데이터(0 기준 양/음)일 때만 양방향 색상
- 높은 값은 따뜻한 색, 낮은 값은 차가운 색 (단, 의미가 반대인 데이터는 반전)
- 셀에 값을 직접 표기하면 "표 + 시각화"로 정확도와 패턴을 모두 잡는다

### 4.8 픽토그램

- 값이 비슷한 데이터에는 쓰지 않는다 (식별 불가)
- 아이콘의 면적·높이로 수량을 표현하지 않는다 — **개수로만**
- 데이터 값을 항상 표시한다
- 아이콘은 5개 또는 10개 단위로 묶는다

### 4.9 워터폴 차트 (Waterfall)

> **언제 쓰나** — 시작값 + 일련의 가감 → 끝값. 변화의 구성 요소를 추적할 때.

- 사용 예: 시작 MRR + 신규 − 이탈 − 다운그레이드 = 끝 MRR
- 증가는 +색(초록), 감소는 −색(빨강), 시작·끝은 회색
- 누적 효과를 따라가도록 막대 사이를 점선으로 연결한다
- 항목 수는 7개 이내, 그 이상은 묶거나 별도 차트

### 4.10 퍼널 차트 (Funnel)

> **언제 쓰나** — 단계가 정해진 선형 프로세스의 각 단계 인원·전환율.

- 단계는 반드시 **시간·논리 순서**로 정렬 — 무작위 정렬 금지
- 각 단계 옆에 절대값과 전환율을 함께 표기 (1,000 → 350, 35%)
- 사용자가 단계를 건너뛰거나 분기되는 경우는 퍼널 ❌, **생키** 사용
- 막대 폭 = 인원 비율 (절대값이 아니라 비율로 그리는 게 표준)

### 4.11 생키 다이어그램 (Sankey)

> **언제 쓰나** — 흐름이 분기·합류하는 경로. 에너지·자금·사용자 여정.

- 노드(stage) 너비는 통과 인원·양에 비례
- 시각적 혼잡도 관리: 5개 미만 주요 흐름만 강조, 나머지는 회색
- 색은 **출발지** 또는 **도착지** 기준으로 통일 (혼용 금지)
- 너무 많은 경로는 가독성 폭락 — 상위 경로만 표시하고 나머지는 "기타"로 묶기

### 4.12 트리맵 (Treemap)

> **언제 쓰나** — 부분-전체 + 계층 구조를 한 화면에. 카테고리별 비중 비교.

- 같은 레벨에서는 같은 색조, 다른 레벨은 명도로 구분
- 셀이 너무 작아지면(<5%) 별도 표로 분리
- 라벨은 셀 안에 들어갈 만큼만 표기, 작은 셀은 툴팁/주석으로

## 5. 색상 원칙

> **왜 색이 어려운가** — 색은 가장 강력한 시각 변수이자 가장 오용되는 변수다. 색은 의미를 가질 때만 쓴다.

### 5.1 팔레트 3종 — 데이터 종류에 맞게

| 팔레트 | 데이터 종류 | 예시 |
| --- | --- | --- |
| **Sequential (순차형)** | 양/크기 — 0 → 큰 값 | 인구밀도, 매출액, 점수 |
| **Diverging (발산형)** | 중앙값 기준 양/음 | 전월 대비 증감, 평균 대비 편차 |
| **Categorical (범주형)** | 순서 없는 그룹 | 제품, 지역, 부서 |

**Sequential** — 옅은 색 → 짙은 색의 한 방향. 색상은 보통 단일 hue (파랑·초록 등) 또는 인접한 두 hue.

**Diverging** — 양 끝에 대조 색(파랑↔빨강 등), 중앙은 회색·연한 노랑. 0 기준점이 명확해야 한다.

**Categorical** — 시각적으로 구분되는 hue들. **6~8개가 인지 한계** — 그 이상은 묶거나 색을 포기하고 다른 변수 사용.

> **흔한 실수** — 카테고리에 sequential 색을 쓰면 "순서가 있어 보이는" 오해를 만든다. 매출액은 sequential, 부서는 categorical.

### 5.2 의미별 토큰

| 용도 | 색상 | 비고 |
| --- | --- | --- |
| 주요 데이터 | 브랜드 컬러 1개 | 차트당 하나 |
| 보조 데이터 | 회색 (`#9E9E9E` 계열) | 맥락·비교 기준 |
| 경고·이상 | 빨강 계열 1개 | 부정·하락에 한정 |
| 긍정·달성 | 초록 계열 1개 | 목표 달성·상승에 한정 |
| 중립 정보 | 파랑 계열 | 정보·링크 |

### 5.3 핵심 규칙

- 동시에 사용하는 색상은 **최대 5개 이하** (categorical은 6~8까지)
- 정보 전달 목적으로만 색을 사용한다 — 장식 금지
- 특정 테마를 연상시키는 조합은 피한다 (예: 빨강+초록 → 크리스마스, 정치 진영 등)
- 팔레트는 기본 색상마다 3~5가지 색조로 구성하고 전체에서 일관되게 사용
- 다중 막대: 같은 색의 명도 변화 또는 색상환 같은 쪽 색상 — **보색 조합 금지**
- 동일 데이터는 모든 차트에서 동일 색상
- 비교 대상(예: 목표 vs 실적)은 항상 같은 색상 페어 유지

### 5.4 접근성 (WCAG)

- 텍스트 명도 대비 ≥ **4.5:1** (WCAG AA)
- 차트 그래픽 요소(막대 경계·축선 등) ≥ **3:1** (WCAG 1.4.11)
- **색상만으로 정보를 전달하지 않는다** (WCAG 1.4.1) → 패턴·레이블·아이콘 병행
- 색맹 친화 팔레트 우선: **Blue-Orange 조합** (Wong, 2011)
- 적록색맹: 전체 인구 약 8% (남성), 0.5% (여성)
- 그레이 스케일로 변환했을 때도 구분 가능한지 확인 (출력·복사 시 필수)
- **검증 도구** — WebAIM Contrast Checker, Coblis 색맹 시뮬레이터, 차트 그레이스케일 캡처

> **✅ 좋은 예** — 회색 막대 5개 중 강조 1개만 브랜드 색
> **❌ 나쁜 예** — 무지개 색 막대 7개, 모든 항목이 동등하게 강조됨 (= 아무것도 강조 안 됨)

## 6. 타이포그래피 & 레이블링 & 주석

| 요소 | 스타일 |
| --- | --- |
| 차트 제목 | 볼드, 14~16pt |
| 부제 (서브카피) | 보통체, 12~13pt, 회색 |
| 축 레이블 | 보통체, 10~11pt, 회색 |
| 데이터 레이블 | 11~12pt, 데이터 바로 옆 또는 위 |
| 주석 (Annotation) | 10~11pt, 강조색 또는 회색 |
| 출처·각주 | 9~10pt, 회색, 차트 좌하단 |

### 6.1 글꼴

- 볼드체와 이탤릭체를 동시에 쓰지 않는다
- 수직 막대의 레이블은 **기울여 배치하지 않는다** — 길면 수평 막대로 전환
- 반전 효과(검은 바탕 + 흰 글씨) 금지 (인쇄·복사 시 깨짐)
- 글꼴 색은 검정 또는 짙은 회색 (#333 권장)
- 한글: 본명조·노토 산스 / 영문: Inter, Roboto, Source Sans

### 6.2 레이블 배치

- 단위는 축 제목에 한 번만 표기 (각 눈금에 반복 금지)
- **범례는 가능하면 제거**하고 데이터 직접 레이블로 대체 (시선 이동 ↓)
- 항목명은 해당 데이터 바로 옆에 붙인다
- 항목명은 한 문장 이내로 간결하게
- 최소 11pt, 레이블 겹침 방지

### 6.3 차트 제목·부제 — 결론 vs 주제

두 가지 접근이 있다. **AI 리포트에서는 결론형(Takeaway)이 표준.**

| 유형 | 예시 | 사용 상황 |
| --- | --- | --- |
| 주제형 (Descriptive) | "분기별 매출" | 탐색용 EDA, 대시보드 위젯 |
| 결론형 (Takeaway) | "Q1 매출, 전년 대비 12% 감소" | 보고서, AI 리포트, 슬라이드 |

**제목+부제 패턴 (권장)**

```
제목: 신규 가입 코호트의 30일 잔존율이 10pp 떨어졌다  (결론)
부제: 2026년 Q1 vs Q4, n=12,400                   (출처·맥락)
```

- 제목은 한 문장, 30자 내외
- 부제는 데이터 출처·기간·표본 크기
- 제목이 결론을 담을 때 독자의 인지 부하가 가장 낮다 (ScienceDirect, 2020)

### 6.4 주석 (Annotation) — 차트 안에서 설명하기

차트는 그림 + 글이다. 주석을 적극적으로 쓰면 차트가 스스로 말한다.

| 패턴 | 사용 상황 |
| --- | --- |
| 기준선 (Reference Line) | 목표·평균·전년 동기 비교 |
| 음영 영역 (Band) | 신뢰구간, 정상 범위, 이벤트 기간 |
| 콜아웃 박스 | 특정 데이터 포인트 부연 설명 |
| 화살표 + 텍스트 | 추세 변화·이상치 지목 |
| 이벤트 마커 | 출시일, 정책 변경 시점 |

**원칙**

- **꼭 필요한 곳에만** — 모든 점을 라벨링하지 말고 핵심만
- 일관된 스타일 (글꼴·색·위치)
- 콜아웃은 데이터를 가리지 않는다
- 직접 라벨이 우선, 주석은 보조
- 라인 차트의 라벨은 **선의 끝점**에 (좌측 범례 회피)

### 6.5 숫자 표기

- **연도** — 공간이 있으면 전체(2024), 없으면 첫 연도만 4자리·나머지는 끝 2자리(2024, '25, '26)
- **분기** — 항상 연도를 병기 (Q1 24)
- **정수와 소수** — 오른쪽 정렬 (소수는 소수점 기준 정렬)
- **단위** — 해당 수에 맞는 가장 높은 단위 (1,200,000 → 1.2M)
- **회사명·기관명** — 축약하지 않는다 (대중적 약어만 예외: SK, LG 등)

## 7. 디자인 6원칙

Knaflic의 *Storytelling with Data* 핵심 원칙을 우리 환경에 맞게 정리.

**① 회색조로 시작하라**
전체를 회색조로 그린 뒤, 강조할 데이터 1개에만 색을 입힌다. → 시선이 자연스럽게 핵심으로 간다.

**② 불필요한 요소를 제거하라**
범례, 눈금선, 테두리, 배경색, 그림자 — 기본값을 모두 의심하라. *"data-ink ratio를 높여라"* (Tufte).

**③ 중요한 데이터를 두드러지게 하라**
강조 데이터는 색·굵기·크기로 시각적 위계를 만든다. **선택과 집중.**

**④ 텍스트를 차트 안으로 통합하라**
항목명은 범례 대신 데이터 바로 옆. 결론은 제목으로. 주석은 화살표로.

**⑤ 스몰 멀티플스를 고려하라**
한 차트에 너무 많이 담지 말고, 항목별로 작은 차트를 격자로 배치한다.

**⑥ 복잡도는 새 정보가 있을 때만 추가하라**
그래픽 요소는 꼭 필요한 만큼만. *"Less is more, but only if less is more."*

## 8. 통계·수치·불확실성 표현

### 8.1 통계량 선택

| 통계량 | 사용 상황 |
| --- | --- |
| 평균값 | 전체 데이터의 총체적 영향력 |
| 중앙값 | 이상치에 영향받지 않는 대표값이 필요할 때 |
| 최빈값 | 가장 자주 나타나는 값 |
| 표준편차 | 데이터 분포의 촘촘함·흩어짐 |
| 이동평균 | 변동성이 커서 추세 파악이 어려울 때 |
| 가중평균 | 중요도가 다른 데이터 포인트 비교 |

> ⚠️ **분포가 치우쳐 있다면 평균은 거짓말한다.** 소득·체류시간 등은 중앙값 우선.

### 8.2 로그 스케일

- **X축** — 짧은 기간 내 세부 변화를 담을 때
- **Y축** — 변화폭의 상대적 중요도(배수 변화)를 보여줄 때
- 일반 독자는 로그 스케일을 오해한다 — **반드시 라벨에 명시** ("로그 스케일")

### 8.3 증감률 & 기준선

- 증감률: `(신규값 - 기존값) / 기존값 × 100%`
- 기준선 100 재설정: `(현재값 / 초기값) × 100`
- 기준선 0 재설정: `{(현재값 / 초기값) × 100} - 100`
- 증감률을 말할 때는 **기준값을 반드시 명시**한다 (예: "전월 대비", "YoY")
- 규모가 다른 두 개체의 증감률은 직접 비교하지 않는다 (1억의 100% vs 1조의 1%)
- 값이 고만고만하면 절대값 대신 차이값·증감률로 차트를 구성한다

### 8.4 백분율

- 두 백분율의 차이는 **퍼센트 포인트(pp)** 또는 베이시스 포인트(bp)로 표현
    - "이탈률이 5%에서 8%로" → "3pp 증가" (60% 증가가 아님)
- 백분율의 평균을 구하지 않는다 — 원래 데이터로 돌아가 새로 계산
- A%의 B% = `A/100 × B%`
- **독자를 대신해 계산하라.** 결과만 보여줘라.

### 8.5 불확실성 시각화

> 점추정만 보여주면 독자는 "정확한 값"이라고 오해한다. 추정·예측에는 **반드시** 불확실성 정보를 곁들인다.

| 표현 방식 | 사용 상황 |
| --- | --- |
| 에러바 (Error bar) | 막대·점 차트의 단일 추정값 |
| 신뢰구간 밴드 (CI Band) | 선 그래프의 시계열 추정 |
| 등급 에러바 (Graded) | 50%/80%/95% 신뢰구간을 농도로 |
| 박스플롯 | 분포 자체를 보여줄 때 |
| 다중 시뮬레이션 선 | 불확실성이 큰 예측 (예: 강수량) |

**원칙**

- 에러바가 무엇을 의미하는지 **항상 캡션·라벨에 명시** — 표준편차? SE? 95% CI?
- 같은 차트에서 서로 다른 의미의 에러바를 섞지 않는다
- 에러바가 너무 많아 시각적 혼잡이라면, 핵심 시리즈만 표시하거나 분리
- 예측·추정값은 **점선·회색**으로, 실측값과 구분
- AI 리포트에서 모델이 추정한 값은 §11.5 환각 방지 규칙을 따른다

## 9. 엣지 케이스 대응

### 9.1 결측 데이터

- 차트로 만들 가치가 있는지 먼저 판단한다 (데이터 완결성 < 50%면 표·텍스트로)
- **막대 차트** — 빠진 부분은 비워두고 각주를 단다
- **선 차트** — 간격을 메우는 선을 잇는다. 짧은 구간이면 데이터 포인트만 표시
- **파이 차트** — 조각이 하나라도 빠지면 사용하지 않는다

### 9.2 특이값 (Outlier)

끊어진 막대(broken bar)를 사용하려면 아래 4가지를 **모두** 충족해야 한다:

- [ ] 특이값의 정확한 수치를 데이터 출처가 제공한다
- [ ] 막대가 최소 10개이고 특이값은 단 1개다
- [ ] 특이값과 두 번째로 큰 값의 차이가 3배 이상이다
- [ ] 특이값이 핵심 메시지가 아니다

→ 하나라도 충족 안 하면 끊지 말고 그대로 보여준다.

### 9.3 최근 변화 포착

장기 시계 차트에서 최근의 작은 변화가 보이지 않을 때 → 해당 구간을 확대해 별도 차트로 삽입한다 (focus + context).

### 9.4 그리드선 & 표

- 표가 작으면 줄 간 음영·선은 불필요
- 행이 많으면 3~5행마다 가는 선 1개
- 표 안에 공간이 있으면 핵심 데이터를 막대 차트로 삽입(in-cell bar)

## 10. 안티 패턴 — 자주 보는 잘못된 차트

> 의도가 없어도 결과적으로 독자를 속이는 패턴들. 대부분 "기본값"에서 시작된다.

### 10.1 잘린 Y축 (Truncated Y-axis)

❌ **문제** — 막대 차트의 Y축이 0에서 시작하지 않으면 5% 차이가 300% 차이처럼 보인다
✅ **해법** — 막대는 무조건 0부터. 변화율을 강조하려면 별도의 증감률 차트로 분리

> 선 차트는 예외 — 0에서 시작할 필요 없지만, "데이터 높이의 2/3" 규칙은 지킨다.

### 10.2 3D 차트

❌ **문제** — 원근법으로 길이·각도가 왜곡된다. 뒤쪽 막대가 가려진다
✅ **해법** — 2D만 쓴다. 3D는 마케팅 자료에서도 권장하지 않는다

### 10.3 파이 5조각 초과

❌ **문제** — 인간은 각도 비교가 약하다. 5개 넘으면 누가 큰지 모른다
✅ **해법** — 누적 막대, 트리맵, 또는 정렬된 수평 막대

### 10.4 이중 Y축

❌ **문제** — 두 데이터의 단위가 다른데 한 차트에 겹쳐 그리면 인위적 상관이 만들어진다
✅ **해법** — 인덱스 100 재기준선, 두 차트 분리, 또는 산점도

### 10.5 무지개 색

❌ **문제** — 카테고리에 무지개 색을 쓰면 모든 항목이 같은 강조 → 메시지 없음
✅ **해법** — 회색 베이스 + 강조 1개 또는 2개

### 10.6 정렬되지 않은 막대

❌ **문제** — 알파벳·무작위 순으로 막대 배치 → 비교 불가
✅ **해법** — 크기 순 정렬 (시간 차원이 있는 경우만 시간 순)

### 10.7 면적·부피 비례

❌ **문제** — 픽토그램·버블의 "지름"으로 값을 표현하면 면적은 제곱으로 커져 과장된다
✅ **해법** — 면적이 값에 비례하도록 계산. 픽토그램은 개수로만

### 10.8 체리 피킹 (구간 선택)

❌ **문제** — 유리한 구간만 잘라 보여주기 (예: 최근 1주만)
✅ **해법** — 충분한 맥락 구간을 함께 표시. 분석 기간 명시

### 10.9 중복 강조

❌ **문제** — 굵은 색 + 큰 폰트 + 박스 + 화살표 + 별표 → 모두 강조되면 강조 없음
✅ **해법** — 강조 요소는 **2개 이내**로 조합

### 10.10 잘못된 차트 유형

❌ **문제** — 시간 변화에 파이, 분포에 막대, 비교에 라인
✅ **해법** — §4 표를 따른다. 미적 이유로 바꾸지 않는다

## 11. AI 리포트 특화 원칙

> **차트의 역할이 다르다** — AI 리포트에서 차트는 단독 매체가 아니라, AI 요약(텍스트)이 말한 내용을 **시각으로 확인**시키는 보조 매체다. 차트가 메시지를 만들지 않고, 메시지를 검증한다.

### 11.1 메시지 정렬

- 차트 단독으로 메시지를 전달하려 하지 않는다
- 제목은 "무엇을 보여주는가"가 아니라 **"무엇을 의미하는가"**로 작성
    - ❌ "월별 매출 추이"
    - ✅ "3월 매출, 전월 대비 12% 감소"
- 차트 제목 = AI 요약의 핵심 문장과 일치해야 한다

### 11.2 인터랙션 부재 대응

- 인터랙티브 없음 → 툴팁 대신 **직접 레이블** 표기
- 애니메이션 없음 → **시각적 위계**로 강조 (굵기, 색상, 크기)
- 호버·필터 없음 → 보조 데이터는 회색·작은 텍스트로 항상 노출

### 11.3 화면·임베드 제약

- 화면 크기 고정 (모바일·임베드·메일) → 폰트 **최소 11pt**, 레이블 겹침 방지
- 다크 모드 대응 — 색상 대신 의미 토큰 사용 (`color/positive`, `color/critical`)
- 메일 클라이언트 호환 — 차트는 **PNG로 출력**, SVG는 일부 클라이언트에서 깨짐
- 너비 가정: 메일 600px, 모바일 360px, 임베드 800px

### 11.4 한국어 UX

- **숫자 포맷** — 한국·미국 `1,000` / 유럽 `1.000` (혼용 금지)
- **단위 표기** — 천(K) 대신 만/억 단위가 한국 독자에게 직관적이면 우선 (예: 1.2억, 3,500만)
- **날짜 포맷** — `YYYY-MM-DD` 통일 또는 지역별 분기 (한국: 2026.04.27)
- **요일** — 월/화/수 한 글자 표기로 공간 절약
- **영문 레이블 대응** — 영문은 한국어보다 약 30~40% 길어진다. 다국어 지원 시 공간 여유 확보
- **문장부호** — 한국어는 가운뎃점(·)·낫표(「 」) 활용, 콜론(:)은 영문보다 자제

### 11.5 AI 환각 방지

- 차트에 표시된 모든 수치는 **데이터 출처 근거가 있어야** 한다
- AI가 보간(interpolation)·추정한 값은 **점선·회색**으로 구분
- 예측 구간은 **신뢰구간 밴드**로 표시, 점추정만 단독 노출 금지 (§8.5)
- 출처 미상 데이터는 차트에서 제외하고 텍스트 주석으로

## 12. 대시보드 설계 원칙

> Stephen Few — *"대시보드는 한눈(at-a-glance)에 핵심 지표를 모니터링하기 위한 도구다."* 보고서·분석과 다르다.

### 12.1 한눈에 보이는가

- **스크롤 없이 한 화면**에 핵심 지표가 모두 들어와야 한다
- 모니터링이 목적이면 차트당 3~5초 안에 상태를 파악할 수 있어야 한다
- 깊은 분석은 별도 페이지·드릴다운으로 분리

### 12.2 정보 위계

```
[1순위] 가장 중요한 지표 — 좌상단, 큰 영역
[2순위] 보조 지표 — 우측·하단
[3순위] 컨텍스트 지표 — 작은 위젯, 회색
```

- 사람의 시선은 **F 패턴 또는 Z 패턴**으로 이동 — 좌상단이 황금 자리
- 같은 카테고리 위젯은 그룹으로 묶기 (게슈탈트 근접성)

### 12.3 차트 선택 — 모니터링용

| 목적 | 추천 |
| --- | --- |
| 단일 KPI | 큰 숫자 + 비교(전기간) + 스파크라인 |
| 시계열 추세 | 작은 선 그래프 + 임계선 |
| 카테고리 비교 | 가로 막대 (소규모) |
| 상태(정상/경고) | 신호등 색상 + 아이콘 |
| 진척도 | 게이지 또는 진행 막대 |

### 12.4 피해야 할 13가지 (Few)

요약하자면:

1. 한 화면 초과
2. 컨텍스트 부족 (수치만 있고 비교 없음)
3. 정밀도 과잉 (소수점 4자리)
4. 부적절한 차트 (게이지 남발)
5. 과한 시각 효과 (3D·그라데이션)
6. 중요한 정보 흐릿하게
7. 데이터 정확도보다 미관 우선
8. 색 남발
9. 채워진 공간만 가득
10. 잘못 정렬·배치
11. 중요도 무관하게 같은 크기
12. 의미 없는 색상 사용
13. 통일성 부족

### 12.5 새로 고침·실시간

- 데이터 갱신 시각을 항상 표기 ("최근 업데이트: 09:32")
- 실시간 갱신은 깜빡임 없이 부드럽게
- 오래된 데이터는 회색·경고 아이콘으로 명시

| 상황 | 결정 |
| --- | --- |
| 데이터가 5개 초과 카테고리 | 파이 → 막대 |
| Y축 범위가 좁아 변화가 안 보임 | 일반 → 인덱스 100 재기준선 |
| 항목명이 길다 | 수직 막대 → 수평 막대 |
| 여러 그룹 비교 | 한 차트 → 스몰 멀티플스 |
| 두 변수 단위가 다름 | 이중축 ❌ → 인덱스화 또는 두 차트 분리 |
| 시계열 + 노이즈 큼 | 원시값 → 이동평균 |
| 분포가 치우침 | 평균 → 중앙값 |
| 메시지가 안 보임 | 색·굵기·크기로 강조 1개만 |
| 단계가 분기됨 | 퍼널 → 생키 |
| 시작값 → 끝값 변화 추적 | 일반 막대 → 워터폴 |
| 카테고리 + 비중 + 계층 | 파이 → 트리맵 |
| 추정값을 보여줌 | 단일 점 → 신뢰구간 밴드 |

## 부록 B. 전주의적 속성 정확도 순위 (Cleveland & McGill)

비교가 목적이라면 위쪽 속성을 우선 사용한다.

```
정확도 ↑
1. 공통 척도 위 위치     ████████████ (최고)
2. 비공통 척도 위 위치   ███████████
3. 길이                  ██████████
4. 방향·각도             ████████
5. 면적                  █████
6. 부피                  ███
7. 색의 명도·채도        ██
8. 색상 (Hue)            ▌
정확도 ↓
```

**해석** — 막대 차트(길이, 3위)가 파이 차트(각도, 4위)보다 비교에 강한 이유.
면적·부피·색은 정확한 비교가 아닌 **카테고리 표시·강조**에 적합.

## 부록 C. 용어집

- **스몰 멀티플스 (Small Multiples)** — 같은 형식의 작은 차트를 격자로 배치해 카테고리별로 비교하는 방법
- **데이터-잉크 비율 (Data-Ink Ratio)** — Tufte의 개념. 데이터 표현에 쓰인 잉크 / 전체 잉크. 높을수록 좋다
- **퍼센트 포인트 (pp)** — 두 백분율의 차이. 5%→8%은 3pp 증가
- **베이시스 포인트 (bp)** — 0.01%p. 금융에서 사용
- **Diverging 색상** — 0 또는 중앙값 기준으로 양방향 그라데이션
- **인덱스 100 재기준선 (Indexing)** — 기준 시점 값을 100으로 잡고 다른 값을 비율로 환산
- **전주의적 속성 (Pre-attentive Attributes)** — 의식 전(<200ms)에 처리되는 시각 속성
- **게슈탈트 원리 (Gestalt Principles)** — 시각 요소를 그룹으로 묶어 인지하는 심리학적 원리
- **Sequential·Diverging·Categorical** — 색 팔레트 3종 (양·발산·범주)
- **신뢰구간 밴드 (CI Band)** — 선 그래프 주변에 음영으로 그린 불확실성 영역
- **에러바 (Error Bar)** — 점·막대에 부착된 불확실성 표시 막대
- **워터폴 / 퍼널 / 생키 / 트리맵** — §4 참조
