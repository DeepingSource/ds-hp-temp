SEAL SDK

비전 AI는 사람을 봅니다.
그 순간, 신원을 지우는 SDK

GDPR도, CCPA도, 국내 개인정보보호법도 얼굴과 신원을 그냥 두지 않습니다. SEAL SDK는 입력 시점에 신원을 지우고, 누구가 아니라 무엇을 어떻게만 남깁니다. 익명화·인식·공간 분석을 모듈로 제공해, 기존 시스템에 그대로 내장합니다.

예시 화면
seal.init.ts
예시
import { SEAL } from "@saai/seal"
const seal = SEAL.init({ mode: "on-device", retain: "none" })
seal.anonymize(["face", "plate"])          // irreversible at capture
const stream = seal.connect(camera)
stream.on("signal", (s) => {
  // Pattern · Detection · Priority · Response · Context · Outcome
  hub.emit(s.category, s.payload)
})
출력$ seal run
    캡처 시점에 얼굴·번호판 제거

    The SEAL Promise

    네 글자에 담은 설계 원칙

    SSecure

    입력 시점에 신원을 지웁니다. 원본 영상은 시스템 어디에도 남지 않고, 익명화된 데이터에서 신원을 되찾을 수 없습니다.

    EEmbeddable

    엣지 장치·온프레미스 서버·기존 애플리케이션에 모듈 단위로 그대로 내장합니다. 영상은 밖으로 나가지 않습니다.

    AAdaptable

    여러 사람을 동시에, 동의 없이도 보호합니다. 입력 소스와 보호 강도를 현장에 맞춰 조정합니다.

    LLightweight

    실시간 스트림을 그 자리에서 익명화하는 경량 런타임. 제약된 환경에서도 그대로 돌아갑니다.

    Integration

    통합 개요

    입력 연결부터 결과 소비까지 단계별 모듈로 구성되어, 필요한 단계만 골라 조합합니다. 익명화가 항상 첫 단계입니다.

    step 1

    SDK 설치

    대상 런타임에 SDK를 추가하고 처리 노드를 초기화합니다.

    step 2

    입력 연결

    RTSP·파일·프레임 스트림 등 표준 입력 소스를 연결합니다.

    step 3

    익명화 우선

    입력 시점에 신원을 지운 뒤, 인식·공간 분석을 필요에 맞게 잇습니다.

    step 4

    결과 소비

    원본 없이, 익명화된 스트림과 분석 신호만 애플리케이션에서 받아 활용합니다.

    FAQ

    자주 묻는 질문

    Secure, Embeddable, Adaptable, Lightweight — SDK의 4가지 설계 원칙을 나타냅니다.
    입력 시점에 제거됩니다. 분석 이전, 데이터가 시스템에 들어오는 첫 순간에 얼굴과 신원을 지우고 — 그 뒤 단계는 익명화된 데이터만 다룹니다.
    네. 화면 안 여러 사람을 동시에, 개별 동의 없이 익명화하도록 설계되어 GDPR·CCPA·국내 개인정보보호법 요건에 맞춥니다.
    보관하지 않습니다. 원본은 시스템 어디에도 남지 않고, 익명화된 결과와 분석 신호만 전달합니다.
    없습니다. 재식별이 불가능하도록 설계되어, 익명화된 데이터에서 신원을 되찾을 수 없습니다.
    익명화를 첫 단계로 두고, 그 뒤에 인식·공간 분석 모듈을 파이프라인으로 잇습니다. 누구가 아니라, 무엇을 어떻게.
    실시간 스트림을 그 자리에서 익명화하도록 설계되었으며, 처리 성능은 입력 해상도와 배치 하드웨어에 따라 달라집니다.
    Privacy by Design을 기본으로, SOC 2 인증을 충족합니다. 어디에 배치하든 같은 프라이버시 기준이 적용됩니다.

    익명화를 처음부터, 설계에 넣습니다

    운영 환경과 연동 요건을 공유해 주시면, 신원 보호를 내장한 적합한 구성을 함께 설계합니다.
